# 引き継ぎメモ

## 目的
`ai` コマンドを `session` 中心から `work` 中心へ切り替え、current work ごとの handoff / runtime state / batch / session restore を持てるようにする

## 確定事項
- 仕事状態の正本は `System/data/ai_router/` 配下に置く
- `.ai_handoff.md` は current work の mirror として扱い、案件全体の正本にはしない
- `Master/output/session_aliases.json` / `Master/output/session_restore_index.json` は session 復元用の compatibility layer として残す
- batch から `Skills/` へは自動で直書きせず、まず `skill_candidate_index.json` に候補を作り、明示レビューで昇格する
- MacBook 移行後に元 session 履歴が無い場合でも、`ai restore` は snapshot と handoff から rehydrate で継続できる状態を正とする

## 判断とその理由（最重要）
- 既存運用を壊さないため、`.ai_handoff.md` と session 復元機能は削除せず、mirror / compatibility layer として残した
- `System/scripts/ai` は既に大きいので、仕事状態ロジックは helper に逃がし、shell 側は導線だけを持つ形を維持した
- `ai work` は曖昧一致で勝手に切り替えると事故るため、query 解決は high confidence の時だけ自動選択し、それ以外は候補表示に留めた
- 長セッションの劣化は「古い会話全文を引きずること」と「何を正本に続けるかが曖昧になること」で起きるので、`session load` 判定、auto-fork、handoff compactor、snapshot-only rehydrate を優先して入れた
- handoff compactor は LLM 依存にせず、本文書き換えと優先順位づけを heuristic で強化した。これで移行後も deterministic に再圧縮できる
- skill は自動昇格させず、`skill_candidate_index.json` に 1 段階下で留め、`ai batch skill promote` の明示操作で `Skills/` に上げる形にした
- MacBook 移行で最も痛いのは local session history 欠落なので、`ai restore` が snapshot-only record を扱えない状態は未完扱いとし、rehydrate fallback を入れて塞いだ
