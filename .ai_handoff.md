# 引き継ぎメモ

## 目的
Addness の現行運用を前提に、`Lステップ / UTAGE / Mailchimp / short.io / Zapier` を使った施策実装を、要件が来たら高精度で進められる状態にすること。

## 確定事項
- 応答と報告は日本語
- ツール理解は `ツールそのものの役割` と `アドネス株式会社の販売導線における役割` を分けて扱う
- 100点の定義は `役割 / ゴール / 必要変数 / ワークフロー / NG / 正誤判断`
- コンテンツ定義は上位原則
  - `コミュニケーションの手段であり、ユーザーをこちらが意図した認識に変換するもの`
  - Addness 固有の定義ではない
- 販売導線の重要概念
  - `シームレスな顧客体験をコミュニケーションコンテンツ単位で描く`
- Mailchimp
  - `Journey = evergreen`
  - `Campaign = 1回きりの segment 配信`
  - 新規実装の main CTA は `short.io` を正にする
  - tag は英語で、基本は `media_funnel_event`
- Lステップ
  - 主戦場は `流入経路 / テンプレート(Flex) / シナリオ step`
  - 基本思想は `まずタグで状態を作る -> その状態に応じてテンプレートやシナリオを適用`
  - `友だち追加時設定` は基本的にあまり触らない
- UTAGE
  - メール本線ではなく `ページ / 会員サイト / 商品 / detail / action / 動画 / メディア` の基盤
  - `登録経路` は `どこから来たかを知りたい時に切る分析軸`
  - 価格、商品名、シナリオ、設定構成、会員サイト解放先が変わるなら別商品に寄せる
- short.io
  - 導線リンクの正本基盤
- Zapier
  - relay layer

## 判断とその理由（最重要）
- いま優先すべきは分析基盤ではなく、施策を即設定できる理解
- コンテンツは Mailchimp だけの話ではなく、販売導線全体の OS
- 各システムは `入力 -> 変換 -> 出力 -> 接続先` で理解しないと実務でズレる
- current 実装に direct LIFF が残っていても、今後の標準は `short.io`
- 動画学習は `見た` で終わらせず、手順 / 判断基準 / NG / live UI 差分に落としてから正本化する
