# 引き継ぎメモ

最終更新: 2026-03-13

## 目的
- `導線ツール` を厳しい 10 点基準まで引き上げる
- 10 点の定義は `役割 / ゴール / 必要変数 / ワークフロー / NG / 正誤判断` を持ち、exact な UI 手順で迷わず実装できる状態

## 今回ここまでで固めたこと

### Lステップ
- `テンプレート` の current 種別は次で固定
  - `標準メッセージ`
  - `カルーセルメッセージ(新方式)`
  - `フレックスメッセージ`
  - `テンプレートパック`
- `フレックスメッセージ` は、機能名だけでなく `画像 / テキスト / ボタン / アクション / デザイン` を作る順と見る順で整理した
- `テンプレートパック` は `message をまとめる箱` ではなく、`認識を変える step` と `行動させる step` を分ける container として読む前提を固定
- `テスト送信` は一覧行右端の `... -> テスト送信`
- `テスト送信` modal の current ラベルは
  - `テスト送信先選択`
  - `友だち名`
  - `キャンセル`
  - `テスト`
- テスト送信先は常に `甲原 海人`
- 重複送信は禁止
- `テンプレートパック` の representative 候補は
  - `YouTube切り抜き 無料相談会告知`
  - `動画編集　進捗報告　テンプレート`
  - `2024.10.29 アンケート`
- 単体 `フレックスメッセージ` の delete は UI ルートは分かっているが、`削除後に一覧 / API の両方で消えたこと` は次回 live で再確認する

### UTAGE
- `代表 page edit を触る順番` を固定
  1. `基本情報`
  2. ページ一覧の行名
  3. 主CTA
  4. `シナリオ`
  5. `アクション`
  6. 遷移先
  7. `デザイン`
  8. `カスタムCSS`
  9. `カスタムJS`
  10. `高速表示モード`
- LP / thanks / ユーザー登録で見る順番も分けた
- `商品 -> detail -> action -> 会員サイト解放` の exact 判断を強化
  - `商品本体`
  - `detail`
  - `アクション`
  - `バンドルコース`
  を別物として見る
- `商品本体だけ見て終わる` を NG に固定
