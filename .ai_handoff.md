# 引き継ぎメモ

## 目的
- `導線ツール` を厳しい 10 点基準まで引き上げる
- 10 点の定義は、`役割 / ゴール / 必要変数 / ワークフロー / NG / 正誤判断` を持ち、exact な UI 手順で迷わず実装できる状態

## 固定前提
- UI を説明する時は、抽象語より画面ラベルを優先する
- テスト送信先は常に `甲原 海人`
- テスト資産は残さない
- `コンテンツ` は上位概念であり、広告 / SNS / LP / メール / LINE / 会員サイトまで含む
- `カルーセルメッセージ(新方式)` と `フレックスメッセージ` は、Lステップ UI 上の実名称で扱う

## 現在地
- Lステップの `テンプレート` 領域は live で end-to-end をかなり確認できている
  - `標準メッセージ`
  - `カルーセルメッセージ(新方式)`
  - `フレックスメッセージ`
  - `テンプレートパック`
- `フレックスメッセージ` は
  - `テンプレート -> 新しいテンプレート -> フレックスメッセージ`
  - `edit メッセージを作成`
  - `画像 / タイトル / テキスト / ボタン / 動画`
  - `ボタン -> アクション設定`
  - 一覧右端 `... -> テスト送信`
  の current 導線を固定済み
- `テンプレートパック` は
  - 作成
  - step 追加
  - `甲原 海人` への `テスト送信`
  - step 削除
  - pack 削除
  まで確認済み
- `アクション管理` は live で current field と save route を固定済み
  - 一覧: `https://manager.linestep.net/line/action`
  - 上部: `新しいフォルダ / 新しいアクション / 並び替え`
  - 新規 modal の action type:
    - `テキスト送信`
    - `テンプレート送信`
    - `タグ操作`
    - `友だち情報操作`
    - `シナリオ操作`
    - `メニュー操作`
    - `リマインダ操作`
    - `対応マーク・表示操作`
    - `イベント予約操作`
    - `コンバージョン操作`
  - 詳細取得: `GET /api/actions/{id}`
  - save: `POST /api/action/register`
  - helper:
    - `python3 System/scripts/lstep_action_catalog.py list --limit 20`
    - `python3 System/scripts/lstep_action_catalog.py inspect --id 27783572`
  - ただし `新しいアクション -> 保存 -> 削除` の end-to-end はまだ未固定
- UTAGE は
  - `ファネル / ページ / 登録経路 / 商品 / detail / action / 会員サイト / 動画 / メディア`
  の役割を理解済み
  - `登録経路` は `どこから来たかを知りたい時に切る分析軸`
  - `商品 -> detail -> action -> bundle`
    の current 基本線は読めている
- Mailchimp は
  - `Journey = evergreen`
  - `Campaign = 単発セグメント配信`
  を固定済み
  - current regular campaign の recent 実測も取れている
  - 直近は `direct LINE` main CTA が多いが、今後こちらが新規で作る標準は `main CTA = short.io`
- short.io は
  - 役割
  - 判断基準
  - URL 管理シート
  - API
  までかなり固い
- Zapier は relay layer として representative を追加済み
  - current 主戦場は `Webhooks by Zapier -> Mailchimp Add/Update Subscriber`

## 現在の自己評価
- Lステップ: `8.9 / 10`
- UTAGE: `8.4 / 10`
- Mailchimp: `8.1 / 10`
- short.io: `9.4 / 10`
- Zapier: `7.9 / 10`
