# 引き継ぎメモ

## 目的
Addness 株式会社の current 運用を前提に、Lステップ / UTAGE / Mailchimp / short.io / Zapier を、要件が来たら速く、事故なく、既存品質以上で実装できる状態にすること。

## 確定事項
- `コンテンツ` は上位定義として扱う
  - `コミュニケーションの手段であり、ユーザーをこちらが意図した認識に変換するもの`
- ツール理解は 2 層で見る
  - ツールそのものの役割
  - Addness における役割
- Mailchimp
  - `Journey` は evergreen 用
  - `Campaign` は 1 回きりの segment 配信用
  - 今後こちらが新規で作る main CTA は short.io を正にする
- UTAGE
  - メール本線ではなく、ページ / 商品 / detail / action / 会員サイト / 動画 / メディアの基盤
  - `登録経路` は広告IDで必ず切る機能ではなく、どこから来たかを知りたい時に切る分析軸
- Lステップ
  - action 設定の主戦場は `流入経路 / テンプレート / フレックスメッセージ / シナリオ配信`
  - `友だち追加時設定` は基本的に主戦場ではない
  - 公式LINEのテスト送信先は常に `甲原 海人`
  - テスト送信は重複禁止
- short.io
  - 導線リンクの正本基盤
  - URL管理シートは `【アドネス全体】URL管理シート`
- Zapier
  - relay layer
  - 今後新規に作る Zap は原則 `甲原` フォルダ配下

## 10点の基準
- 最初の入口を誤らない
- exact な UI / API 手順で迷わない
- current / legacy の罠を避けられる
- 保存、送信、削除まで短時間で再現できる
- 例外発生時も見る場所が分かる

`最終的にできた` では 10 点にしない。`迷わず / 速く / 事故なく / 再現できた` 時だけ 10 点に置く。
