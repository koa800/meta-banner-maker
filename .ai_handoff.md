# 引き継ぎメモ

## 目的
Addness の現役導線を、Lステップ / UTAGE / Mailchimp / short.io を中心に、要件が渡されたら高精度で実装できる状態まで理解すること。

## 確定事項
- short.io は短縮URL作成ツールではなく、導線リンクの正本基盤として扱う。
- Mailchimp はメール本線。
  - Journey = evergreen
  - Campaign = 単発の segment 配信
- 今後こちらが新規に作る Mailchimp の main CTA は、Journey / Campaign を問わず short.io を正にする。
- UTAGE はメール本線ではなく、ページ / 会員サイト / 商品 / detail / action / 動画 / メディアの基盤。
- Lステップの action 設定の主戦場は `流入経路 / テンプレート(Flex) / シナリオ step`。
- Lステップでは、まずタグで状態を作ってから、テンプレートやシナリオを適用する思想が current。
- スキルプラス表記で統一する。
- 導線データ基盤は将来課題。今は施策をすぐ実装できる運用理解を優先する。

## 判断とその理由（最重要）
- 先に 1 ツールを 100 点に寄せるべき、という方針に合わせて short.io を先に仕上げた。理由は役割が閉じていて、判断基準を型化しやすいから。
- 今は動線データ基盤ではなく、施策を即実装できる運用理解を優先する。理由は、甲原さんの目的が「こういう施策をしたい」と言った時に即実装できることだから。
- Lステップは action 管理画面だけでなく、`流入経路 / テンプレート / シナリオ step` のアクション設定を理解しないと実務で足りないため、そこを主戦場として整理した。
- Mailchimp は current に direct LIFF の historical 実装もあるが、今後の Addness 標準は short.io 経由に寄せる。理由は、失効対策、差し替え容易性、クリック分析、運用統一に強いから。
- UTAGE と Mailchimp は、見た目の URL と実際のリンク先がズレる事故があるので、設置後にクリック確認するルールを共通化した。
