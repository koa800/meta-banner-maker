# 引き継ぎメモ

## 目的
- `導線ツール` を厳しい 10 点基準まで引き上げる
- 10 点の定義は `役割 / ゴール / 必要変数 / ワークフロー / NG / 正誤判断` を持ち、exact な UI 手順で迷わず実装できる状態

## 確定事項
- UI を説明する時は、抽象語より画面ラベルを優先する
- テスト送信先は常に `甲原 海人`
- テスト資産は残さない
- `コンテンツ` は上位概念であり、広告 / SNS / LP / メール / LINE / 会員サイトまで含む

## 現在地
- Lステップの template 領域で、`標準メッセージ / カルーセルメッセージ(新方式) / フレックスメッセージ / テンプレートパック` の live end-to-end が通った
- existing representative template の分解も進めた
  - `SNS活用 1on1無料相談会`
  - `▼無料相談はこちら【YouTube切り抜き 無料相談会】`
  - `YouTube切り抜き 無料相談会告知`
  - `3月 スキルプラス合宿 集客`
  を current の良い例として整理し、`何の認識を変えて / 何を行動させるか` を正本化した
- `標準メッセージ`
  - item id `249318424`
  - `POST /api/message/test`
  - `DELETE /api/templates/249318424`
- `カルーセルメッセージ(新方式)`
  - item id `249321068`
  - save route `POST /line/template/edit_v2`
  - `タイトル` は空でも保存成功
  - `アクション設定` は未設定のままでも保存成功
  - `テスト送信` は UI では通ったが、API route は headless network capture で再取得できていない
  - `DELETE /api/templates/249321068`
- `フレックスメッセージ`
  - `+ ブロックを追加`
  - `画像 / タイトル / テキスト / ボタン / 動画`
  - `ボタン -> アクション設定`
    - `アクション実行`
    - `URLを開く`

    - `電話をかける`
    - `LINEアカウントを友だち追加する`
    - `メールを送る`
    - `その他`
  - 作成 -> `甲原 海人` への `テスト送信` -> 削除 まで通った
- `テンプレートパック`
  - 新規作成入口は `テンプレート -> 新しいパック`

  - `本文` step を 1 件追加
  - `甲原 海人` への `テスト送信`
  - step 削除
  - pack 削除
  まで通った
- `GET /api/templates` から current の種類判定を取れる
  - `form_type=8 / editor_version=0` = `標準メッセージ`
  - `form_type=30 / editor_version=1` = `カルーセルメッセージ(新方式)`
  - `form_type=60 / editor_version=2` = `フレックスメッセージ`
  - `form_type=40 / editor_version=10` = `テンプレートパック`
