# 引き継ぎメモ

## 目的
Addness 株式会社の現行運用を前提に、Lステップ / UTAGE / Mailchimp / short.io / Zapier を高精度で実装・改修できる状態にすること。

## 確定事項
- コンテンツは Addness 固有ではなく、上位定義として扱う
  - `コミュニケーションの手段であり、ユーザーをこちらが意図した認識に変換するもの`
- ツール理解は 2 層で見る
  - ツールそのものの役割
  - Addness における役割
- Mailchimp
  - Journey は evergreen 用
  - Campaign は 1 回きりの segment 配信用
  - 今後こちらが新規で作る main CTA は short.io を正とする
- UTAGE
  - メール本線ではなく、ページ / 商品 / detail / action / 会員サイト / 動画 / メディアの基盤
  - `登録経路` は広告ID単位で必ず切る機能ではなく、どこから来たかを知りたい時に切る分析軸
- Lステップ
  - action 設定の主戦場は `流入経路 / テンプレート(Flex) / シナリオ step`
  - `友だち追加時設定` は主戦場ではない
- short.io
  - 導線リンクの正本基盤
  - URL 管理シートは `【アドネス全体】URL管理シート`
- Zapier
  - relay layer
  - 今後新規に作る Zap は原則 `甲原` フォルダ配下

## 判断とその理由（最重要）
- まず short.io を 100 点に寄せた
  - 1 つの領域で 100 点の構造を掴むと、他ツールにも同じ型を転用しやすいから
- 動画学習は `見る` ではなく
  - 章分解
  - 手順抽出
  - 判断基準抽出
  - NG 抽出
  - live UI 再現
  - 知識化
  の順で処理するように修正済み
- ツールの役割は短い 1 語で切らない
  - 入力
  - 変換
  - 出力
  - 接続先
  - 制約
  - 失敗条件
  まで持つ
- 実装 readiness は高いが、今後の精度をさらに上げるには
  - relay layer としての Zapier
  - コンテンツ意図分析
  の厚みが次のレバー
