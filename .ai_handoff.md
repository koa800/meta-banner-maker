## 判断とその理由（最重要）
- 100点は「機能を全部触れること」ではない
  - 何のためにそのツールを使うのか
  - 何が良い / 悪い / NG か
  を判断できることが先
- short.io を先に深く詰めたのは、役割が閉じていて、100点の感覚を他ツールへ転用しやすいから
- Lステップは friend-add default を主戦場と捉えず、実際に current で多用されている `流入経路 / テンプレート / シナリオ` の action に軸を移した
- UTAGE の登録経路は、現在の運用例として広告ID単位が多いが、本質は「流入元を分けて分析したい時に切る」こと
- Mailchimp の content 理解は、件名やリンクを拾うだけでは弱いので、メールの役割と制作意図まで読む前提へ更新した
