# 引き継ぎメモ
## 目的
- `Master` を、甲原さんのOS・信念・テーマ・現在の事業実装を統合した判断脳として整えること
- 最上位ゴール `世界中の「ありがとう」の数を増やす` から、実務判断まで下ろせる構造にすること
- 併走中の Meta広告CR分析は、その上位定義と矛盾しない形で継続できるよう参照先を残すこと

## 確定事項
- 信念: `俯瞰的選択と関係の積み上げが愛という世界を作る`
- 最上位ゴール: `世界中の「ありがとう」の数を増やす`
- 仕事: `人の役に立つこと`
- ありがとう: `人の課題が解決された結果として生まれる反応`
- 市場: `特定の欲求がまだ満たされていない集団`
- 欲求: `言語化されていない目的。もっと良くしたい方向性はあるが、まだピン留めされていないもの`
- 目的: `欲求を言語化したもの`
- 理想: `その人が主観的に「こう在れたら良い」と感じる、目的が満たされた状態像`
- 現実: `理想に対して、その人が今実際に置かれている状態`
- 問題: `理想と現実のギャップ`
- 課題: `理想と現実のギャップの原因`
- 状態: `ある時点において、その人が理想に近づけるか、遠ざかるかを決めている条件の総体`
- 需要: `サービスに対する欲求`
- 供給: `需要を満たす商品・サービス`
- 価値: `相手が主観的に「自分にとって意味がある」「理想に近づく」と認識するもの`
- オファー: `ある場面で、相手に受け取ってほしい取引条件の提示`
- 取引成立: `オファーが受け取られ、交換行為が成立した瞬間`
- 事業: `ある市場に対して供給を届ける活動`
- マーケティング: `需要に供給を届ける全ての活動`
- 教育: `できないことをできるようにすること`
- できる: `ある課題に対して、再現性をもって解決行動を取れること`
- 成長: `できることが増えること、またはできることの再現性・範囲・深さが広がること`
- 自立: `自分で理想と現実を捉え、問題と課題を見つけ、必要な供給を受け取り、行動し続けられる状態`
- 信頼: `ある対象に対して、自分が期待していることが満たされると見込んでいる状態`
- 不安: `期待していることが満たされない可能性を感じている状態`
- 安心: `期待していることが満たされない可能性が、十分に小さいと感じている状態`
- 納得: `そのオファーを受け取ることが、自分にとって意味があり、妥当で、今受け取ってよいと内的に整合している状態`
- オファーを受け取れる状態: `見ない / 読まない壁、納得しない壁、行動しない壁を超えている状態`
- スキルプラスは最上位ゴールではなく、`仕事ができない人を、課題解決できる側へ近づける` 現在の主要実装の一つ
- Meta広告分析の大きい変数は `CR / 運用 / LP / 時期`
- Meta広告分析では `顧客の質` のような抽象語を使わず、`属性 / LTV / 返金率 / 何に対しての何率か` に分解する

## 判断とその理由（最重要）
- `Master` を `スキルプラス専用` と捉えると脳が狭くなるため、最上位を `ありがとう` と `課題解決` に置き直した
- `用語定義.md` は Sheets 同期で上書きされるため、まずは Markdown/JSON の冒頭に `手動確定概念` を足してズレを防いだ
- `CVR` のような抽象語は誤読を生むため、Meta分析では必ず `何に対しての何率か` に具体化する方針にした
- `顧客の質` は評価者側の雑な圧縮なので使わず、`属性 / LTV / 返金率 / 率の母数` に分解して読む前提で固定した
- 壁打ちの質を上げるには質問だけで返すより仮説を先に置く方が良いので、その原則を判断軸に反映した
