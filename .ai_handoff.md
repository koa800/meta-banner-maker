# 引き継ぎメモ

## 目的
アドネス株式会社の current 運用を前提に、Lステップ、UTAGE、Mailchimp、short.io、Zapier を高精度で実装できる状態まで理解を詰めること。

## 確定事項
- `コンテンツ` は Addness 固有ではない上位定義
  - `コミュニケーションの手段であり、ユーザーをこちらが意図した認識に変換するもの`
- システムは広義で扱う
  - 人
  - ツール
  - AI エージェント
  をすべて `役割を持つシステム` として定義する
- 役割は必ず 2 層で見る
  - ツールそのものの役割
  - アドネス株式会社の販売導線における役割
- Mailchimp
  - `Journey = evergreen`
  - `Campaign = 1回きりの segment 配信`
  - 新規実装の `main CTA` は `short.io` を正とする
- UTAGE
  - `登録経路` は広告IDだから切るのではなく、`どこから来たかを知りたい時に切る分析軸`
- Lステップ
  - action 設定の主戦場は
    - 流入経路
    - テンプレート / Flex
    - シナリオ step
  - `友だち追加時設定` は基本的に主戦場ではない
- UTAGE の商品分岐
  - 価格
  - 商品名
  - 流すシナリオ
  - 設定構成
  - 会員サイト解放先
  のどれかが変わるなら別商品に寄せる
- Zapier
  - 今後 Addness の業務で新しく作る資産は、原則 `甲原` フォルダ配下に置く
  - 既存 current Zap は無理に移動しない

## 判断とその理由（最重要）
- 今は `動線データ基盤` を先に作る段階ではない
  - 先に `施策を受けたらすぐ実装できる基礎理解` を 10 点に近づける方が効率が高い
- `コンテンツ` はツール別のローカル定義ではなく、広告、SNS、LP、メール、LINE、会員サイト、セミナーまで貫く上位原則として扱う
- `Mailchimp = 認識変容の本線`
  - ただし 1 語で雑に切らず、入力 / 変換 / 出力 / 接続先で定義する
- `Lステップ = 状態管理と次行動の実行制御`
  - 長文教育の場ではない
- `UTAGE = 登録 / 購入 / 視聴 / 会員化を成立させる実行基盤`
- `short.io = 導線リンクの正本基盤`
- `Zapier = relay layer`
  - 顧客体験そのものではなく、次のコンテンツを出せる状態へ裏で変換する
