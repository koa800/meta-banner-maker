# 前提更新ルール

最終更新: 2026-03-09

ここは、`前提` レイヤーをいつ自動更新してよいか、いつ確認が必要かを固定する場所です。認識ズレを防ぐため、意味変更の境界をここで明確にします。

## 自動で進めてよいこと

- Google Sheets の用語追加をローカル正本へ同期する
- 既存正本へのリンク整理や入口整理をする
- 意味を変えない要約や参照先の明記を足す
- output レビューに、既存基準に沿ったメタ情報を足す

## 甲原さんに確認してからやること

- 新しい用語定義をこちらで新設する
- 既存の用語定義の意味を変える
- 目的、判断軸、優先順位の中身を変える
- 仮説や単発事例を `rules` や `Skills` に昇格させる
- 既存の脳資産を上書きして、ニュアンスを変える

## 迷ったときの原則

- 迷ったら `1段階下に留める`
- 迷ったら `既存正本をそのまま参照する`
- 迷ったら `新規定義を作らず、未整理として残す`

## 用語定義の扱い

- `定義保留` は「定義したいが、まだ本人の言葉として固まっていない」状態として扱う
- こちらの都合で空欄を埋めず、本人の定義が出るまでは `保留` のまま残す
- ただし、甲原さんが会話で頻繁に使い、判断や表現のズレを生みやすい言葉は `定義候補` として先に拾う
- `定義候補` は `用語 / 仮置きメモ / なぜ合わせたいか` まで残し、意味の確定は確認後に行う
- `前提` と `rules` に跨る重要語は、曖昧なまま `rules` や `Skills` に昇格させない
