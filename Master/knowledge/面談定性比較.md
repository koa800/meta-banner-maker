# 面談定性比較

最終更新: 2026-03-12

## 目的

CDP に入った面談定性を `高LTV / 低LTV / 非成約` で比較し、顧客理解を `knowledge` と `rules` に戻す。

## 正本

- 機械可読な正本: `System/data/interview_insights_analysis.json`
- 再生成コマンド: `python3 System/interview_insights_sync.py analyze`

## `高LTV` の扱い方

### 1. 比較分析での定義

- このファイルでいう `高LTV` は、`interview_insights_sync.py analyze` の percentile 区切りで `232256` 以上を指す
- `低LTV` は `2980` 以下、`非成約` は `LTV <= 0`
- これは **面談定性を比較するための分析用しきい値** であり、事業全体の唯一の正解ラインではない

### 2. 広告運用での作業仮説

- 広告から販売までの実務では、`高LTVの人` と `高LTVになりそうな人` を分けて扱う
- `高LTVの人` は結果定義であり、全事業共通では `買った価値の投下先が実際にある人`、スキルプラスでは `学んだ内容の差し込み先が実際にある人` を指す
- `高LTVになりそうな人` は集客定義であり、全事業共通では `買った価値の投下先の素地があり、広告から販売までの教育でその投下先が明確になる人`、スキルプラスでは `学んだ内容の差し込み先の素地があり、広告から販売までの教育で何に使うかが具体化する人` を指す
- したがって広告企画では、`高LTVの人を直接当てる` ではなく `高LTVになりそうな人` を増やす意味で使う
- 教育の役割は、差し込み先をゼロから作ることではなく、既にある仕事 / 事業 / 顧客 / 発信 / 運用との接続先を顕在化し、具体化すること
- ただし、**現時点では `高LTVになりそうな人` の人物像もまだ解像度向上の途中** であり、今あるのは強いシグナルを束ねた作業仮説である

#### 確定

- `現在の悩み` の頻出語だけでは、高LTV / 低LTV / 非成約を切り分けにくい
- 面談定性比較では、差分は `現在の悩み` より `理想の未来` と `過去の解決策` に出やすい
- DProリサーチでは、LTV差の最大ドライバーは `悩み` より `職業`

#### 強い推定

- `事業者・経営者` は会社員より平均LTVが高い
  - 既存分析では `¥609,269 vs ¥224,992`
- `AIファネル` は高LTV顧客の主要供給源
  - 既存分析では `¥500,000以上` の高LTV顧客の `55%` が AI ファネル経由
- したがって広告企画では、`事業者・経営者` `AIを事業に導入したい層` `過去にすでに何かを試している層` を、`投下先 / 差し込み先の素地がある` 強いシグナルとして扱う

#### 未解明

- `高LTVの人` を、`属性 / 欲求 / 理想の未来 / 過去の解決策 / 流入経路` をまたいで一意に説明する定義はまだない
- `現在の悩み` では切れず、`理想の未来` と `過去の解決策` の件数もまだ薄い
- 面談定性比較の正本では、`職業 / 年齢 / 流入経路` と `理想の未来 / 過去の解決策` を同じ粒度で横断比較し切れていない
- したがって今の広告運用では、`高LTVそのものを当てにいく` のではなく `高LTVになりそうな人` を狙って検証する段階と捉える

### 3. 用語の使い分け

- `高LTV` を単なる `高単価購入者` の意味で雑に使わない
- 面談定性比較で `高LTV` と言うときは、まず `232256` 以上の分析セグメントを指す
- 事業運用で `高LTVの人` と言うときは、購入後に価値の投下先 / 差し込み先が実際にある人を指す
- 広告企画で主語にすべきなのは `高LTVになりそうな人` であり、現時点では `事業者・経営者比率を上げる` `AIを事業導入したい層を増やす` `過去に何かを試している層を寄せる` をその強いシグナルとして扱う
- `45歳でも / 月収50万 / 秘密教えます / 楽に` のような訴求は、差し込み先が曖昧な層を広く集めやすい前提で扱う

## 現時点の定義案

### 全事業共通で `LTVが高い人`

#### 分析上の定義

- まず分析では `LTV >= 232256` を `高LTV` と置く
- これは面談定性比較のしきい値であり、人物像の完成定義ではない

#### 事業運用上の定義

- 全事業共通で `LTVが高い人` とは、`買った価値の投下先が、実際にある人`

#### この定義に含める要素

- `何となく良さそう` ではなく、買った価値を差し込む現実がある
- 買った価値を `1回使う` のではなく、既存の仕事や事業の中で `回収し続けられる`
- そのため、購入後の活用・継続・追加投資が起きやすい

#### 強いシグナル

- 既存分析では、`現在の悩み` より `職業` の方がLTV差に効いている
- 既存分析では、`経営者・役員` の平均LTVは `会社員` より高い
- これは `経営者・役員だから高LTV` という意味ではなく、`価値の投下先を既に持っている確率が高い` ための強いシグナルとして読む

#### 定義に使わないもの

- `現在の悩み` の頻出語だけ
- `CPAが安かった` という入口の数値だけ
- `年齢が高い / 若い` のような単一属性だけ
- `課題が具体` のような、購入者なら多くが当てはまる条件だけ

### 全事業共通で `LTVが高くなりそうな人`

#### 集客実務での定義

- 全事業共通で `LTVが高くなりそうな人` とは、`買った価値の投下先の素地があり、広告から販売までの教育で、その投下先が明確になる人`

#### 実務で見るシグナル

- 既に `仕事 / 事業 / 顧客 / 集客導線 / 発信 / 営業 / 運用` のどれかがある
- 抽象的な願望だけでなく、`今ある何かを変えたい` 文脈がある
- 過去にすでに何かを試しており、改善先がある
- 教育によって `何に使うか` が具体化しやすい

#### 集客で避けたいズレ

- 可能性や雰囲気だけを買う層
- `いつか使えたらいい` で止まる層
- 投下先がなく、教育でゼロから作らないと成立しない層

### スキルプラス事業で `LTVが高い人`

#### 分析上の定義

- 2026-03-12 時点の CDP では、`スキルプラス入会者かつ LTV > 0` が `4037` 行
- その `75 percentile` は `798000`
- したがって、スキルプラス事業の分析用には、現時点で `LTV >= 798000` を `高LTV` の目安として置ける

#### 事業運用上の定義

- スキルプラス事業で `LTVが高い人` とは、`学んだ内容の差し込み先が、実際にある人`

#### この定義に含める要素

- `学びたい` だけでなく、学んだものを差し込む現実がある
- 差し込み先は、`自分の業務 / 自分の事業 / 顧客案件 / 既存の集客導線 / 既存の営業 / 今ある発信や商品` など
- つまり、`知識を持ちたい人` より `今ある何かを変えるために学ぶ人` が高LTV寄り
- そのため、入会後も使いながら回収しやすく、継続や上位プラン接続が起きやすい

#### 現時点で確認できること

- 高LTV群の平均LTVは `953654`
- 低LTV群の平均LTVは `88727`
- 高LTV群の中途解約率は `8.8%`
- 低LTV群の中途解約率は `17.0%`
- 高LTV群は実質 `オールインワン / プライム` に集中している

#### まだ未検証の本質

- CDP の `連続アクション日数 / 累計アクション日数 / ゴール達成数 / イベント参加回数` は、2026-03-12 時点でスキルプラス行の大半が未入力
- したがって、本来は `差し込み先がある -> 行動が続く -> 成果が出る -> 継続する` まで含めて定義すべきだが、今は `プラン / 解約 / 属性` までしか実証できていない
- 構造上は、スキルプラスの高LTVは `差し込み先の有無` が起点になり、その結果として `ゴール達成数 / ゴール達成率 / 連続アクション日数` が増える形で説明されるべき
- ここは今後、CSデータ同期が進んだら定義を更新する

### スキルプラス事業で `LTVが高くなりそうな人`

#### 集客実務での定義

- スキルプラス事業で `LTVが高くなりそうな人` とは、`学んだ内容の差し込み先の素地があり、広告から販売までの教育で、何に使うかが具体化する人`

#### 実務で見るシグナル

- 既に `自分の業務 / 自分の事業 / 顧客案件 / 既存の集客導線 / 既存の営業 / 今ある発信や商品` のどれかがある
- `AIを学びたい` ではなく、`今ある何かを変えたい` 文脈がある
- 教育で差し込み先を言語化すると、購入理由と活用先がつながる
- 入会後の行動や成果が、その差し込み先に接続しやすい

#### 集客で避けたいズレ

- `自由になりたい / 稼ぎたい` のみで差し込み先が見えない層
- 特典や無料プレゼントには強く反応するが、入会後の活用先が曖昧な層
- 差し込み先をゼロから作らないと回収が始まらない層
