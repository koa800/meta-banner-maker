# Success Learning AI（SLS）構造・操作ガイド

最終更新: 2026-03-04

## 概要

Success Learning AI（サクセスラーニング）は、アドネスが開発・運営する受講生向けの学習プラットフォーム。
「AIラーニングシステム」として、受講者に最適な難易度のアクションを提示し、ゲーム感覚で学習を進められる仕組み。

## 核心コンセプト: アクションマップ（AM）

### アクションマップとは

SLSの学習単位。受講者が「ゴール（達成すべき状態）」に向かって、段階的にアクション（STEP）を進めていく仕組み。

```
コース（例: 広告運用代行コース）
├── アクションマップ1「広告CRの企画をつくろう！」
│   ├── STEP 1: ...
│   ├── STEP 2: ...
│   └── STEP 3: ...（完了→次のAMへ）
├── アクションマップ2「Meta広告のアカウントを開設しよう！」
│   ├── STEP 1: ...
│   └── ...
└── アクションマップ7「PDCAを回して成果を出そう！」
    └── ...
```

### AM命名ルール
- 動詞: 「つくろう！」で統一
- 語尾: 「しよう！」「書こう！」「磨こう！」も可
- 生成AI系: 「生成AIで〜」で統一
- 文字数: 18文字以内目安
- 具体性: タイトルだけで何をするか分かること

### AM品質評価基準（フィードバック用）
1. **MECE**: ゴール達成に必要なステップが漏れなく・重複なく揃っているか
2. **分かりやすさ**: ターゲットが知らない言葉が説明なしで使われていないか
3. **行動ベース**: 各ステップが具体的な行動で書かれているか
4. **制作物完成**: 全ステップ完了後に制作物が手元にある状態になるか

## 受講生の学習フロー

### 標準的な進行

```
1. LINE登録（Lステップ経由）
2. オンボーディング（4つの簡単アンケート回答）
3. SLS登録（student.success-learning.ai/register）
4. セットアップコース受講
   └── アクションマップのSTEPを1つずつ完了
5. リッチメニュー開放（Lステップ連携）
6. コワーキングスペース利用開放
7. 全機能解放アンケート回答
8. 全機能解放 → 好きなコースを自由に受講
```

### 習得段階の定義（みかみ基準）

| 段階 | 説明 |
|------|------|
| 記憶 | 情報を覚えただけ |
| 理解 | 学んだことの意味が分かる |
| 実践 | 実際に使える |
| 応用 | 自分の文脈に応用できる |

**「営業をスタートしても良い」の最低ライン**:
広告CR制作→Meta広告アカウント設定→出稿までの流れがイメージできる状態 = サクセスラーニングのSTEP2まで完了

## 質問回答システム（SLS側）

### フロー
```
受講生がSLS上で質問
  → Googleスプレッドシート「回答用：SLS」に自動追加
  → ローカルエージェントが60秒ごとに検出
  → AI回答案を生成（Pinecone類似検索 + Skills知識）
  → LINE秘書グループに通知
  → 講師が承認（1=そのまま / 2=修正 / 3=動画回答）
  → 承認後スプレッドシートに書き込み
  → ナレッジとしてPineconeに蓄積（自動学習ループ）
```

### 技術的な仕組み

| コンポーネント | 場所 | 役割 |
|---|---|---|
| qa_monitor.py | System/line_bot_local/ | スプレッドシート監視（60秒ポーリング） |
| local_agent.py | System/line_bot_local/ | 新規質問検出→サーバー送信、承認済み回答→シート書き込み |
| qa_handler.py | System/line_bot/ (Render) | AI回答生成（Pinecone類似検索 + Skills知識 + GPT-4o-mini） |
| app.py | System/line_bot/ (Render) | LINE通知・承認コマンド処理 |

### 承認コマンド

| コマンド | 動作 |
|----------|------|
| `1` | AI回答をそのまま承認 |
| `2 修正内容` | 修正内容で承認 |
| `3` | 動画回答フラグ（講師が手動対応） |

リプライで特定質問を指定可能（リプライなしは最新の未処理質問）。

### ナレッジ蓄積ループ
承認するたびにPineconeにベクトル化して蓄積。次回の類似質問検索で自動参照され、AI回答精度が向上していく。

### SLSシートの特徴（L-stepシートとの違い）
- **「目的」「やったこと」列がない** → 質問内容のみ
- col_question: H列（質問内容を教えてください！）
- col_status: J列（回答済み TRUE/FALSE）
- col_answer: L列（回答）
- col_lstep_link: N列（member=xxxからUID取得）

### L-stepシートとの比較

| 項目 | L-stepシート | SLSシート |
|------|-------------|-----------|
| 質問元 | Lステップの回答フォーム | SLS上のQ&A機能 |
| 目的・やったこと | あり（F列・G列） | なし |
| 質問列 | H列 | H列 |
| 回答列 | Q列 | L列 |
| 状態列 | O列 | J列 |
| 対象シート | サポートLINE/スタンダード/エリート/プライム（4枚） | 回答用：SLS（1枚） |

## SLS × Lステップ連携

### オンボーディングでの連携ポイント

| タイミング | SLS側 | Lステップ側 |
|-----------|-------|-----------|
| SLS登録完了 | ユーザー作成 | 流入経路①トリガー → テンプレート送信 |
| セットアップ完了 | STEP完了 | 流入経路②トリガー → リッチメニュー変更 |
| コワーキング解放 | 特定AM完了 | 流入経路③トリガー → テンプレート送信 |
| ゴール設定完了 | 目標回答 | 流入経路④トリガー → テンプレート送信 |
| 全STEP完了 | コース修了 | 流入経路⑤トリガー → 全機能解放 |

**連携方法**: SLSのコース内にLステップの流入経路URLを埋め込み、STEPアクション完了時にクリック → Lステップ側でトリガー発火

## CS（カスタマーサポート）の本質

SLSに関するCSの核心は**テクニカルサポートではない**。広告・マーケティングのアクションに対する**コンサル・添削が中心**。

### よくある質問カテゴリ（過去の質問回答より）
- **マインド**: 習得の定義、作業者と唯一無二の人材の違い、課題解決の考え方
- **マーケティング**: ポジショニングマップ、単価の決め方、リピート試算
- **広告**: NGワード・審査突破、配信面、クリエイティブの予算・日数、ROAS
- **営業**: 営業スタートの最低ライン（STEP2まで完了）
- **制作**: 制作スピード向上、AI画像の信頼性
- **SEO**: 悪評記事対策、サジェスト対策

### みかみの回答スタイル
- 抽象的な概念を具体例で噛み砕いて説明
- 「〜だと思います」「〜ではないでしょうか」の柔らかい語尾
- 質問の背景にある本質的な課題を指摘
- 即答できない質問には「こういう考え方があります」と思考の枠組みを提供
