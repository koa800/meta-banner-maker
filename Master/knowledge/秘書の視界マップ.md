# 秘書の視界マップ

最終更新: 2026-03-17

## 情報ラベル

- 所有元: internal
- 開示レベル: role-limited
- 承認必須: conditional
- 共有先: 僕 / 上司 / 並列

## 目的

秘書が「甲原さんと同じ目線で判断できる」ために、何が見えていて何が見えていないかを可視化する。
見えないものは分析できない。分析できないものの根本原因は特定できない。

## ブラインドスポット（見えていないもの）

### A. 広告パフォーマンスの詳細（最重要）

| 見えていないもの | なぜ重要か | データの持ち主 |
|----------------|----------|--------------|
| **CR単位のCPA/CTR/CPM/CVR** | 根本原因分析の最小単位。「なぜCPAが上がったか」はCR単位でしか分からない | 広告運用担当（新井さん等） |
| **配信面の変化** | 予算変更・競合増加で配信面がズレるとCPA高騰の原因になる | 広告運用担当 |
| **CR別の予算消化推移** | どのCRに予算が集中しているか、学習状態はどうか | 広告運用担当 |
| **広告セット/キャンペーン構造** | 運用の全体像。何をテスト中で何が本番か | 広告運用担当 |

**インパクト**: これが見えないと「CPA上がってます」としか言えない。見えれば「CRxのCPMが高騰。配信面ズレの可能性。予算引き上げ3日目」と言える。

### B. 導線の各ステップ転換率

| ステップ | 見えていないもの | データの持ち主 |
|---------|----------------|--------------|
| 広告 → LP | CTR（CR単位） | 広告運用担当 |
| LP → LINE登録 | LP転換率 | Lステップ担当 |
| LINE登録 → セミナー予約 | 予約率 | Lステップ担当 |
| セミナー → 購入 | 成約率 | セールス / UTAGE |
| 購入 → 継続 | 継続率・解約率 | CS / UTAGE |

**インパクト**: 導線のどこがボトルネックか一発でわかる。今は売上とCPAしか見えないので、どこが詰まっているか推測するしかない。

### C. 受講生の状態

| 見えていないもの | なぜ重要か | データの持ち主 |

## 視界を広げる方法

**原則: 技術的なAPI接続より先に、人に聞く。**

### 情報取得フロー

```
1. Addnessを確認 → 今どの領域が動いているか把握
2. 見えていないデータがある → 担当者に依頼してデータをもらう
3. 分からない知見がある → 甲原さんに直接聞く
```

### 依頼方法

| 相手 | 方法 |
|------|------|
| 甲原さん | LINE直接 |
| チームメンバー | Addnessタスク or LINEグループ（甲原さん人格で） |

### 依頼の目安

- 異常検知でアラートが出た → 担当者にデータ詳細を依頼
- Addnessで新しいゴールが動き出した → その領域のデータを確認
- 定期レポートで見えない部分がある → 週次で担当者にデータ共有を依頼
