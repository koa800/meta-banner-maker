# 経理月次運用

最終更新: 2026-03-10

## 目的

確定申告直前に詰まらないための、月次の経理ルーチン。
`毎月ここまでやれば、来年は一気に楽になる` という最低ラインをまとめる。

## 入口

- `freee`
- `/Users/koa800/Desktop/確定申告`
- `Google Drive > 個人に関わるもの > 【個人】経理関係`

証憑はまず `Desktop/確定申告` に置き、長期保管は Drive に寄せる。  
AI は `freee -> Desktop -> Drive` の順に見る。
処理済みの証憑は Drive の `年/確定申告整理/用途` へ移し、`Desktop/確定申告` は次回投入用に空ける。

## 依頼の仕方

甲原さんは次のどれか一言で足りる。

- `今月の経理を整えて`
- `確定申告フォルダを見て振り分けて`
- `freee の未処理を潰して`
- `Apple と Amazon を切って`

用途が曖昧なときだけ補足する。

## 月1回やること

### 1. freee の未処理を潰す

- 口座
- Amex
- Apple
- PayPay
- PASMO

未処理を翌月に持ち越さない。

### 2. チャージ系を切り分ける

- `PayPayチャージ` はそのまま経費にしない
- `PASMOチャージ` もそのまま経費にしない
- 実支払い・実利用を月内で切り分ける

### 3. Apple を分ける

- `iCloud / Apple Music` は私費
- `SNS・事業アプリ` は事業用
- 用途不明はその月のうちに判断

### 4. Amazon を分ける

- 事業用: その場で領収書保存
- 私用: そのまま私費
- 混在注文: 商品単位で切る

### 5. 領収書を整理する

- 紙は写真化
- PDF はそのまま保存
- 月別に寄せる
- 事業用か私用か、名前で見分けられるようにする
- 処理済みは Drive の年別フォルダへ抜く
- 事業用の証憑は freee のファイルボックスに入れて、該当取引へ添付する

### 6. 月末時点の残件を 0 か保留一覧にする

- `未処理`
- `保留`
- `用途確認待ち`

のどれかに必ず落とす。

`何となく残す` を禁止する。

## 週1回やること

- `freee の visible 未処理` を確認する
- `Desktop/確定申告` に新しい証憑が入っていないか見る
- `Apple / Amazon / PayPay / PASMO` の混在明細だけ先に切る
- `今月の保留一覧` を 5件未満に保つ

## 月次ルール

### 交際費

- `誰と / 何の目的で / いくら` を残す
- 高額なプレゼントは、その月のうちにメモを残す

### 旅費交通費

- GO / PASMO / 飛行機 / 新幹線を混ぜない
- 旅行と出張が混じるなら、その月に切る

### 衣装・美容

- 撮影専用衣装は事業用
- 通常美容は私費
- 眉毛など当日調整系は、その場で用途メモを残す

### 立替金

- 会社や他人の費用を立て替えたら、その月で `立替金` として切る
- 精算入金を売上に混ぜない

## 3か月に1回やること

- 住居関連の按分がズレていないか確認
- 光熱費の領収書漏れ確認
- 証憑が弱い高額支出の補強
- freee の消費税設定・固定資産台帳の確認
- `ソフトウェア` 勘定に SaaS や小口端末が紛れ込んでいないか確認
- freee の自動登録ルールが現状とズレていないか確認
- 消費税方式の前提が変わっていないか確認
  - 個人事業者で `2027年` から簡易課税を使うなら、通常は `2026-12-31` までに届出が必要
  - 現状の 2025 実数ベースでは `一般課税` が優位

## 月次の完了条件

- `freee の visible 未処理` が小さい
- `高額 / 旅行 / プレゼント` だけが人間判断待ち
- 事業用証憑は `Desktop` か `Drive` のどちらかに必ずある
- `保留一覧` に理由が書かれている
- `Apple / Amazon / PayPay / PASMO` の混在が放置されていない

## 年末にやること

- `10万円超` の一覧を作る
- 年末注文・翌年受領を切り分ける
- 国民年金・国保・医療費の証憑をまとめる
- 国民年金・国保が帳簿上 `事業主貸` になっているか確認する
- 源泉徴収額の一覧を確定する

## やらないこと

- チャージだけで経費化
- 私用品を後から曖昧に事業へ寄せる
- 高額プレゼントをメモなしで残す
- 年またぎ資産を注文日だけで前年計上する

## この運用のゴール

- freee の visible 未処理を毎月小さく保つ
- 12月末に `高額論点` だけ残す
- 確定申告期は `集計` と `最終判断` だけにする
- AI に `経理を整えて` と言えば、そのまま着手できる状態を維持する
