# 経理運用ルール

最終更新: 2026-03-10

## 目的

甲原さんの個人事業の経理を、`説明可能 / 再利用可能 / 次回は迷わない` 形で運用するための正本。

このファイルは `汎用スキル` ではなく、まず `甲原さん専用の経理ルールブック` として使う。
来年以降も繰り返し使える判断だけをここに残し、単年固有の論点は `Master/output/経理/` に残す。

## このファイルの役割

- 日常の経理振り分けルールの正本
- freee や証憑整理で迷ったときの判断基準
- 次回以降に AI が自律処理してよい範囲の明文化

### 依頼の最低フォーマット

- `この支払いは事業用。反映して`

用途が曖昧でなければ、詳細説明は要求しない。

## freee の位置づけ

- 現時点では `切らない` を基本線とする
- 理由は、`帳簿の正本 / 口座同期 / 自動で経理 / 電子申告 / 消費税申告 / 固定資産台帳 / データエクスポート` の役割を freee がまとめて持っているから
- このルールブックは `freee を置き換えるもの` ではなく、`freee を速く正しく使うための判断レイヤー`
- 将来 freee を切るなら、少なくとも次を別で用意してから
  - 口座・カード同期の仕組み
  - 帳簿の正本
  - 証憑保管
  - 所得税と消費税の申告フロー
  - 固定資産台帳
  - 出力・保全の手順

## freee 自動登録ルールの設計

### まず分ける

- `完全自動登録してよい`
- `推測表示までに留める`
- `自動化しない`

### 完全自動登録してよいもの

- `CLAUDE.AI SUBSCRIPTION` -> `通信費`
- `OPENAI *CHATGPT SUBSCR` -> `通信費`
- `STREAMYARD.COM` -> `通信費`
- `TLDX` -> `通信費`
- `GOOGLE *GSUITE / Google Workspace` -> `通信費`
- `1PASSWORD` -> `通信費` または `支払手数料`
- `LINE公式アカウント` -> `広告宣伝費`
- `GOOGLE *YOUTUBE MEMBER` -> `広告宣伝費`
- `DMM` の教材系 -> `新聞図書費`
- `キューエネスでんき` -> `水道光熱費`
- `上下水道料金` -> `水道光熱費`
- `お名前.com` -> `通信費`
- `ダイレクト出版` -> `新聞図書費` または `研修費`

### 推測表示までに留めるもの

- `Apple`
  - 事業用と個人用が混在する
- `Amazon`
  - 事業用と私用が混在する
- `QuickPay`
  - 交際費 / 会議費 / 私費が混在する
- `PayPayチャージ`
  - チャージは経費ではなく、実支払いの裏どりが必要
- `PASMOチャージ`
  - 実利用に分ける必要がある
- `スターバックスカードチャージ`
  - 実利用レシートが必要
- `Resident Insurance`
  - `60%按分` のため自動一本化しない
- `MODE`
  - 同種ルールはあるが、撮影当日かどうか確認が要る

### 自動化しないもの

- `伊勢丹 388,300円` のような高額交際費
- 旅行費用
- 10万円以上の新規高額支出
- 会社立替か個人経費か曖昧な明細
- 美容・衣服・日用品の混在支出

### freee に入れるべき実務ルール

- `条件が明確なSaaS` は `自動登録`
- `混在しやすいもの` は `推測表示`
- `高額 / 按分 / リスク高` は `手動判断`
- 既に同期済みの明細に新ルールを当てたいときは、freee の `最新の自動登録ルールを適用` を使う
- 自動登録されないときは、`税区分の競合 / ルール作成時刻 / 条件不一致 / 優先順位` を確認する

### 目標

- `月次で visible 未処理を小さく保つ`
- `確定申告期に残るのは高額論点だけ`

## オフライン領収書の処理

### 基本方針

- 紙レシートは `写真化`
- PDF はそのまま保存
- `事業用 / 私用 / 保留` を分ける
- freee に入る明細と対応付ける

### オフライン領収書でやること

1. 画像か PDF にする
2. ファイル名を `日付_店名_金額_用途` に近づける
3. Drive の `年/Brain` または `年/確定申告整理/用途` に移す
4. 事業用だけを freee のファイルボックスへ入れる
5. 対応する取引に添付する

### Drive の標準保管先

- `YYYY年/Brain`
  - Brain 領収書の正本
- `YYYY年/確定申告整理/保険・医療`
- `YYYY年/確定申告整理/光熱費`
- `YYYY年/確定申告整理/交際費`
- `YYYY年/確定申告整理/住居関連`
- `YYYY年/確定申告整理/衣装代`
- `YYYY年/確定申告整理/Amazon・機材・書籍`
- `YYYY年/確定申告整理/交通・決済`
- `YYYY年/確定申告整理/還付・通知`
- `YYYY年/確定申告整理/美容費`

用途が固まらないものだけを `Desktop/確定申告` に残し、処理済みは Drive へ抜く。

### freee 側で使う機能

- `ファイルボックス`
- `まとめてアップロード`
- `Google Drive から取り込み`
- `取引に添付`
- `重複チェック`

### AI が担当すること

- ファイル名の標準化
- 年別 / 月別の整理
- 事業用 / 私用の一次判定
- freee での対応明細の候補出し
- Drive 側の再配置

### ファイル名の標準形

基本形はこれに寄せる。

- `YYYY-MM-DD_店名_金額_用途.pdf`
- `YYYY-MM-DD_サービス名_金額_用途.jpg`

用途は以下に寄せる。

- `交際費`
- `会議費`
- `旅費交通費`
- `通信費`
- `広告宣伝費`
- `新聞図書費`
- `研修費`
- `住居関連`
- `保留`

### 人が最低限やること

- 写真を撮る
- 用途が曖昧なときだけ補足する

### freee と組み合わせる意味

- ファイルボックスで書類を保管できる
- 取引と証憑を紐づけられる
- 重複チェックがしやすい

## 判断の基本方針

- 法律に沿う
- 後から理由を説明できるものだけ入れる
- 保守的すぎず、説明可能なら必要経費に寄せる
- `チャージ` と `実際の支払い` を分ける
- `誰が払ったか` だけでなく `何に使ったか` と `いつ使い始めたか` で判定する
- 明細だけで確定できないものは `保留` に落とし、推測で登録しない

## ソースの優先順位

1. 領収書 / 請求書 / 利用明細
2. freee の明細と登録状況
3. ユーザーの明示説明
4. 過去の処理実績

## 売上・立替の基本ルール

### 売上に入れる

- 個人事業として受けた報酬
- 個人名で受けた入金で、事業対価と説明できるもの

### 売上に入れない

- 会社費用の立替精算
- 前年申告の還付金
- 利子や事業外資金

### 現時点の固有ルール

- `アドネス株式会社` の立替精算は `売上` ではなく `立替金の回収`
- `KASHIKA 493,800円` は個人の経費に入れない
- `Facebook代 月2,000円` は立替精算
- `新宿区系 1,100,000円` は前年申告の還付金として `今年の収入に入れない`

## チャージ系のルール

### PayPay

- `銀行 -> PayPayチャージ` は経費ではない
- 経費にするのは `PayPayで実際に払った取引`
- 判定は `支払い履歴CSV + 領収書 + 店名 + 用途`

### PASMO

- `銀行 -> PASMOチャージ` は経費ではない
- 実際の移動分だけを `旅費交通費` にする
- 現在の運用ルールは `90% 事業 / 10% 私用`

### スターバックスカード

- `チャージ` は経費ではない
- 実利用のレシートがある前提で、会議・打ち合わせ利用分を `会議費` で処理する
- 2025年の暫定ルールは `80% 事業 / 20% 私用`
- 紙レシートが全件あり、事業専用性がさらに固まるなら `100% 事業` に寄せる余地あり

## サービス別ルール

### AI・SaaS

- `Claude` は事業利用分を経費
- `ChatGPT` は事業利用分を経費
- `1Password` は事業用パスワード管理として経費
- `Google Workspace` は経費
- `StreamYard` は完全に事業用
- `TLDX` は Zoom 文字起こし用途として完全に事業用
- `Gamma` や `MeisterLabs` のような海外SaaSも、事業利用なら経費
- `freee` 利用料は事業経費
- `PLAUD` 本体のような `10万円未満の事業用端末` は `消耗品費`
- `ソフトウェア` 勘定は `固定資産として扱うソフト` に限る
- SaaS や小口アプリ、10万円未満の端末を `ソフトウェア` に入れると、確定申告で固定資産警告の原因になる

### Apple / iTunes

- `iCloud増量` は個人用
- `SNSや事業アプリ系` は事業用
- `Apple Music 個人契約` は個人用
- 用途不明なら、まず個人用に寄せる
- `Apple Store` の物販で、事業用の端末・周辺機器なら `消耗品費` か `固定資産`

### Amazon

- `確定申告フォルダで共有された事業用証憑あり` のものだけ経費候補
- それ以外の Amazon 小口は原則 `私用`
- `本・教材` は `新聞図書費`
- `撮影機材・事業用周辺機器` は `消耗品費` か `固定資産`

### 教材

- `ダイレクト出版` は教材費
- `Brain` はマーケティング教材として、年ごとに振り分けて管理
- `DMM` の教材系は `新聞図書費`
- `楽天カードサービス 6,600円` は教材費
- `びじねす幼稚園` は研修費
- `広告運用教材 217,800円` は、`振込証跡 + コンテンツ実体` があるので所得税側は前向き

## 支出カテゴリごとのルール

### 交際費

- 仕事仲間との会食
- 事業相談
- 仕事関係者へのプレゼント
- `店名 / 金額 / 相手 / 目的` のメモを残す

### 会議費

- 少額のカフェ打ち合わせ
- スタバ実利用分

### 旅費交通費

- 交通機関
- 事業出張のホテル
- 事業利用の PASMO 実利用分
- GO タクシーはカード明細との重複に注意

### 広告宣伝費

- LINE公式アカウント
- YouTube Member
- SNS広告関連
- `MODE（メンズ眉毛）` のように、`撮影当日の見た目調整` で広告宣伝費領収書があるもの

### 通信費

- AIツール
- SaaS
- ドメイン / サーバー

### 事業主貸

- `国民年金`
- `国民健康保険`
- これらは `必要経費` ではなく、確定申告の `社会保険料控除` で処理する
- 事業口座から払っていても、帳簿では `事業主貸 / 対象外`
- SEO自動化ツール

### 新聞図書費

- マーケティング本
- 教材本
- DMM などの学習コンテンツ

### 水道光熱費

- 電気
- ガス
- 水道
- 住居兼事務所は按分

### 地代家賃

- 現在ルールは `60% 事業`
- 家賃保証料や住居関連保険も同率按分が基本

## 2026年の標準処理

### 原則このまま進めてよいもの

- `Claude / ChatGPT / StreamYard / TLDX / Google Workspace / 1Password`
- `LINE公式アカウント / YouTube Member`
- `DMM / ダイレクト出版 / Brain / 事業用書籍`

- `スタバ実利用の会議費`

## 低リスク税金対策の原則

- 最優先は `お金を新しく使うこと` ではなく `取り漏れゼロ`
- 税務リスクが低く、AI だけで進めやすい順番は以下
  - `freee 未処理を月内で潰す`
  - `証憑を Drive 正本へ移す`
  - `固定資産 / ソフトウェア誤分類を定期点検する`
  - `売上 / 立替 / 還付金 / 事業主貸の混線を防ぐ`
  - `年末に消費税方式を再比較する`
- `2027年` から簡易課税を使うかは、`2026年10月〜11月` に再試算してから決める
- 現在の 2025 実数ベースでは `一般課税` が第一候補

## ルール更新の条件

### その場で rules に上げる

- 同じ判定が 3回以上出た
- 例外が少なく、自動登録ルールに落とせる
- 次回の判断速度を明確に上げる

### output に留める

- 今年だけの特殊事情
- 高額単発
- リスクが高く、都度説明が必要

## 衣装・美容のルール

### 経費に入れる

- `BEAMS` のように、撮影・イベント・プロフィール用途で完全に事業用と説明できる衣装
- 撮影当日の眉毛調整など、案件単位で事業関連性が強いもの

### 経費に入れない

- 通常の美容室
- ホワイトニング
- 日常の身だしなみ全般

理由:
- 仕事上の必要性があっても、私生活効用との切り分けが弱いものは入れない

## 固定資産ルール

- `10万円未満`: 原則その年の経費
- `10万円以上`: 固定資産か少額減価償却資産の検討
- `30万円以上`: 基本は固定資産
- `注文日` ではなく `受領 / 使用開始日` で年を切る

### 現時点の固有ルール

- `MacBook Pro 328,800円` は事業専用100%
- `EIZOモニター 161,500円` は 2026年分
- `モニターアーム 14,800円` は 2026年分

## 証憑の強さ

### 強い

- 領収書
- 請求書
- PDF明細
- 決済画面

### 中

- 銀行振込明細
- クレカ明細
- PayPay / PASMO / GO の利用履歴

### 弱い

- 記憶だけ
- 摘要だけの不明明細

## AI が自律処理してよい範囲

### 自動で進めてよい

- freee の未処理明細のうち、用途が固いものの登録
- チャージと実支払いの切り分け
- 証憑ファイルのリネーム
- 年度別フォルダ整理
- `私用` と明示されたものの除外

### 確認を入れる

- `10万円以上` の新規高額支出
- `交際費 10万円超`
- 旅行費用の全額経費化
- 私用混在が大きい美容・衣服・日用品
- 売上か立替精算か曖昧な入金
