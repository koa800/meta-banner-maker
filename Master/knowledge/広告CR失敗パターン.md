# 広告CR失敗パターン

最終更新: 2026-03-10

Meta広告CRの失敗は、`誰が作ったか` より `どこで崩れたか` で読む。
ここでは `勝ちCR 340件` を基準に、`非大当たり 9821件` を比較している。

## 集計対象

- 勝ち基準: `System/data/cr_bigwinners_raw.csv`（340件）
- 失敗ソース: `System/data/meta_cr_dashboard/full_failure_raw_latest.csv`（9821件）
- 失敗側の範囲: full_failure_raw_latest.csv から非大当たり 9821件を読み込み
- 注意: 失敗側は long-range の full raw。Meta広告 CR一覧の非大当たりを全件で読んでいる。

| CTR | 1.09% | 1.06% |

## まず読むべき本質

- 失敗全量 `9821` 件のうち、`1922` 件は `上流は通るが後ろで失敗` の候補に入る。
- つまり、失敗CRの主因を `フック不足` だけで説明しない。約束の質、クリック後の期待値、LP・オファー・導線の崩れを先に疑う。
- 一方で `タイトルの家系` には偏りがある。言い回し単体で勝てるわけではないが、同じ家系が繰り返し外れているなら失敗候補として重く見る。
- 失敗全量 `9821` 件の中だけでも、同一アセットの再利用群は `1076` 群 `3927` 件ある。CR単体ではなく `配信対象（オーディエンス） / 広告ID / 広告セットID / LP / 時期` を切って読む必要がある。

## Meta広告分析から見えた SNS広告共通の失敗メカニズム

- ここで見ているのは Meta広告CR だが、`フックを広く取りすぎて広いオーディエンスに寄る失敗` という構造自体は SNS広告全般で起こる。
- 失敗全量の内部だけでも、`同一アセット × 同日 × 同LP` まで揃えて数値を比べられる群が `802` 群 `2346` 件あり、そのうち配信対象差を読める群が `138` 群ある。
- さらに `ノンタゲ vs 類似` を直接比べられる群は `21` 群あり、CTR は `15/21` 群で広い側が低く、CPA は比較可能な `1/8` 群で広い側が重かった。
- 勝ちCR 340件の再利用群でも、同じ比較ができる `4` 群では CTR が `4/4`、CPA が `2/2` で広い側が不利だった。良いクリエイティブでも、広いオーディエンスに寄ると質が崩れやすい。
- つまり `広いフックで人を集める -> オーディエンスが広がる -> 浅い反応層に学習する -> CTRやフック率の見た目より後ろの質が弱くなる` を、SNS広告の代表的な失敗メカニズムとして常に疑う。

### なぜ広いフックになるのか

今の理解では、`広いフック` の良し悪しは `広いか狭いか` ではなく、`狙っているターゲット層に対して広いか / 狙っていない層の比率を急増させるか` で判断する。

- OK
  - 狙っているターゲットの中で、冷たい層から熱い層まで広く取れている
  - つまり `狙っている人の母集団の中で広い` 状態
- NG
  - 狙っていない人の比率が急増する
  - 反応は取れるが、ターゲットとしていない人まで大量に集める

本質的な失敗は、`広いフック` そのものではなく `狙っていない層にまでフックして、Meta の学習がそちらに寄る` ことである。

- `広い属性呼びかけ`
  - `あなた / 皆さん / 毎年やってない?` のような呼びかけは、自己認識コストが低い
  - 自己認識コストが低いこと自体は良い。数値につながる
  - 問題は、その呼びかけに `ターゲットを絞るワード` が弱いと、狙っていない人まで混ざりやすいこと
- `強い感情語`
  - `無駄にした / 失敗したくない / 人生変える` は、広く人を揺らしやすい
  - これは `街中で騒いでいる人を見る` のに近く、強いトリガーとして機能する
  - 問題は、揺らした相手がターゲットかどうかであり、感情を動かすこと自体ではない
- `無料特典`
  - 反応の閾値を下げること自体は良い
  - 問題は、`関係ない人にも届く条件` が重なると、狙っていない層まで拾いやすくなること
- `著名人/権威`
  - 著名人そのものがフックになる
  - `その人を知っている / 信頼している` という理由で、フックが1つ増える
  - これは悪いことではない。問題は、その著名人フックで入る人が `商品コンセプトとズレた層` を増やすかどうか
- `広い危機訴求`
  - `また一年無駄にした / このままだと終わる` は、多くの人が持つ後悔や不安に触れる
  - ただし危機の中身を支える `対象ワード` が弱いと、課題の種類が違う人まで集めやすい

### どこで対象を絞っているのか

対象を絞る主な要素は、`ワード` である。

- `フリーランスに興味がない方はスキップしてください`
  - `フリーランス` が対象を絞っている
- `AIイラスト素材を探す時代は終了`
  - `AI` と `イラスト素材` が対象を絞っている

つまり、広い呼びかけがあっても、同じシーン内で `対象ワード` が入っていれば、狙う層の中で広く取ることは可能である。

### 買う理由はどこで作るのか

フックの役割は、必ずしも `買う理由` をその場で完結させることではない。

- 30秒以下なら `CR + LP`
- 60秒以上なら `動画単体で買う理由` まで作れることもある
- 実際には `CR + LP + 検索` のように複数接点で成立することもある

したがって、フックの良し悪しは `その瞬間で買わせられるか` より `次の接点へ誰を送っているか` で見る。

### 商品コンセプトとの関係

後ろの設計は `商品コンセプト` に依存する。

- 商品コンセプトが広いなら、導線も広く受けられる
- 商品コンセプトが狭いなら、導線も狭く設計する必要がある

要するに、`広いフック` は悪ではない。悪いのは `商品コンセプトや狙う層に対して、関係ない人の割合を増やす広がり方` である。

### 具体例から見える因果

- `1年後悔` 系は、`また一年無駄にした / 毎回毎回 / あなたに / プレゼント / 人生変える` が同時に出やすい
  - これは `後悔している人` に広く反応させる構造
  - ここで見るべきは `検討の深さ` ではなく、`狙っている層の後悔か / 関係ない層の後悔まで拾っているか`
- `林さんCR` のように `全部学べる / これもいいね / 総合力` と広く魅力を並べる型は、何かしらには興味がある人を拾える
  - ただし本質は `ベネフィットが広い` ことより、`林さんを知っている / 信頼している` 人のフックが1つ増えることにある
  - つまり、林さん自体が反応トリガーになっている可能性を先に見る
- `お前今日何したんだよ` のような叱咤系は、感情の揺れは強い
  - これは `広く揺らす` 方向に働きやすい
  - ただし、揺れた相手がターゲットかどうかを見ないと判断を誤る

今の段階では、`広い属性呼びかけ` や `強い感情語` は「広く揺らしやすい条件」とまでは言える。
ただし、評価軸は `広いかどうか` ではなく `狙っているターゲットを広く取れているか / 狙っていない層が増えているか` である。

## この知識の使い方

- まず `どこで崩れたか` を決める。`誰が作ったか` や `単語単体` に逃げない。
- `上流は通るが後ろで失敗` に入ったら、CR単体の改善より `約束の質 / LP / オファー / 導線` を優先して見る。
- `押されるが集客単価が重い` は、興味は取れているので `質の低いクリック` か `期待値ズレ` を疑う。
- 同一アセット比較は `asset -> 配信対象（オーディエンス） -> 広告ID / 広告セットID -> LP -> 時期 -> 冒頭` の順で切る。順番を飛ばして `このCRは強い/弱い` と断定しない。
- 同じ家系が失敗側に偏っていても、1回では rules に上げない。複数スナップショットか下流数値が重なってから rules 化する。
