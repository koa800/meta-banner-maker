# DS.INSIGHT 評価レポート

> 最終更新: 2026-03-04
> ツールURL: https://dsinsight.yahoo.co.jp/

---

## DS.INSIGHTとは

LINEヤフーの行動ビッグデータ（Yahoo! JAPANの検索データ）を分析できるリサーチツール。
他のリサーチツール（DPro、Meta広告ライブラリ）が「供給側 = 広告主が何をしているか」を分析するのに対し、DS.INSIGHTは「需要側 = ユーザーが何を考えているか」の生データを提供する。

---

## 結論

Addnessにとって非常に有用なリサーチツール。活用は大きく2軸:

### 1. ターゲット理解（広告CR・リサーチ全般）

制作に入る前に、ターゲットへの理解を深めるために使う:
- ターゲットが**どんな言葉を使うか**（共起KW、検索語）
- その言葉の裏にある**本当の需要・インサイト**は何か
- **興味関心の方向先**がどこに向かっているか

→ 「何を言うか」を決める材料。CR訴求設計、LP設計、コンテンツ企画の全てに先立つ。

### 2. タイミング設計（導線 = Lステップ・メールマーケ）

Journeyの「特定KW検索の何日後に何が起きるか」を使う:
- 特定のトリガー（例: 「スキルプラス」検索）の後、何日目にどんな興味関心が発生するか
- そのタイムラインに合わせてオファー・コンテンツの配信タイミングを設計する

→ 「いつ言うか」を決める材料。Lステップシナリオ設計に直結。

## Trend（急上昇キーワード）詳細

### 機能構成

- **トレンドマップ**: 横軸=検索ボリューム、縦軸=トレンドスコア の散布図。3ヶ月/1年のスパン
- **急上昇ランキング**: 1日単位の突発的な検索ボリューム急上昇を検知。一過性のトレンドも多い
- **メールレポート**: トレンドマップの注目KWと急上昇KWを週2回（月曜・木曜）自動配信

### トレンドマップの4エリア定義（DS.LAB公式）

| エリア | トレンドスコア | 検索ボリューム(3ヶ月) | 検索ボリューム(1年) | 意味 |
|--------|-------------|-------------------|------------------|------|
| ネクストブレイク | 0.75 ~ 1.0 | ~5,000未満 | ~20,000未満 | 確実に伸びているがまだ認知が低い |
| ブレイク | 0.75 ~ 1.0 | 5,000以上 | 20,000以上 | 確実に伸びていて認知も高い |
| ポテンシャル | 0.5 ~ 0.75未満 | ~5,000未満 | ~20,000未満 | まだ伸びは不確定、潜在的 |
| ブーム | 0.5 ~ 0.75未満 | 5,000以上 | 20,000以上 | 認知は高いがトレンドスコアが低い（成熟/衰退期） |

**トレンドスコアの定義:**
- 右肩上がりで伸び続けている → スコア高い
- 一度伸びたが収束した → スコア低い
- 伸びが持続しない → スコア低い

### CR訴求に使えるかの判断基準（甲原の指針）

検索ボリュームの推移を見て判断する:

1. **角度が高まっている** → 関心が高くなっている。訴求に使える
2. **母数が増えてきた** → 関心がある。訴求に使える
3. **下がってきた** → 関心が薄れてきている。訴求の優先度を下げる
4. **完全に下がった** → 関心がない。訴求に使わない

### Addnessでの活用

- 「AI」「副業」カテゴリの急上昇キーワードを月次チェックし、新しいトレンドの予兆を捉える
- 例: ChatGPT → Gemini → Claude のようなツールトレンドのシフトを検知
- 新しいCR訴求のキーワード候補を発見する素材として活用
- トレンドマップで「ネクストブレイクエリア」に入ったキーワードは特に注目

## Place（来訪分析）詳細

- 特定キーワードを検索した人が、どの実店舗・施設を訪問しているかを分析
- **Addnessには不向き**: オンライン完結の事業モデルのため、実店舗来訪データの活用場面がほぼない
- 実店舗ビジネスの広告クライアントを持つ場合には有用だが、現時点では優先度低

### Journeyの時間軸の使い方（甲原の指針）

Journeyの時間区切りをどう使うかは、**自社のリソースとアプローチ範囲**で決める:

1. **どのタイミングでアプローチを仕掛けに行くか** — 全期間を網羅する必要はない。リソースに合わせてポイントを絞る
2. **同期間の複数キーワードは、そのターゲット層でまずやりきる** — 同じ時期に出ているキーワードは同じ層の関心を示している

## 他ツールとの差別化

| ツール | 分析対象 | 分かること |
|--------|---------|----------|
| DPro | 競合の広告CR | 何を言っているか（供給側） |
| Meta広告ライブラリ | 競合の出稿状況 | 誰が勝っているか（供給側） |
| DS.INSIGHT | ユーザーの検索行動 | ユーザーが何を考えているか（需要側） |

DProとMeta広告ライブラリが「供給側」の分析なのに対し、DS.INSIGHTは「需要側」の生データ。
この組み合わせで市場理解の解像度が一段上がる。

## データの留意点

### Yahoo! JAPAN検索ユーザーの偏り

DS.INSIGHTのデータはYahoo! JAPANの検索データがベース。以下の偏りがある:

- **年齢層が高め**: Google検索と比較して40代以降の利用者が多い。若年層（20代以下）の行動はこのデータだけでは捉えきれない
- **「マス」に近い層**: ITリテラシーが平均的〜やや低い層が多い印象。テック系・スタートアップ系の人はGoogle検索に偏る
- **地方にも厚い**: Yahoo! JAPANは地方でのシェアがGoogleより相対的に高い

### Addnessにとっての意味

- スキルプラスのターゲット（40代最多、30〜50代で71%）とYahoo!ユーザー層は**むしろ合致している**
- 「会社員で副業を検討している40代」はまさにYahoo!で検索する層
- このデータソースはAddnessの主要ターゲットの行動を捉えるのに適している
- ただし**20代以下のターゲティングにはこのデータだけでは不十分**。若年層を狙う場合はSNS分析（Instagram、TikTok）で補完する必要がある

## 最も重要な使い方: 表面ではなく背景の需要を読む

DS.INSIGHTが出してくれるのは「キーワード」という表面データ。だがこのツールの本当の価値は、**キーワードの裏にある需要構造を深く読み取ること**にある。

### 甲原の実践例

「起業」を調べている人のJourneyデータに「アイデア」「本」が出てきたとする。

- 表面的な分析: 「アイデア」というキーワードで訴求しよう
- 深い分析: 「起業したいけどアイデアがない」という悩みが背景にある。この人には「起業のアイデア」という切り口が刺さる。単にキーワードを拾うのではなく、**そのキーワードを調べている人が何を求めているのか**を読み取る

### つまり

```
DS.INSIGHTが出すもの = キーワード（表面データ）
                ↓
人間が読み取るもの = そのキーワードを調べる背景にある需要（インサイト）
                ↓
訴求に落とし込む = 表面のキーワードではなく、背景の需要に刺す言葉を作る
```

これは広告リサーチスキルの「避けたいコスト × 満たしたい需要」のフレームワークとそのまま繋がる。DS.INSIGHTはインサイト発見のための素材を提供するツールであり、**インサイトを読み取る作業は深い思考が必要**。

### スキルプラスへの応用例

Journeyデータに「ココナラ」「クラウドワークス」が検索前に出ている場合:

- 表面: 「ココナラ」「クラウドワークス」というキーワードを使おう
- 深い読み: この人たちは「スキルを売る場所を探しているが、まだ売れるスキルがない」という状態。だからスキルプラスにたどり着いた。訴求は「スキルを身につけて売れる側になる」が刺さる

Journeyデータに「スキルプラス 評判」が検索後1〜7日に出ている場合:

- 表面: 評判ページのSEOを強化しよう
- 深い読み: この人は「興味はあるけど信用していない」段階。検索後すぐに評判を調べるということは、まだ不安が解消されていない。Lステップの初期配信で**受講生の具体的な変化ストーリー**を出すことで、この不安を先回りして解消できる

## 具体的な活用方法

### 活用1: 月次の市場温度チェック（定点観測）

**目的**: 市場の需要変化を定期的に捉え、訴求やコンテンツのズレを早期発見する

**毎月取得するデータ:**

| データ | プロダクト | キーワード | 見るポイント |
|--------|-----------|-----------|------------|
| 検索ボリューム推移 | Basic | 「スキルプラス」 | ブランド認知が上がっているか・下がっているか |
| 前後検索キーワード | Journey | 「スキルプラス」 | ユーザーの検討行動に変化がないか |
| 共起キーワード | Basic | 「AI 副業」 | 市場で話題になっている関連キーワードの変化 |
| 急上昇キーワード | Trend | AI・副業カテゴリ | 新しいトレンドの予兆 |

**アクション**: 前月と比較して変化があれば、CR企画やLステップシナリオの調整を検討

### 活用2: Lステップシナリオ設計の裏付け

**目的**: オファーのタイミングをデータで決める（勘ではなくデータ）

**Journeyデータの活用フロー:**

```
1. 「スキルプラス」のJourneyデータを取得
2. 検索後の行動タイムラインを確認
   - 1〜7日後: 評判チェック → 信頼構築コンテンツをオファー
   - 7〜31日後: ツール学習 → 実践コンテンツをオファー
   - 31日以降: 収益化 → 収益化ロードマップをオファー
3. Lステップのシナリオ配信日をこのタイムラインに合わせる
4. 月次で再取得し、タイムラインに変化があればシナリオを調整
```

### 活用3: 広告CR訴求の言葉選び

**目的**: ターゲットが実際に使っている言葉・気にしていることを訴求に反映する

**共起キーワードの活用フロー:**

```
1. 「副業」「AI スキル」等の共起キーワードを取得
2. ユーザーが気にしていること（バレる、確定申告、在宅 等）を抽出
3. その「気にしていること」= 避けたいコストとして、CR訴求に盛り込む
   例: 「バレる」が共起上位 → 「会社にバレずに」ではなく「本業と両立しながら」で間接表現
4. 新しい共起キーワードが出てきたら、新しいCR訴求の候補として検討
```

### 活用4: 競合モニタリング

**目的**: 競合サービスの認知変化やユーザーの比較行動を把握する

**確認するキーワード:**
- 「SHIFT AI」「udemy」「スタディング」等の競合名
- それぞれのJourneyで、ユーザーがどこからどこに流れているかを把握
- 「スキルプラス」のJourney上に新しい競合名が出てきたら要注意

## 運用ルール

- DS.INSIGHTのデータ取得は月次で実施（月初に前月分を確認）
- 取得結果はこのファイルに追記せず、リサーチレポート（dpro_research_YYYYMMDD.md）にまとめる
- 大きな変化があった場合のみ広告リサーチスキル（Skills/1_広告/広告リサーチ.md）のデータを更新
- Yahoo!検索データの偏りを常に念頭に置き、若年層のデータはSNS分析で補完する
- メールアラートを活用し、主要キーワードの変化を自動通知で受け取る

---

## メールアラート設定（推奨）

DS.INSIGHTのメールアラート機能で、以下のキーワードを登録しておくと月次チェックの補完になる:

| キーワード | 目的 |
|-----------|------|
| スキルプラス | ブランド認知の変化検知 |
| AI 副業 | 市場トレンドの変化検知 |
| 副業 スキル | ターゲット需要の変化検知 |

---

## 解決済みの疑問点

### Journey N数 → 見える

ランキングタブの「重複ボリューム」列で、各キーワードを検索した人数（N数）が確認できる。
「検索時間差」列で平均何日前/後かも数値で分かる。詳細は上記ランキングデータ参照。

### 共起キーワードの分類 → DS.INSIGHTの自動カテゴリを使う

DS.INSIGHTが提供するカテゴリ分類をそのまま活用する。

### Trendの定義 → トレンドスコア0.75以上が「確実に伸びている」

トレンドマップの4エリア定義は上記Trend詳細セクション参照。
急上昇ランキングは1日単位の突発的変化で、一過性のトレンドも多い。

### CR持続性の判断 → 検索ボリューム推移の角度と母数で判断

甲原の指針: 角度が高まる=関心増、母数増=関心あり、下がる=薄れ、完全低下=関心なし。

## 月次レポートのフォーマットパターン

月次のDS.INSIGHTデータをどのフォーマットでまとめるか。4つのパターンを洗い出した。

### 採用フォーマット: ハイブリッド型

定点観測 + 変化検知 + インサイトを1つにまとめたフォーマット。
月次（または2週間に1回）で実施する。

```
# DS.INSIGHTレポート YYYY年M月

## 1. 定点観測（主要KW）

| キーワード | 今回 | 前回 | 変化 | 判断 |
|-----------|------|------|------|------|
| スキルプラス | 7,300 | 7,100 | +2.8% | 微増。認知は維持 |
| 副業 | 156,000 | 157,300 | -0.8% | 安定。市場は健在 |
| AI 副業 | xx,xxx | xx,xxx | +x.x% | （角度/母数で判断） |

## 2. 変化があった項目

### Journey: ○○が新規出現
- 「○○」が検索後7日のランキングに新規出現（重複Vol 120人、特徴度2.8）
- 前回にはなかったキーワード

### Trend: △△がネクストブレイクエリアに移動
- トレンドスコア0.82、検索ボリューム3,200
- 前回はポテンシャルエリアだった

### Basic: 共起KW「□□」が上位に浮上
- 前回15位 → 今回8位

（変化がない項目は省略）

## 3. このデータから何が言えるか

### 市場の空気
（1-2文で温度感を要約）

### インサイト
1. ○○が出てきた背景には「△△したいけど□□がない」という需要がある
   - データ根拠: Journey検索時間差-XX日、重複Vol XX人
   - 推奨アクション: CR訴求に「□□」の切り口を追加検討

2. Trendで△△が伸びているのは、市場が○○に移行している兆候
   - データ根拠: トレンドスコア推移 0.55 → 0.82
   - 推奨アクション: 次回のCR企画で取り入れるか検討

### 次回の注視ポイント
- ○○の重複Volが増え続けるか（今回120人 → 次回XX人）
- △△がブレイクエリアに移行するか
```

**このフォーマットのポイント:**
- セクション1: 主要KWは毎回必ず記録。時系列で追える
- セクション2: 変化がある項目だけ書く。変化がなければ短い
- セクション3: 毎回必ずインサイトを書く。データの裏にある需要構造を読み取り、推奨アクションまで落とし込む

**頻度:** 月次 or 2週間に1回
