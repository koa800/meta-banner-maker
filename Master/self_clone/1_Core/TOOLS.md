# 使えるツール一覧

> **目的**: エージェントが使用できるツールの一覧と使い方を定義する
> **読み込み対象**: メインエージェント、サブエージェント（両方）
> **更新頻度**: 中（新しいツールを追加したら更新）

---

## 📋 このファイルについて

このファイルは、エージェントが使用できる**ツール**を定義します。

- **検索ツール**: 情報を検索する
- **分析ツール**: データを分析する
- **生成ツール**: コンテンツを生成する
- **管理ツール**: タスクを管理する

---

## 🔍 検索ツール

### Web検索

**機能**: インターネット上の情報を検索する

**使い方**:
```
1. 検索キーワードを決める
2. 検索を実行
3. 結果を評価する
4. 必要に応じて追加検索
```

**注意点**:
- 情報の信頼性を確認する
- 複数のソースを参照する
- 最新の情報を優先する

---

### データベース検索

**機能**: 過去の記憶・記録を検索する

**使い方**:
```
1. 検索対象を決める（感情ログ、インプット記録、フィードバックなど）
2. キーワードで検索
3. 関連する記録を抽出
4. 文脈を確認
```

**注意点**:
- 記録の日付を確認する
- 文脈を考慮する
- 古い情報は現在と異なる可能性がある

---

## 📊 分析ツール

### テキスト分析

**機能**: テキストデータを分析する

**使い方**:
```
1. 分析対象のテキストを用意
2. 分析の目的を明確にする（価値観抽出、パターン発見など）
3. 分析を実行
4. 結果を整理
```

**注意点**:
- 文脈を考慮する
- 量だけでなく質も見る
- バイアスに注意する

---

### データ分析

**機能**: 数値データを分析する

**使い方**:
```
1. データを収集
2. データを整理
3. 分析手法を選ぶ（集計、比較、相関など）
4. 結果を可視化
```

**注意点**:
- データの品質を確認する
- 適切な分析手法を選ぶ
- 結果を過度に一般化しない

---

## ✍️ 生成ツール

### 文章生成

**機能**: テキストコンテンツを生成する

**使い方**:
```
1. 目的を明確にする（SNS投稿、ブログ記事など）
2. SOUL.mdとIDENTITY.mdを参照
3. 文章を生成
4. 評価基準でチェック
5. 必要に応じて修正
```

**注意点**:
- 価値観に合致しているか確認
- 言語スタイルが再現されているか確認
- 事実と意見を区別する

---

### アイデア生成

**機能**: 新しいアイデアを生成する

**使い方**:
```
1. テーマを明確にする
2. SOUL.mdの「発想パターン」を参照
3. 各パターンでアイデアを生成
4. 評価・選別
```

**注意点**:
- 複数のパターンを試す
- 独自性を重視する
- 実現可能性も考慮する

---

## 🗂️ 管理ツール

### タスク管理

**機能**: タスクを管理する

**使い方**:
```
1. タスクをリストアップ
2. 優先順位をつける
3. 期限を設定
4. 進捗を記録
```

**注意点**:
- 重要度と緊急度を区別する
- 定期的に見直す
- 完了したタスクは記録する

---

### 記憶管理

**機能**: 記憶を整理・保存する

**使い方**:
```
1. 記録すべき情報を特定
2. 適切なカテゴリに分類（感情ログ、インプット、フィードバック）
3. フォーマットに従って記録
4. 定期的に見返す
```

**注意点**:
- 重要な情報を優先する
- 日付を明記する
- 文脈を残す

---

## 🔄 ツールの組み合わせ

### 典型的なワークフロー

#### 文章作成のワークフロー

```
1. Web検索（情報収集）
   ↓
2. テキスト分析（情報整理）
   ↓
3. アイデア生成（構成を考える）
   ↓
4. 文章生成（執筆）
   ↓
5. 評価（品質チェック）
   ↓
6. 記憶管理（記録）
```

#### 判断のワークフロー

```
1. データベース検索（過去の類似ケースを探す）
   ↓
2. データ分析（選択肢を比較）
   ↓
3. SOUL.mdの判断軸を参照
   ↓
4. 判断
   ↓
5. 記憶管理（判断の記録）
```
