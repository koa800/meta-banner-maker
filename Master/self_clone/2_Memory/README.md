# 記憶システム

> **目的**: 過去の記憶を蓄積し、エージェントが「学習」できるようにする
> **読み込み対象**: メインエージェントのみ
> **更新頻度**: 高（継続的に追加）

---

## 📋 このディレクトリについて

このディレクトリは、エージェントの**記憶**を保存します。

- **emotions/**: 感情ログ（感情と思考の関係性）
- **inputs/**: インプット記録（読んだ本、見た動画など）
- **feedback/**: フィードバックループ（実行→結果→学習）

## 💭 emotions/ - 感情ログ

### 目的

感情の動きを記録し、思考・判断・創造性との関係性を可視化する。

### ファイル形式

**ファイル名**: `YYYYMM.md`（例: `202602.md`）

**内容**:
```markdown
# 感情ログ - YYYY年MM月

## YYYY-MM-DD HH:MM：[感情の状態]

- **感情の種類**: [興奮, 不安, 怒り, 喜び, 落ち着き, 焦り, 違和感, 共感など]
- **感情の強度**: [1-10]
- **きっかけ**: [何がこの感情を引き起こしたか]
- **身体的な状態**: [疲労度, 睡眠時間, 体調など]
- **この感情の時に生まれた思考・アイデア**:
  - [思考1]
  - [思考2]
- **この感情の時に下した判断**:
  - [判断1]
- **振り返り**:
  - [この感情は思考にどう影響したか]
```

### 記録のタイミング

```
□ 強い感情を感じた直後
□ 重要な判断を下した直後
□ アイデアが浮かんだ直後
□ 週に1回、まとめて記録（HEARTBEAT.mdの週次チェック時）
```

## 🔄 feedback/ - フィードバックループ

### 目的

実行した行動の結果を記録し、判断軸やパターンを継続的に改善する。

### ファイル形式

**ファイル名**: `YYYYMM.md`（例: `202602.md`）

**内容**:
```markdown
# フィードバックループ - YYYY年MM月

## YYYY-MM-DD：[行動のタイトル]

### 計画
- **目的**: [何を達成したいか]
- **仮説**: [こうすれば、こうなるはず]
- **使った判断軸/パターン**: [どの判断軸やパターンを使ったか]

### 実行
- **実際にやったこと**: [具体的な行動]
- **実行時の感情**: [どんな感情状態だったか]

### 検証
- **結果**: [何が起きたか、数値データがあれば記載]
- **仮説との差異**: [仮説通りだったか、違ったか]
- **なぜこの結果になったか（分析）**: [原因の考察]

### 改善
- **次に活かすこと**: [次回、どう改善するか]
- **マスターファイルの更新内容**: [どのファイルを、どう更新するか]
```

### 記録のタイミング

```
□ 重要な行動を実行した直後
□ 結果が出た直後（数日〜数週間後）
□ 週に1回、まとめて記録（HEARTBEAT.mdの週次チェック時）
□ 月に1回、全体を振り返り、傾向を分析（HEARTBEAT.mdの月次チェック時）
```

## 📝 記憶の管理

### ファイルサイズが大きくなった場合

```
1. 月別にファイルを分割する（すでに実施済み）
2. 重要な記録を抽出し、SOUL.mdやIDENTITY.mdに反映する
3. 古い記録は、アーカイブディレクトリに移動する
```

### 記憶の品質を保つために

```
□ 記録する際は、具体的に書く
□ 感情や思考のプロセスも記録する
□ 数値データがあれば記録する
□ 定期的に見返し、パターンを発見する
```
