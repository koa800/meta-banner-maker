# セットアップガイド

> **目的**: セルフクローン構築システムのセットアップ手順を説明する
> **対象**: 初めてこのシステムを使う人

---

## 📋 セットアップの流れ

```
ステップ1: データ収集
    ↓
ステップ2: データ分析・抽出
    ↓
ステップ3: ファイル作成
    ↓
ステップ4: テスト実行
    ↓
ステップ5: 評価・改善
```

---

## 📚 ステップ1: データ収集

### 必要なデータ

あなたの思考・価値観・言語スタイルを抽出するために、以下のデータを収集します。

#### 必須データ

```
□ SNS投稿（過去3-6ヶ月分）
  - Twitter/X, Facebook, LinkedIn, Instagramなど
  - できるだけ多くの投稿を収集

□ ブログ記事・note記事
  - 過去1年分程度
  - 長文の記事ほど、思考パターンが明確に表れる

□ メール・チャット履歴
  - 仕事のメール、友人とのチャット
  - 自分の「素」の言葉が表れる
```

#### 推奨データ

```
□ 日記・メモ
  - 個人的な思考の記録

□ プレゼン資料・レポート
  - 専門分野の知識や思考プロセス

□ 動画・音声の文字起こし
  - YouTube動画、ポッドキャスト、講演など
```

### データ収集の方法

詳細は `DATA_COLLECTION.md` を参照してください。

---

## 🔍 ステップ2: データ分析・抽出

### 抽出する情報

収集したデータから、以下の情報を抽出します。

```
1. 価値観（審美眼）
   - 何を大切にしているか？
   - 何が好きで、何が嫌いか？

2. 発想パターン（ひらめき）
   - どのように考えるか？
   - どのようにアイデアを生むか？

3. 判断軸
   - どのように決断するか？
   - 何を優先するか？

4. 言語スタイル
   - どのように話すか？
   - どんな言葉を使うか？
```

### 抽出の方法

詳細は `EXTRACTION_TEMPLATE.md` を参照してください。

---

## ✍️ ステップ3: ファイル作成

### 作成するファイル

抽出した情報を、以下のファイルに記入します。

#### 1_Core/（コアファイル）

```
□ SOUL.md
  - 抽出した「価値観」「発想パターン」「判断軸」を記入

□ IDENTITY.md
  - 抽出した「言語スタイル」を記入

□ USER.md
  - あなたの基本情報、興味関心、現在のプロジェクトを記入

□ HEARTBEAT.md
  - 定期チェック項目を確認・カスタマイズ

□ AGENTS.md
  - そのまま使用（カスタマイズ不要）

□ TOOLS.md
  - そのまま使用（必要に応じてツールを追加）
```

#### 2_Memory/（記憶ファイル）

```
□ emotions/
  - 最近の感情ログを記録（過去1ヶ月分程度）

□ inputs/
  - 最近のインプットを記録（過去1ヶ月分程度）

□ feedback/
  - 最近のフィードバックを記録（過去1ヶ月分程度）
```

### ファイル作成のコツ

```
1. 完璧を求めない
   - まずは骨格を作る
   - 使いながら、少しずつ埋めていく

2. 具体例を多く入れる
   - 抽象的な説明だけでなく、具体例を入れる
   - AIは具体例から学習する

3. 自分の言葉で書く
   - テンプレートをそのまま使わない
   - 自分の言葉で書き直す
```

## 🧪 ステップ4: テスト実行

### テストの流れ

```
1. AIに7つのファイルを読み込ませる
   ↓
2. テストプロンプトを実行
   ↓
3. 出力を評価
   ↓
4. 問題があれば、ファイルを修正
   ↓
5. 再テスト
```

### テストプロンプト例

#### テスト1: 文章生成

```
【依頼】
以下のテーマについて、私の思考OSで文章を書いて：

■ テーマ: [最近話題のニュース]
■ 文字数: 300字程度
■ 形式: SNS投稿

■ 指示
- 私の「審美眼」に基づいて、価値観を反映して
- 私の「言語スタイル」に基づいて、文体を再現して
```

#### テスト2: アイデア出し

```
【依頼】
以下のテーマについて、私の発想パターンでアイデアを出して：

■ テーマ: [新しいプロジェクトのアイデア]

■ 指示
- 私の「ひらめきパターン」を使って、3つのアイデアを出して
- 各アイデアに、なぜこのパターンでこのアイデアが生まれたか説明を付けて
```

#### テスト3: 判断代行

```
【依頼】
以下の選択について、私ならどちらを選ぶか判断して：

■ 状況: [具体的な状況]
■ 選択肢: A: [選択肢A] / B: [選択肢B]

■ 指示
- 私の「判断軸」に基づいて、判断して
- 判断理由を、私の価値観から説明して
```

## 📊 ステップ5: 評価・改善

### 評価基準

詳細は `3_Evaluation/EVALUATION_CRITERIA.md` を参照してください。

### 評価のポイント

```
□ 価値観が反映されているか？
□ 発想パターンが再現されているか？
□ 判断軸が一致しているか？
□ 言語スタイルが再現されているか？
□ 「自分が書いた/考えたもの」と区別がつかないか？
```

### 改善のサイクル

```
1. 評価結果を確認
   ↓
2. 問題点を特定（どのファイルに問題があるか）
   ↓
3. ファイルを修正（具体例を追加、表現を調整など）
   ↓
4. 再テスト
   ↓
5. 評価スコアが4.0以上になるまで繰り返す
```

---

## 🎯 セットアップ完了の目安

### 合格ライン

```
□ 評価スコア4.0以上
□ 「自分らしい」と感じられる出力が生成される
□ 大幅な修正なしで、そのまま使える
```

### 完璧を求めすぎない

```
- 最初から100点を目指さない
- 70-80点で「合格」とし、使いながら改善する
- 使えば使うほど、精度が向上する
```

## 🔄 セットアップ後の運用

### 日常的な運用

```
□ AIとの対話開始時に、7つのファイルを読み込ませる
□ 出力を評価し、必要に応じて修正
□ 週1回、USER.mdとmemory/を更新
```

### 月次の運用

```
□ HEARTBEAT.mdの月次チェックを実行
□ SOUL.md, IDENTITY.mdを見直し、必要に応じて更新
□ 評価スコアを確認し、精度を測定
```

## 🆘 トラブルシューティング

### 質問1: 出力が「自分らしくない」

```
回答: 以下を確認してください：
1. SOUL.mdに具体例が十分に入っているか？
2. IDENTITY.mdに実際の発言例が入っているか？
3. 抽出したデータが古すぎないか？（最近3-6ヶ月のデータを使う）
```

### 質問2: 評価スコアが3.0未満

```
回答: 以下を試してください：
1. データ量を増やす（SNS投稿、ブログ記事を追加）
2. 具体例を追加する（SOUL.md, IDENTITY.mdに実例を追加）
3. テンプレートを自分の言葉で書き直す
```

### 質問3: どのファイルから始めればいいかわからない

```
回答: 以下の順番で進めてください：
1. IDENTITY.md（言語スタイルが最も目に見えやすい）
2. SOUL.md（価値観、発想パターン、判断軸）
3. USER.md（基本情報、興味関心）
4. memory/（過去1ヶ月分の記録）
```
