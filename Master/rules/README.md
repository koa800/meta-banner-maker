# rules レイヤー

最終更新: 2026-03-09

`Master/rules/` は、再現ルールと NG を置く場所です。

- 入口: `rules.md`
- 具体ルール: `広告・導線ルール.md`
- 役割: 成功失敗から抽出した判断ルールを残す
- 昇格先: 3回以上再利用できる原理は `Skills/`

ルール化の前に、`事実 -> 観察 -> 解釈 -> 確度` を分けて残す。

## 昇格の目安

- `output 止まり`: 単発、または仮説が多い
- `knowledge 候補`: 事実と観察はあるが、まだ禁止や推奨に圧縮できない
- `rules 候補`: `確定` または `強い推定` が 2件以上あり、次回の判断に効く
- `Skills 候補`: 人や案件をまたいで 3回以上再利用でき、条件と手順を独立して説明できる
- `Skills 昇格可`: 上記に加えて、`対象条件` `手順` `失敗しやすい条件` が切り分けられ、甲原固有の前提を外しても機能する

迷ったら rules に上げず、1段下に留める。

## Skills 昇格の即時ルール

- 甲原さんが会話で `これスキルにして` と明示したものは、通常の再利用回数より優先して `Skills 候補` として扱う
- ただし、その場で最低限 `対象条件` `やり方` `失敗しやすい条件` を分けられないものは、いったん `rules` または `knowledge` に留める
- `即時昇格` はできるが、`定義が曖昧` `本人依存が強すぎる` `単発事例しかない` 場合は、その理由を残して一段下に留める
- 昇格時は `なぜ skill 化するか` を短く残し、以後は `Skills/` を正本にする

## 失敗知識の優先

- うまく行く条件は多く、失敗する条件は限られる
- そのため、勝ち筋の網羅より先に `失敗パターンの抽出` を重く見る
- 失敗CRや失敗導線は、単発レビューより `型の抽出` を優先する
