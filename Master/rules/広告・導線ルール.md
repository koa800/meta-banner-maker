# 広告・導線ルール

最終更新: 2026-03-10

このファイルは、既存の LP / CR 分析とファネル構造分析から抽出した「次回の判断にそのまま使うルール」を置く。

## 参照元

- `Master/addness/meta_ads_大当たりCR分析.md`
- `Master/addness/funnel_structure.md`

## 推奨

### 1. CR の主軸は `危機感 / 変化 / 実用` から組む

- 大量配信の主軸は危機感訴求で取る
- CPA を下げたいときは変化訴求を優先候補に置く
- CPO と後ろの質まで取りたいときは実用訴求を優先する

### 2. 実用訴求は `何が学べるか` を具体にする

- `プロンプト` `勉強法` `ChatGPTにどう指示するか` のように学習内容を具体化する
- クリック前の約束と LP の約束を揃える
- 抽象的なベネフィットだけで押し切らず、受け取れる中身を明示する

### 3. 危機感訴求では `間違い` 型を優先検証する

- `これ間違いです` 型はフック率の異常値が出ている
- 単発で終わらせず、対象者とテーマを変えて横展開候補として扱う
- ただし煽りだけで終わらせず、LP で本質訴求に着地させる

### 4. ターゲットは必ず表に出す

- `会社員` `フリーランス` `副業` のように、誰向けかを CR の時点で明示する
- 刺さる人数を広げるより、刺さる解像度を上げる

### 5. 動画を基本形にする

- 大当たりの主戦場は動画
- 画像は例外運用と考え、急募系など勝ち筋が確認できた場合だけ使う

### 6. 広告集客の大きい変数は `CR / 運用 / LP / 時期` で切る

- 広告集客で大きく効く変数は、まず `CR`、次に `運用`、次に `LP`
- ただし、CR には `旬` があるので `時期` も必ず含める
- `今見られている` を `恒久的に強い` と誤認しない
- 分析順は `CR自体 -> 運用差 -> LP差 -> 時期差` を基本にする

### 7. `時期` は日付ではなく `社会背景 / 市場背景 / 旬` まで含めて読む

- 同じCRでも、時代や時期で見られ方は変わる
- 例えば `年末の後悔訴求` のように、メッセージと社会的空気が噛み合うと通年より伸びることがある
- `今月強い` をそのまま rule 化せず、`なぜその時期に強いのか` を市場背景まで含めて観察する

### 8. 新規発明より `型の横展開` を優先する

- 当たったテーマは別ファネル、別ターゲット、別クリエイティブへ横展開して検証する
- 毎回ゼロから新角度を作るより、勝ち型の変奏を増やす

### 9. 導線は `CR -> LPの約束 -> 最初のメール件名 -> 面談化CTA` で確認する

- 入口 LP が複数でも、下流のシナリオや Journey が共通化されていることがある
- その場合、LP だけ差し替えても、最初のメール件名や CTA がズレると違和感が出る
- CR の約束と LP の約束だけでなく、その後の Mailchimp 件名と面談化 CTA まで一続きで確認する

### 10. short.io の slug まで導線設計に含める

- 導線設計は LP や LINE 名だけで終わりではない
- `どの short.io slug を使うか` と `URL管理シートにどう残すか` までを1セットで扱う

### 11. 高フック率でも `勝ち` と決め打ちしない

- `3秒フック率が高い = 事業成果が高い` ではない
- 実際に、フック率 50% 超の CR でも `はずれ` や `許容KPI未達成` が混ざっている
- 判定は `フック率 -> CTR -> CPA -> CPO -> 面談化以降` の順に落として確認する

### 12. 同じインサイトでも `言い方` は価値訴求に寄せる

- 同じ需要でも、直接的な稼ぎ訴求より `一生使える` `本業でも武器になる` `時間を取り戻せる` のような価値訴求へ寄せる
- 直接訴求で CPA が取れても、LTV が崩れるなら軸にはしない
- ユーザーに `自分にも使える` を想起させるが、`楽して稼げる` とは言わない

## NG / 注意

### 1. CPA だけで勝ち負けを判断しない

- CPA が安くても CPO や後ろの ROAS が崩れるパターンがある
- 判断は最低でも `CPA + CPO + 面談化以降` を並べて行う

### 2. 急募 / FOMO を主軸にしない

- 集客単価が良く見えても、後ろの質が落ちやすい
- 使うなら補助訴求として扱い、基幹訴求にはしない

### 3. 著名人・権威を主軸にしない

- 短期の CPA / CPO が良くても、Meta の学習が著名人興味層に寄る
- 自社ブランド単体の広告効率を壊しやすい
- 権威は補助要素に留め、軸は自社訴求で作る

### 4. UTAGE の `アーカイブ` 表示を停止判定に使わない

- 現役 LP から `div-archived` 扱いのページへ送っているケースがある
- `アーカイブ = 未使用` と決め打ちしない

### 5. URL 管理シートの `最新フラグ` だけで現役判定しない

- 現状はシート運用が揺れている
- 現役判定は実際の CTA、シナリオ、Journey、遷移先で確認する

### 6. 共通シナリオ配下で LP の約束を勝手に変えない

- AI Meta のように LP は複数でも、オプトイン処理と下流 Journey は共通のことがある
- 入口の約束を変えるなら、共通メールの件名と CTA の整合までセットで見る

### 7. `年齢 + 金額 + 秘密` の直接訴求を主軸にしない

- `45歳でも` `月収50万` `秘密教えます` のような並びは低LTV層を呼びやすい
- CPA が機能しても、顧客の質まで良いとは限らない
- 同じ需要を扱うなら、年齢や金額の即物訴求ではなく、変化後の状態や価値で見せる

### 8. `楽に / 面倒くさがりでも / 頑張らない` を直球で言わない

- ハードルを下げる意図があっても、そのまま言うと顧客の質を落としやすい
- 代わりに `具体手順` `段階的な変化` `短時間でできる範囲` でハードルの低さを伝える

### 9. 同じ型の派生でも 1本ずつ別物として評価する

- `尻込み` のように同じモチーフでも、当たりと外れが混在する
- 型の横展開は有効だが、モチーフが同じという理由で次作を自動的に当たり扱いしない
- 特に `フック率だけ高い派生` は誤判定しやすい

### 10. `許容KPI達成` をそのまま採用判断に使わない

- `はずれ` なのに `許容KPI達成` の CR が実在する
- 局所KPIを満たしていても、後ろの質や総合判定が悪ければ主軸にしない
- 判定は `許容達成か` ではなく、`次回も使う価値があるか` で見る

### 11. LP の失敗理由を LP 単体だけで断定しない

- LP が外れたように見えても、原因は `CRの約束` `LPの約束` `最初のメール件名` `主CTA` `下流導線` にまたがる
- 少なくとも `オプトインCVR / CPA / CPO / 面談化以降` のどこで崩れているかを見てから判断する
- 数値が揃わない段階では、原因は `仮説` として output に残し、確定ルールに上げない

### 12. 同じ LP 番号に勝ち負けが混在するなら、LP 単体を犯人扱いしない

- 同じ LP1 でも、同一モチーフの CR 群に `目標KPI達成` と `未達` が混在することがある
- この場合、LP番号だけを見て `このLPは悪い` と判断しない
- まず `CR差分` `配信条件` `最初のメール件名` `CTA` の違いを比較する

### 13. 同一クリエイティブ比較は `配信対象 / LP / 時期` を固定せずに断定しない

- 同じ動画URLでも `類似1%` と `ノンタゲ`、`LP1` と `LP2`、配信時期の違いで CTR と CPA は大きく変わる
- 実際に、同一アセット・同日・同LPでも `類似1%` と `ノンタゲ` の差だけで CPA が 7〜9倍ずれる例がある
- クリエイティブの優劣を判断するときは `asset -> 配信対象 -> LP -> 時期 -> 冒頭` の順で切る

### 14. Meta では `広告ID / 広告セットID` の差も配信面ブレとして扱う

- 同じクリエイティブ・同じオーディエンスでも、`広告ID` や `広告セットID` が変わると配信面が少しブレることがある
- 実運用では、同じクリエイティブを別の広告セットIDで複数本出して差分を見る前提で読む
- `同じCRなのに数字が違う` ときは、creative の善し悪しに直行せず `配信対象 / 広告ID / 広告セットID / LP / 時期` を順に切る

### 15. 広いフックでオーディエンスを広げすぎない

- ビジュアルフック、著名人、強い違和感などで `広く反応する入口` を作りすぎると、浅い反応層や低LTV層に学習が寄りやすい
- SNS広告では `CTRやフック率は悪くないのに、オプトイン以降が弱い` 形で表れやすい
- 上流が通るのに後ろで負けるときは、`広いフック -> 広いオーディエンス -> 質の弱い流入` を先に疑う

### 16. Meta広告CRの分析で `動画の中身` を飛ばさない

- 数値だけで `このCRは強い/弱い` と断定しない
- 特に `上流は通るが後ろで失敗` は、実際の冒頭文を見て `広い危機訴求 / 著名人 / 無料特典 / 汎用呼びかけ` が強すぎないかを確認する
- `同じクリエイティブでも数字が違う` ときは、数値比較の前提として動画の実内容が本当に同じか、冒頭文まで見て揃える

## 目安

- 大当たり動画のフック率は `25%前後以上` を基本ラインとして見る
- ただし二極化系のように、フック率が低くても LP 以降で深く刺さる例外はある
- 例外を認める条件は `CVR / CPO / 面談化` で裏が取れていること

## 次に見返す条件

- 同じルールが別ファネルや別案件でも再現したとき
- `CR -> LP -> メール -> CTA` のどこかでズレて失敗したとき
- short.io と URL 台帳の運用を自動化するタイミングが来たとき
