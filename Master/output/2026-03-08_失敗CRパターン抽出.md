# Output Review

- 日付: 2026-03-08
- 種別: 既存資産の整理
- 成果物名: 失敗 CR パターンの抽出
- 対象: Master の `rules / output` レイヤー整備
- 目的: 既存の失敗シグナルを knowledge のまま埋もれさせず、再発防止ルールに変える
- 仮説: 失敗側の条件を明文化すると、次回の CR / LP 判断で誤判定を減らせる
- 成果物リンク: `Master/rules/広告・導線ルール.md`
- 参照元: `Master/addness/meta_ads_cr_dashboard.md`, `Master/addness/dpro_research_20260304.md`
- 事実: `はずれ` や `許容KPI未達成` の CR が存在し、フック率 50% 超でも外しているケースが確認できる
- 観察: 数字が強く見えるフックでも、事業成果には直結していないケースがある。反面教材系の訴求は CPA が取れても顧客の質を崩しうる
- 解釈: 失敗判定では `フック率` や `刺さりそう感` のような局所指標を過信しない方が良い
- 確度: `高フック率でも外す` は確定 / 低LTV化リスクは強い推定
- 結果: `高フック率でも外す` `年齢+金額+秘密の直接訴求は低LTVを呼びやすい` `楽に系の直球は避ける` などの失敗ルールを追加
- 成功 / 失敗: 成功。勝ち筋だけでは見抜けない誤判定ポイントを `rules` に補強できた
- knowledge に戻す学び: 同じモチーフでも派生ごとに成績は割れる。特にフック率の強さは `勝ち` の代替指標にならない
- rules に戻すこと: 高フック率の過信を避けること、直接的な稼ぎ訴求を主軸にしないこと、同じ型でも派生単位で評価すること
- Skills に昇格できそうか: まだ早い。失敗判定の基準が別案件でも再利用できるかを見てから
- 次にやること: LP 側の失敗事例も追加し、`CRで外した理由` と `LPで外した理由` を分けて蓄積する

## Looker由来の失敗CR 具体事例（初回5本）

参照元: `Master/addness/meta_ads_cr_dashboard.md` の `2026/02/15〜02/28` スナップショット

### 1. 高橋海斗/尻込み＿ちえみさん/LP1-CR02203

- 事実: `はずれ / 許容KPI未達成 / 停止中 / CTR 4.0% / CPA ¥2,168 / フック率 54.5%`
- 観察: フック率も CTR も強く見えるのに総合判定は外れている
- 解釈: `入口反応の強さ` は勝ちの代替指標にならない
- 確度: 確定

### 2. 高橋海斗/尻込み_ちえみさん/LP1-CR02291

- 事実: `はずれ / 許容KPI達成 / 出稿中 / CTR 3.7% / CPA ¥2,265 / フック率 52.5%`
- 観察: 許容KPIを達成していても CR 判定では外れが存在する
- 解釈: `許容KPI達成 = 採用` と決め打ちすると、後ろの質を見落とす
- 確度: 確定

### 3. 甲原海人/Ver1/LP1-CR02259

- 事実: `はずれ / 許容KPI未達成 / 停止中 / CTR 0.7% / CPA ¥4,808 / フック率 34.7%`
- 観察: 高フックでも高CTRでもなく、入口段階から弱い
- 解釈: これは `後ろが悪い型` ではなく `そもそも入口で刺さっていない型`
- 確度: 強い推定

### 4. 鈴木織大/AI学ぶなwithヒカルさん/LP3-CR00013

- 事実: `はずれ / 許容KPI未達成 / 停止中 / CTR 1.2% / CPA ¥9,711 / フック率 30.2%`
- 観察: 消化額が約5万円に達しても集客は5件に留まっている
- 解釈: LP3 や訴求テーマ以前に、CR時点で十分に刺さらなかった可能性が高い
- 確度: 強い推定

### 5. 鈴木織大/未来の俺/LP1-CR02261

- 事実: `微妙 / 許容KPI未達成 / 停止中 / CTR 1.4% / CPA ¥4,204 / フック率 37.6%`
- 観察: フックは最低限あるが、CTR と CPA が弱く総合では止まっている
- 解釈: `少し気になる` までは作れても、クリックしたいほどの自分事化に届いていない
- 確度: 強い推定

## 今回見えた失敗の型

1. `高フック・高CTRでも総合では外れる型`
2. `許容KPI達成でも後ろの質で外れる型`
3. `入口から刺さっていない型`
4. `気になる止まりで自分事化に届いていない型`

## 今回の補足

- `尻込み` モチーフは同期間でも `目標KPI達成 / 許容KPI未達成 / はずれ` が混在している
- したがって、モチーフ単位で `勝ち / 負け` を決めず、派生1本ごとに評価する必要がある
- このレビューは Looker 数値の一次整理なので、`なぜ外れたか` の深掘りは LP と面談定性を重ねて行う
