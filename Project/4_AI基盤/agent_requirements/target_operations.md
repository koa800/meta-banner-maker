## Tier 1: 最優先（Meta広告の集客ボトルネックを潰す）

### C1: 広告CR企画
- **頻度**: 随時 / **所要時間**: 1-3時間/回
- **現状**: Lv.0
- **既存資産**: Skills/1_広告/CR企画.md、Meta広告大当たりCR分析データ
- **目標**: AIが企画案を複数生成→甲原が勝ち筋を選択（Lv.1）

### B5: 広告リサーチ
- **頻度**: 随時 / **所要時間**: 30-60分/回
- **現状**: Lv.0
- **既存資産**: DPro接続済み、Skills/1_広告/広告リサーチ.md
- **目標**: AIが競合CR収集→分析レポート生成（Lv.1）

### C5: CR効果検証・ABテスト方針
- **頻度**: 週次 / **所要時間**: 15-30分/回
- **現状**: Lv.0
- **既存資産**: KPIデータ（媒体×ファネル粒度）、Meta広告 Looker Studio
- **目標**: AIが勝ち負けを整理し、次の検証案まで提案（Lv.2）

### B1+B2: 広告数値の日次確認+パフォーマンス判断
- **頻度**: 毎日 / **所要時間**: 20-30分/日
- **現状**: Lv.0（手動）→ KPIキャッシュ・異常検知は既にLv.3
- **ギャップ**: Meta広告 Looker Studio は共有済みだが、クリエイティブ単位データの構造化共有がまだ弱い
- **既存資産**: kpi_summary.json、kpi_anomaly_detector.py、Looker Studio CSV
- **必要なもの**: Meta広告 Looker Studio を前提にした日次判断フローの整備、必要に応じてCR単位データ共有
- **目標**: AIが判断提案→甲原が承認のみ（Lv.2）

### B3: 広告数値レポート作成
- **頻度**: 週次 / **所要時間**: 30分/回
- **現状**: Lv.1
- **既存資産**: weekly_stats、経営会議資料自動作成
- **目標**: 完全自動生成+異常時のみ人間確認（Lv.3）

### F2+F3: 日次ゴール進捗確認+期限超過管理
- **頻度**: 毎日 / **所要時間**: 45-90分/日（44名分）
- **現状**: Lv.1（日向が一部カバー→停止中）
- **既存資産**: Addness API接続済み、hinata_agent.py、addness_to_context.py
- **目標**: AIがサマリ+異常検知→甲原が判断に集中（Lv.2）
