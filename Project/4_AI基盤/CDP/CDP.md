# CDP（顧客マスターデータ）

最終更新: 2026-03-05
ステータス: **設計完了・初期構築済み**

---

## 概要・目的

アドネスの顧客データが複数のGoogleスプレッドシートに散在している状態を解消し、**1つのマスターシートに統合**する。

**目的:**
- 導線分析（どの広告→どのセミナー→どの商品で購入したか）
- セグメント別ナーチャリング（流入経路・ファネル進捗・LTVでグルーピング）
- 顧客解像度の向上（定性情報の一元管理）

**CDPスプレッドシート:** [CDP](https://docs.google.com/spreadsheets/d/1qjU279OVD0i4h2AdQzkYIsZCfA1BeiUKLHNg7i2a2fk/edit)

## 金額表記ルール

- 通貨マーク: ¥を付ける（例: ¥800,980）
- 桁区切り: 3桁ごとにカンマ（例: ¥1,200,000）
- 対象カラム: 着金売上、返金額、LTV

---

## 登録・名寄せルール

- **登録条件**: 全員登録（メール、LINE、電話番号、いずれか1つでもあれば）
- **名寄せ（自動統合）**: メールアドレス一致 or 電話番号一致 → 統合
- **LINE名のみ**: 自動統合しない（重複リスクが高い）。後からメール/電話番号が判明したら統合
- **重複排除**: 複数ソースで同じメールアドレスがあれば既存レコードに統合（新規追加しない）

---

## データソース管理

各データソース = 1行。3グループ構成（11列）。

| グループ | カラム | 用途 |
|---------|--------|------|
| **ソース情報** | データソース名 / スプレッドシートID / タブ名 / ソース種別 | どのシートからデータを取るか |
| **ステータス** | 有効 / ソース最終更新日 / ステータス | 更新されているか（アクティブ/未使用）の検知 |
| **同期実績** | 最終同期日時 / 新規追加数 / 更新列数 / エラー数 | 同期のたびに自動記録 |

---

## マスター追加ルール

データソースごとに、どのカラムをどう取り込むかを定義する別タブ。

| カラム | 用途 |
|--------|------|
| データソース名 | どのソースのルールか |
| 対象カラム | CDPのどのカラムに入れるか |
| ソース側の列 | ソースのどの列から取るか |
| 追加条件 | マスターに追加する条件（例: 空でないこと） |
| 名寄せキー | email / phone / —（名寄せに使うかどうか） |
| メモ | 備考 |

## 容量試算・スケール戦略

- **現在**: 58列 x 10万人 = 580万セル（Google Sheets上限1,000万セルの58%）
- **対応可能**: 〜17万人程度まではシート分割不要
- **将来**: 17万人超 → アクティブ/アーカイブでシート分割
