## 概要・目的

アドネスの顧客データが複数のGoogleスプレッドシートに散在している状態を解消し、**1つのマスターシートに統合**する。

**目的:**
- 導線分析（どの広告→どのセミナー→どの商品で購入したか）
- セグメント別ナーチャリング（流入経路・ファネル進捗・LTVでグルーピング）
- 顧客解像度の向上（定性情報の一元管理）

**CDPスプレッドシート:** [CDP](https://docs.google.com/spreadsheets/d/1qjU279OVD0i4h2AdQzkYIsZCfA1BeiUKLHNg7i2a2fk/edit)

## カラム設計（全59列）

### システム管理（3列）
顧客ID / 作成日 / 最終更新日（※同期・インポート時に自動更新）

### ID情報（4列）
メールアドレス（名寄せ第1キー・商品購入時のメール） / メールアドレス2（UTAGEオプトイン時のメール。購入メールと異なる場合のみ記録） / 電話番号（名寄せ第2キー） / LINE名（参照用）

### 基本情報（13列）
姓 / 名 / フリガナ（姓） / フリガナ（名） / 性別 / 生年月日 / 年齢（※1） / 都道府県 / 市区町村 / 住所1 / 住所2 / 職業 / 年収

**※1 年齢の決定ルール:**
1. 生年月日がある → 今日時点の正確な年齢を自動計算（例: `35`）
2. 生年月日がない → アンケート回答の年代を使用（例: `30代`）

### 定性情報（3列）
現在の悩み / 理想の未来 / 過去の解決策

- `目標` は `理想の未来` に改名した
- U〜W列は行1で `定性情報` として結合し、W列とX列の間をグループ境界にする
- `収録URL` がある行で、上記3列のどれかが空のものは `interview_insights_sync.py` が自動補完する
- `収録URL` セルに複数URLが入っている場合は、対応している収録URLをすべて読み、1人分の定性情報として統合してから書き込む
- `過去の解決策` は `理想の未来` の次の列に固定する
- シートに入れる文は短く保ち、`現在の悩み` / `理想の未来` は 90文字以内、`過去の解決策` は 70文字以内を目安に圧縮する
- 面談全文や richer な抽出結果（要約 / 顧客タイプ / 根拠メモ）はシート列を増やしすぎず、ローカルキャッシュに保持する
- 複数URLのうち一部で取得失敗があっても、取れた面談だけで先に統合する。全URLで失敗したときだけその行を保留にする

### 流入情報（7列）
初回流入日 / 初回流入経路 / 最新流入日 / 最新流入経路 / 総流入経路数 / CR名 / CRリンク

### ファネル進捗（7列）
セミナー予約日（カンマ区切り） / セミナー予約合計 / 個別予約日（カンマ区切り） / 個別予約合計 / 個別着座日（カンマ区切り） / 担当営業 / 収録URL

### 商品購入（9列）
初回購入日 / 初回購入商品 / 最新購入日 / 最新購入商品 / 累計購入回数 / 購入商品 / 着金売上 / 返金額 / LTV

### スキルプラス利用状況（13列）
入会日 / プラン / サポートURL / 講義保管庫ID / 連続アクション日数 / アクション最終日 / 累計アクション日数 / ゴール達成数 / 初回イベント参加日 / 最終イベント参加日 / イベント参加回数 / クーリングオフ日 / 中途解約日

## データソース管理

CDPカラム単位のカラムマッピング形式。各カラムの参照先と運用状況を1枚で管理する。

**列構成（14列）:**

| 列 | 内容 |
|----|------|
| CDPカラム | 外部ソースから同期するカラム名（自動生成/自動計算/手動は含まない） |
| グループ | カラムのグループ名 |
| ソース元 | データソース（プルダウン選択） |
| 優先度 | 同一カラムに複数ソースがある場合の優先順位（1が最優先） |
| スプレッドシートURL | 参照先のURL |
| タブ名 | 参照先タブ |
| 参照先列 | 参照先の列名 |
| 正規化 | 取り込み時の変換ルール |
| 入力条件 | 参照データを顧客マスタに入力するための条件 |
| ステータス | 正常/未同期/停止（自動更新） |
| 最終同期日 | 同期スクリプトが自動記録 |
| 更新数 | ソーススプレッドシートの参照先列における実データ件数（同期時に自動カウント） |
| エラー数 | 同期時のエラー件数 |
| メモ | 備考 |

**複数ソース対応:** 同じCDPカラムに複数行を追加し、優先度で順序を指定。同期時は優先度1から順にチェックし、最初に値が見つかったものを採用する。

### CS管理シートの返金額同期

`返金額` は通常のメール1キー同期ではなく、CS管理シート専用ロジックで同期する。`データソース管理` には監視用に3行を持ち、実際の取り込みは `cdp_sync.py` が以下の条件で処理する。

| 優先度 | タブ | 返金額参照列 | 突合キー | 入力条件 |
|------|------|-------------|---------|---------|
| 1 | `管理用_クーオフ・中途解約以外` | P列 `返金額` | I列アドレス → H列LINE名 → G列本名 | F列=`完了` |
| 2 | `管理用_2025.1.25-クーオフ` | T列 `返金額` | K列アドレス → J列LINE名 → I列本名 | X列 `返金予定日` が入っている行のみ |
| 3 | `管理用_20250125-中途解約` | X列 `返金額/請求額` の**赤文字のみ** | K列アドレス → J列LINE名 → I列本名 | AB列 `返金日` が入っている行のみ |

- 突合優先順位は `メールアドレス > LINE名 > 本名`
- `LINE名` fallback は短すぎるキー（2文字以下、数字のみ）を除外して誤突合を防ぐ
- `中途解約` タブは Google Sheets API で書式を読み、赤文字セルのみ返金額として採用する
- `返金額` 更新時は `LTV = 着金売上 - 返金額` も同時に再計算する

**同期対象外のカラム（Python自動計算 or システム管理のため、データソース管理に含まない）:**
顧客ID / 作成日 / 最終更新日 / 総流入経路数 / セミナー予約合計 / 個別予約合計 / 累計購入回数 / 購入商品 / LTV

**集計カラムの自動計算（ARRAYFORMULAからPython計算に移行済み）:**
Google SheetsのARRAYFORMULA計算限界（約53,000行）を超えたため、cdp_sync.pyが直接値を書き込む方式に変更。
- セミナー予約合計 = セミナー予約日のカンマ区切り数
- 個別予約合計 = 個別予約日のカンマ区切り数
- 累計購入回数 = 購入商品のカンマ区切り数
- LTV = 着金売上 - 返金額
- 同期時にソースカラム更新と同時に自動計算。一括再計算は `backfill-computed` コマンド

**ステータス判定ルール（自動更新）:**
- 正常: 直近の同期が成功した（同期成功時に自動セット）
- 未同期: まだ一度も同期されていない（新規登録時にセット）
- 停止: 直近の同期でエラーが発生した（同期エラー時に自動セット）

**鮮度チェック（ステータスとは別の監視レイヤー）:**
- 7日以上未同期のソースを `check_source_staleness()` で検出
- Orchestrator経由でLINE通知（ステータスは変更しない）
- 確認が必要な通知では、`CDP / データソース管理` のリンクと、該当ソースのリンクを付ける
- 正常終了時の通知は更新件数だけを短く出し、ユーザーが介入できない内部処理の成功は通知しない

**ソース元（プルダウン選択肢）:**
Lステップ / UTAGE / Mailchimp / Univapay / MOSH / Invoy / 決済システム / 銀行振込 / 講義保管庫 / スプレッドシート

## 登録・名寄せルール

- **登録条件**: メールアドレスまたは電話番号が必須（LINE名のみでは登録しない）。さらにイベント発生（アンケート回答・セミナー予約・購入等）が必要。LINE登録のみ・メール登録のみの人は集客データシートに格納
- **名寄せ（自動統合）**: メールアドレス一致（メールアドレス2も含む） or 電話番号一致 → 統合
- **LINE名のみ**: 自動統合しない（重複リスクが高い）。後からメール/電話番号が判明したら統合
- **重複排除**: 複数ソースで同じメールアドレスがあれば既存レコードに統合（新規追加しない）
- **除外リスト**: CDPマスタの除外リストタブが Single Source of Truth。メール・電話番号のいずれかに一致したら除外。集客データシートの同期でも同じ除外リストを参照。LINE名は重複が多いため除外キーに使わない
- **スパム自動検知**: 除外リストに加え、同期時にパターンマッチでテスト/いたずらメールを自動除外（`is_spam_email()`）
  - test系: `test@`, `test123@` 等
  - 同一文字繰り返し: `aaaa@gmail.com` 等（フリーメールのみ）
  - テストドメイン: `example.com`, `test.com` 等
- **更新方式**: 名寄せで既存行を更新。新規顧客のみ最下部に追加
- **複数ソース**: 同一カラムに複数ソースがある場合、優先度順に値を探し最初のヒットを採用

### 同期の競合ルール

| 状況 | ルール |
|------|--------|
| 新規顧客 | 優先度1のソースから値を取得。なければ優先度2を確認 |
| 既存顧客（値が空） | 優先度順にソースを確認し、最初に見つかった値を記録 |
| 既存顧客（値が入っている） | **ソース側の値で上書き**（ソースが最新と見なす）。変更前後の値をログに記録 |
| カンマ区切り追記カラム | セミナー予約日・個別予約日・個別着座日・購入商品は上書きではなく追記（重複スキップ） |

## データ保護

### 電話番号の保護
- 顧客マスタの電話番号列は **Plain Text形式** に設定済み
- Googleスプレッドシートが数値に自動変換して桁が壊れるのを防止
- 同期時にバリデーション実行（科学的記数法検出・桁数チェック）

### メールアドレスの重複検知
- 新規メール登録時にレーベンシュタイン距離で類似メールを検出（閾値: 距離2以下）
- 同一ドメインのメールのみ比較（パフォーマンス最適化）
- 警告ログを `System/data/cdp_email_warnings.json` に記録

### 入力バリデーション（同期・インポート時に自動適用）
- **メールアドレス**: RFC準拠の形式チェック（`is_valid_email()`）
- **電話番号**: 10〜11桁チェック・科学的記数法検出（`validate_phone()`）
- **年齢**: 数字1〜3桁 + optional 代/歳 のみ許可（`is_valid_age()`）
- **URL系（収録URL / サポートURL / CRリンク）**: http/https で始まる形式のみ許可（`is_valid_url()`）

### 変更ログ
- 同期時に全変更の before/after を `System/data/cdp_change_log.json` に記録
- タイムスタンプ・行キー・カラム・旧値・新値・ソースを保持
- 最大10,000件を保持（古い方から自動削除）

4. メールアドレスがない人はCDPマスタと突合できず、集客データシートの役割（転換率分析）に乗らない
→ LINE登録者の管理・アプローチ・計測はすべてLステップで完結。シートに持つ必要なし

## 同期の仕組み

### 自動同期フロー

``` 
データソース管理タブ（マッピング）
  → cdp_sync.py がマッピングを読み取り
    → ソースごとにシートを読み込み
      → 除外チェック → 名寄せ → バリデーション → 書き込み
        → ステータス・同期日・更新数をデータソース管理に記録
```

- `顧客マスタ` / `データソース管理` / `除外リスト` は同期後に罫線を最終行まで自動補正する

### 排他制御

- 同期開始時にロックファイル（`System/data/cdp_sync.lock`）を取得
- ロック中に別の同期が実行されるとブロック（「同期がロック中です」エラー）
- 10分経過でロック自動解除（デッドロック防止）
- 同期完了 or エラー時にロック自動解除

### 増分同期

- ソースごとの最終同期状態を `System/data/cdp_sync_state.json` で追跡
- 前回同期時の行数を記録し、差分（新しい行）のみ処理
- `--full` オプションで全件同期も可能

### 面談定性の自動補完

`収録URL` が入っている顧客について、AILEAD の面談収録から定性情報を抽出し、`現在の悩み` / `理想の未来` / `過去の解決策` を埋める。

```text
顧客マスタの収録URL
  → interview_insights_sync.py が AILEAD GraphQL から transcript を取得
    → LLM で顧客の悩み・理想の未来・過去の解決策を要約
      → 空欄のときだけ CDP の既存列へ書き戻し
        → 詳細はローカルキャッシュに保存
```

- 既存列に値がある場合は上書きしない（`--force` で上書き可能）
- まず Anthropic API を試し、クレジット不足やAPI失敗時は Claude CLI に自動フォールバックする
- `confidence=low` の結果はシートへ書き込まず、キャッシュだけ残す
- 定性列は増やしすぎず、顧客理解と広告/LP/営業分析に直結する3項目だけに絞る
- キャッシュ: `System/data/interview_insights_cache/<callResultId>.json`
- 状態管理: `System/data/interview_insights_state.json`
- Orchestrator: `interview_insights_sync` が **毎時50分** に最大20件を実行
- backlog 解消: `interview_insights_backfill` が **毎日01:20** に最大240件をまとめて処理
- 比較分析: `interview_insights_analysis` が **毎日05:20** に `System/data/interview_insights_analysis.json` を再生成
- 成功通知は送らず、止まったときだけ `CDP / 顧客マスタ` のリンク付きで通知する

### ソース変更検知

同期前にソースの構造変化を自動検出し、知らない間に違うデータを取り込むことを防ぐ。

| 検知パターン | 動作 |
|-------------|------|
| マッピング対象の列名が消失 | まず列名揺れを自動修復（記号差分・表記ゆれを吸収）。直せない場合のみ **同期を中断** |
| 列の追加・削除（マッピング外） | アラート表示して続行 |
| 行数が50%以上急減（100行以上のソース） | アラート表示して続行 |

- ヘッダーのスナップショットを `cdp_sync_state.json` に保存し、次回同期時に比較
- 初回同期時はスナップショットがないため検知なし（2回目以降から有効）
- 自動修復に成功した列名は `データソース管理` に書き戻し、次回以降も新しい列名を正として使う

### バッチ書き込み

- 既存行の更新: `batch_update()` で500件ずつチャンク分割
- 新規行の追加: `update()` で500行ずつ範囲指定書き込み（`append_rows()`はグリッド膨張リスクあり不使用）
- グリッドが足りない場合は自動拡張
- カンマ区切りカラム（セミナー予約日、個別予約日、個別着座日）は上書きではなく追記（重複チェック付き）

### CLIコマンド

```bash
python3 cdp_sync.py sync              # 全ソース自動同期（増分）
python3 cdp_sync.py sync --dry-run    # ドライラン
python3 cdp_sync.py sync --full       # 全件同期
python3 cdp_sync.py sync-leads         # 経路別タブ→集客データ同期のみ
python3 cdp_sync.py check-phones      # 電話番号バリデーション
python3 cdp_sync.py backfill-computed  # 集計カラム（合計・LTV）一括再計算
python3 cdp_sync.py status            # ステータス確認
python3 cdp_sync.py exclusion-list    # 除外リスト表示
python3 interview_insights_sync.py ensure-schema                  # `理想の未来` + 定性情報レイアウトを整える
python3 interview_insights_sync.py bootstrap-session             # AILEAD セッションを保存
python3 interview_insights_sync.py status                        # 面談定性補完の進捗確認（row_status_counts で updated / partial_updated / low_confidence / error を確認）
python3 interview_insights_sync.py --limit 20 sync              # 収録URL付き行を最大20件処理
python3 interview_insights_sync.py --limit 60 backfill          # backlog を複数バッチで前進させる（夜間は最大600件目安）
python3 interview_insights_sync.py --dry-run sync               # 書き込まず更新候補だけ確認
python3 interview_insights_sync.py --row 56428 sync             # 特定行だけ処理
python3 interview_insights_sync.py fetch "<収録URL>"            # AILEAD / YouTube / Loom を 1件以上抽出してJSON確認
python3 interview_insights_sync.py analyze                      # LTV別の面談定性比較を JSON + Master/knowledge Markdown に更新
```
