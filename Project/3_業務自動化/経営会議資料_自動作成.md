# 経営会議資料 自動作成プロジェクト

最終更新: 2026-03-07
ステータス: v1実装完了（自動実行可能）

---

## 概要

毎週金曜の経営会議に向けて、広告チーム報告資料をAIが自動生成する。
甲原さんは最終確認と補足のみで済む状態を目指す。

---

## 資料の目的と設計思想

### 誰に、何を伝えるか

- **対象**: 経営陣（みかみさん等）
- **時間**: 約3分間の口頭発表
- **前提**: 経営陣は事前に読まない。発表中も熟読しない。「言って伝わる、見て伝わる」レベルが必須
- **核心**: 「何がボトルネックで、そのために何をするか」の自己宣言（承認を求める場ではない）

### 構成のロジック（順序に意図あり）

```
1. 総評 + NEWS     → 結論を一撃で伝える（良かったか悪かったか）
2. 月の目標        → 自分たちはここを目指している
3. 現状（月次KPI）  → 今ここにいる
4. 変化（12週推移） → 傾向が上向きか下向きか
5. 勢い（7日間KPI） → 直近の勢いは良いか悪いか
6. プロジェクト進捗  → 目標と現状のギャップを埋めるためにやっていること
7. ボトルネック     → 今一番解決すべき課題
8. 改善アクション   → 具体的に何をするか
9. その他共有事項   → 補足
```

話のロジックとして「結論→目標→現状→アクション」が綺麗に流れる構成。
認知負荷をできるだけ下げながら、かつ伝わりやすくすることを意識。

## 各セクション詳細

### 1. タイトル
- フォーマット: `2026/X/X　アドネス経営会議`
- 中央揃え、太字、下線、フォントサイズ20
- 完全自動（日付計算）

### 2. 【総評】

**5段階評価 + 端的なコメント（1〜2文）**

```
【総評】
3/4
2/5
└CPA改善傾向だが着金売上が目標ペースに届いていない。
```

- 1行目: 数値の基準日（会議2日前）
- 2行目: X/5 の評価
- 3行目: 端的な状況説明

#### 5段階評価の基準

| 評価 | 意味 | 判定基準 |
|---|---|---|
| 5 | 最高 | 目標を大幅に上回るペース |
| 4 | やや良い | 目標達成できるスピードで進んでいる |
| 3 | 普通 | ギリギリいけるかどうか |
| 2 | やや悪い | ちょっといけなさそう |
| 1 | 全然ダメ | 目標に全然届かない |

**評価の考え方:**
- 「月の目標に対して達成できるペースか」が最大の軸
- 価値をちゃんと作っている / 目標に向かう速度が速い → 高評価
- 速度が遅い / 見通しが足りない → 低評価
- プロジェクトの進捗も加味した包括的評価
- ROASだけ高くても予算を下げただけなら本質的ではない（複合的に判断）

### 3. <NEWS！>

- 良い成果物や報告が上がってきたものを記載
- 「みんなに伝えたらエネルギーが上がるか」が選定基準
- 情報源: グループチャット、Slack、甲原さんへの定期確認
- チェックマーク付き箇条書き

### 4. 【X月目標と現状】

テーブル形式:

| 行 | 内容 |
|---|---|
| X月の目標 | ①着金売上:X億円（X億円/週）②集客数:X万人（X万人/週） |
| X月の進捗状況(X/1〜X/X) | Looker Studioスクショ + コメント |
| X月着地予想 | 粗利:X円、着金売上:X円、集客数:X人 |

- 目標値は前回資料から自動引き継ぎ（変更は甲原さんが指示）
- 進捗コメント: 「目標に対して、集客数△。」のように端的に
- 着地予想: 単純な日割り計算 + 甲原さんの補足（上振れ/下振れ要因）

### 5. 過去12週間推移

- Looker Studioスクショ（集客数・個別予約数・広告費・CPA の4グラフ）
- コメント: 傾向の変化点に注目。「傾きが変わったタイミング」が重要
- 経営陣に「上向きか下向きか」を一目で伝える役割

### 6. 過去7日間推移

- Looker StudioスクショKPIカード
- 「今が良いかどうか」= 直近の勢いを示す
- コメント: 前週比で特筆すべき変化を記載

### 7. 【プロジェクト進捗】

テーブル形式:

| 主なプロジェクト | 評価 | 詳細情報 |
|---|---|---|
| ①Web広告から集客数4万人 | 色付き評価 | 原因+状況 |
| ②サブスク導線で単月ROAS100% | 色付き評価 | 原因+状況 |
| ③UGC口コミ発生プロジェクト | 色付き評価 | 原因+状況 |

- 主に3つ（本当は1〜2が理想だが3つ挙げている）
- プロジェクトは頻繁に変わらない。変更時は甲原さんに確認
- 評価は数値から推定。判断できなければ甲原さんに確認

#### 評価の色分け（5段階）

| 評価 | 色 | 意味 |
|---|---|---|
| 悪い | 赤 | 全然進んでいない |
| やや悪い | オレンジ | 遅れている |
| 普通 | — | 可もなく不可もなく |
| やや良い | 黄緑 | 順調 |
| 良い | 緑 | 非常に順調 |

- 目標に対してちゃんと進んでいるかが基準
- 詳細情報には原因まで書く（何が進んでいる/止まっているか明確に）

### 8. 【ボトルネックと改善アクション】

テーブル形式:

| ボトルネック | 改善アクション |
|---|---|
| 具体的な課題 | 具体的なアクション（期限+成果物） |

- 毎週変わりうる（同じものが続く場合はアクションが不十分だった）
- 「自分たちでこうします」という宣言の場
- アクションは具体的であるほど良い（期限+成果物レベル）
- 経営陣に助けを求める場ではない

### 9. 【その他・共有事項】

- 箇条書き
- 週の活動から共有すべき事項を抽出

## KPI基準値

### 優先度（経営陣が気にする順）

1. **着金売上** — 最重要
2. **ROAS** — 300%=まあOK、350%=結構良い
3. **集客数** — 目標の万人/週ペースか
4. **CPA** — 3,000円以下=良い、4,000円以上=悪い

### 注目すべき変化の判定

- 単純な%変動ではなく「過去の傾向（傾き）」を見る
- 傾きが変わったタイミング = 状況に変化があった
- 広告費を増やせば集客数は増えるので、単純な数値増減だけでは判断しない

## 実行フロー

```
金曜 15:00 自動起動（Orchestrator → 秘書）
  |
  v
[1] データ収集
  - Looker Studio にアクセス（Chrome MCP）
  - 月次KPIスクショ取得（期間フィルタ変更→スクショ）
  - 過去7日間スクショ取得（期間フィルタ変更→スクショ）
  - 過去12週グラフスクショ取得
  - スクショからAI Visionで数値読み取り
  |
  v
[2] 前回資料の読み取り
  - Google Docから前回の目標値を取得
  - プロジェクト一覧を取得
  |
  v
[3] AI起草
  - 着地予想: 日割り計算
  - 総評: 5段階評価 + コメント
  - NEWS: 週の活動から抽出
  - 12週コメント: 傾きの変化に注目
  - 7日間コメント: 前週比の特筆事項
  - プロジェクト評価: 数値から推定
  - ボトルネック: 数値悪化ポイントから特定
  - 改善アクション: 具体的なアクション提案
  |
  v
[4] LINE確認
  「経営会議資料の下書きができました → [Google Docリンク]
   総評: X/5 — 〇〇〇
   NEWS: 〇〇〇
   補足・修正があれば返信してください」
  |
  v
[5] 甲原さんの返信を反映
  - 修正内容を理解してGoogle Docsを直接編集
  - 「修正しました → [リンク]」と報告
  |
  v
[6] 最終出力
  - Google Docs API でドキュメント先頭に新ページ挿入
  - スクショ画像 + テキスト + テーブルを構造化挿入
```

### Lookerフィルター操作の技術詳細

Looker StudioのAngularフォームはform_inputでは値変更が反映されない。
JavaScriptのnativeInputValueSetterパターンで直接DOM値を変更し、input/changeイベントを発火させる:

```javascript
const setter = Object.getOwnPropertyDescriptor(window.HTMLInputElement.prototype, 'value').set;
setter.call(input, '新しい値');
input.dispatchEvent(new Event('input', { bubbles: true }));
input.dispatchEvent(new Event('change', { bubbles: true }));
```

### スクショ座標計算

MCP screenshot → macOS screencapture の座標変換:
```
MCP座標 × (1890/1552) × 1.6 = Retina座標
Y座標 += 422（Chrome UIオフセット: タブバー+アドレスバー+ブックマークバー）
```
