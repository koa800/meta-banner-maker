## 概要・目的

CDPからセグメントを切り出し、AIがメール本文をライティングし、Mailchimp APIで配信設定・送信まで自動完結するフローを構築する。

**目的:**
- セグメント別のメールマーケティングを自動化
- AIライティング → 承認 → 送信のワークフロー確立
- 将来的に承認率が安定したら全自動化

## 技術的実現可能性: 結論

**全て実現可能。** Mailchimp Marketing APIで以下が全てプログラマブルに操作できる:

| やりたいこと | API | 制約 |
|-------------|-----|------|
| CDPコンタクトの一括同期 | `POST /lists/{id}/members` (バッチ対応) | 500件/リクエスト、同時接続10 |
| タグの付与・削除 | `POST /lists/{id}/members/{hash}/tags` | — |
| セグメント作成 | `POST /lists/{id}/segments` | タグ条件でフィルタ可能 |
| キャンペーン作成 | `POST /campaigns` | type: regular, recipients で対象指定 |
| メール本文の設定 | `PUT /campaigns/{id}/content` | HTML or プレーンテキスト |
| 配信スケジュール | `POST /campaigns/{id}/actions/schedule` | UTC指定 |
| 即時送信 | `POST /campaigns/{id}/actions/send` | 取消不可 |
| 配信結果の取得 | `GET /reports/campaigns/{id}` | 開封・クリック・バウンス等 |

**公式Pythonライブラリ:** `mailchimp-marketing`（pip install mailchimp-marketing）

**API制限:**
- 同時接続: 10
- バッチ: 500件/リクエスト（大量同期はバッチエンドポイント使用）
- レート: 429エラー時はリトライ

## 設計

### 全体フロー

```
CDP（Google Sheets: 56,538人）
  ↓ cdp_mailchimp.py sync
Mailchimpオーディエンス同期（メール・名前・タグ）
  ↓ cdp_mailchimp.py segment "セミナー予約 & 未購入"
CDPでフィルタ → 該当者に cdp_ タグ付与
  ↓ cdp_mailchimp.py draft --segment cdp_seminar_no_purchase --subject "..."
AIがメール本文をライティング → Mailchimpキャンペーン下書き作成
  ↓ Slack/LINEで承認依頼
甲原さんが確認 → 承認
  ↓ cdp_mailchimp.py send {campaign_id}
Mailchimp APIで送信
  ↓ 配信後
結果レポートを自動取得 → Slack通知
```

### セグメント設計

CDPの全58列を使って、Mailchimp単体では不可能な高精度セグメントが切れる:

| セグメント例 | CDP条件 |
|-------------|---------|
| セミナー参加 → 未購入 | セミナー予約日 あり & 累計購入回数 = 0 |
| LTV上位（リピーター） | LTV > ¥500,000 |
| Meta広告経由 × フリープラン未購入 | 初回流入経路 contains "Meta" & 購入商品 not contains "フリープラン" |
| 休眠顧客（最終購入90日以上） | 最新購入日 < 90日前 |
| スキルプラス会員 × アクション停止 | 入会日 あり & アクション最終日 < 30日前 |

### タグ命名規則

```
cdp_{セグメント名}
例: cdp_seminar_no_purchase, cdp_ltv_high, cdp_dormant_90d
```

既存タグ（UTAGE_*, freeplan_* 等）とは `cdp_` プレフィックスで明確に分離。

### Journey と Campaign の使い分け

- `Journey`
  - 今後も同じシナリオを流すと決まっている時に使う
  - 役割は evergreen 用の自動化
  - 例: オプトイン後教育、個別3日間、購入後一定シナリオ
- `Campaign`
  - 1回きりのメール配信で、きちんと segment を切って送る時に使う
  - 役割は promotion 用の単発配信
  - 例: 期間限定プロモーション、特定条件の一斉案内
- current で確認できた representative campaign
  - `3/11 AIキャリアセミナーPR_2通目`
    - `Subscribed` 全体へ 255,250 通
  - `3/10 セミナーPR_3通目 (フリープラン)`
    - `freeplan_Buy` tagged contacts に 5,044 通
  - つまり current の campaign は `今この条件に当てはまる人だけへ単発で打つ` 運用になっている

つまり Mailchimp は
- `同じ顧客体験を繰り返し自動で流す = Journey`
- `今この相手にだけ1回送る = Campaign`
で切るのが current の正しい理解

### Addness current tag rule

- Mailchimp の tag name は英語で作る
- 基本構造は `media_funnel_event`
- 例
  - `meta_ai_optin`
  - `x_skillplus_webinar1_optin`
  - `linead_sns_consult_reserve`

理由:
- Mailchimp では日本語 tag が消えたり崩れたりしやすい
- 媒体 / ファネル / イベントを英語で固定した方が運用事故が少ない

### AIライティング

- モデル: Claude（Anthropic API）
- インプット: セグメント情報 + 過去の配信実績（開封率・クリック率） + ブランドトーン
- アウトプット: 件名 + プレヘッダー + HTML本文
- テンプレート: 既存のMailchimpテンプレートをベースに本文部分のみAI生成

### 承認フロー

```
Phase 1（初期）: AI下書き → Slack通知 → 甲原承認 → 送信
Phase 2（安定後）: AI下書き → 自動送信（承認率が安定したら移行）
```

### Mailchimp ↔ CDP 双方向同期

Mailchimp側にしかないデータ（開封履歴・クリック履歴・バウンス）をCDPに還流:
- 配信後にレポートAPIで結果取得
- CDPに「メール開封率」「最終クリック日」等のカラムを追加（将来）
- これによりCDP側でさらに精度の高いセグメントが切れる

### 顧客行動データとして追う指標

Mailchimp 単体で閉じず、導線全体では次の 5 点を 1 本で追う必要がある。

1. メール送信数
2. 開封数 / 開封率
3. リンククリック数 / クリック率
4. 遷移先 UTAGE ページの訪問者数
5. 遷移先ページでのコンバージョン数

この 5 点をつなげて初めて、どこで離脱しているかを判断できる。

### 動線データ基盤の目的

CDP にそのまま入れず、別の `動線データ` として持つ前提で考える。  
第一目的は `分析用の可視化` と `マーケティング施策立案のためのデータ収集`。

スキルプラス事業でまず見るべき主要変数は次の 4 つ。

1. CPA
2. 個別相談数
3. 営業成約率
4. 解約率

この 4 変数を改善するために、`送信 -> 開封 -> クリック -> 訪問 -> 行動` の導線データを別レイヤーで持つ。

### current のコンテンツ構造と制作者の意図

Mailchimp の current メールは、単に件名を量産しているのではなく、かなり明確な役割分担で作られている。

#### 1. 本線 7桁オプトインの役割

- 代表例
  - `AIカレッジFB広告_メインシナリオ1通目 (copy 05)`
  - `SNS_Meta広告_7桁_1通目`
- 共通する意図
  - いきなり売らず、まず `みかみ` の人物ストーリーと authority を入れる
  - 「第1話 / 第2話 ...」で open loop を作り、連続閲覧を促す
  - 広告で感じた期待を `自分にもできそう` に変える
- current で見えている CTA
  - AI 本線 1通目は `https://skill.addness.co.jp/meta-ai2`
  - SNS 本線 1通目は `https://skill.addness.co.jp/metaad-sns2`
- 本文で実際にやっていること
  - AI 本線 1通目は `東大生社長みかみがAI未経験から年収2億ベースで稼ぐまでの軌跡` を主題にして、人物ストーリーと実績で期待を維持している
  - SNS 本線 1通目は `元東大生社長みかみが年商2億稼ぐまでの軌跡` を主題にして、動画やストーリーを複数差し込みながら `もっと見たい` 状態を作っている
- 良いと判断できる点
  - subject が `ストーリー / 権威 / 続き` を担っている
  - 本文は長文でも、役割が `売る前の期待維持` なので整合している

#### 2. 途中の直オファー系メールの役割

- 代表例
  - `AIカレッジFB広告_AI導入プロコンサル (copy 02)`
  - `センサーズFB広告_本気コンサルオファー (copy 05)`
- 共通する意図
  - 本線で温めた読者へ、急に距離を縮める
  - `緊急告知 / 直接コンサル` のラフさで、堅い売り込み感を下げる
  - 個別相談やコンサルへの遷移を強く取りにいく
- current で見えている CTA
  - AI 側は `https://skill.addness.co.jp/meta-ai4`
  - SNS 側は `https://skill.addness.co.jp/metaad-sns5`
- 本文で実際にやっていること
  - AI 側は `【緊急告知！！】直接コンサルしましょうか？笑` というラフな件名で入り、個別に近い距離感を先に作ってから CTA へ寄せている
  - SNS 側の `本気コンサルRe` は `間も無く締め切ります` で urgency を前面に出し、迷っている人を押し込む役割が強い
- 良いと判断できる点
  - 件名と本文のトーンが `思いつき感 / 個別感` に寄っていて、直売りメール特有の警戒を下げている
  - 本線メールと比べて CTA がかなり明確

#### 3. スキルプラス横展開メールの役割

- 代表例
  - `スキルプラスフリープラン1通目 (copy 03)`
  - `スキルプラスフリープラン1通目 (copy 02)`
- 共通する意図
  - AI / SNS で集めた人に対し、別の価値提供口としてスキルプラスを差し込む
  - `無料 / 期間限定 / 挑戦しやすさ` を前面に出し、より広い層へ口を広げる
- current で見えている CTA
  - AI 側は `https://school.addness.co.jp/p/TzYRwepfqzFq?...`
  - SNS 側は `https://school.addness.co.jp/p/hNuqvGx9VJgZ?...`
- 本文で実際にやっていること
  - SNS 側 2通目は `たった10時間で1,300名が押し寄せた...無料のアレの正体とは？` という件名で、無料性と social proof を使って参加ハードルを下げている
  - AI / SNS ともに、高単価の直接訴求ではなく `無料で入れる別の価値提供口` として差し込んでいる
- 良いと判断できる点
  - 本線のストーリーで温度を上げた後に、低いハードルの offer を置いている
  - いきなり高単価へ行かず、無料体験やフリープランを挟んでいる

#### 4. 個別3日間シナリオの役割

- 代表例
  - `AIカレッジFB広告_個別3日間シナリオ_1日目 (copy 02)`
  - `センサーズFB広告_個別3日間シナリオ_1通目ABテストA`
- 共通する意図
  - `あなた専用` という言葉で、自分ごと化を一段上げる
  - 3日間で urgency と social proof を積み、面談や作成会へ押し込む
  - 1日目はプレゼント、2日目は反響、3日目は締切で詰める
- current で見えている CTA
  - AI 側は `school.addness.co.jp/p/uUfErKGsQYZ2?...`
  - SNS 側は `school.addness.co.jp/page/KAtTy0tucas2?...`
- 本文で実際にやっていること
  - SNS 1通目 ABテストB は `【今だけ無料！】"あなた専用" SNS運用 1st副業スタートプログラム開催！！` と promise を先に立て、自分ごと化を最短で取りにいく
  - AI 側の current では `💬Google創業者に聞いた、僕らがGAFAを超える方法` のように、大きい夢や思想を使って `みかみ側の世界観に共感した人だけを濃くする` 役割を持つものもある
- 良いと判断できる点
  - `件名 -> 冒頭コピー -> CTA` が同じ圧力でつながっている
  - AB テストでは promise を変えつつ、構造は大きく崩していない

#### Mailchimp コンテンツ全体の理解

- Mailchimp は `長文で教育しながら温度を上げる` 本線
- 件名は `物語 / 権威 / 危機感 / 緊急性 / 個別感` のどれかを強く担う
- 本文は長くてもよく、その代わり `1通ごとの役割` がかなり明確
- Lステップのような短い operational message ではなく、`教育と心理変容` が主目的
- current の良し悪しは、文章量ではなく `この1通で相手の心理をどこまで進めるか` が明確かどうかで判断する
