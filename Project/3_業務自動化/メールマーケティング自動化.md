## 概要・目的

CDPからセグメントを切り出し、AIがメール本文をライティングし、Mailchimp APIで配信設定・送信まで自動完結するフローを構築する。

**目的:**
- セグメント別のメールマーケティングを自動化
- AIライティング → 承認 → 送信のワークフロー確立
- 将来的に承認率が安定したら全自動化

## 技術的実現可能性: 結論

**全て実現可能。** Mailchimp Marketing APIで以下が全てプログラマブルに操作できる:

| やりたいこと | API | 制約 |
|-------------|-----|------|
| CDPコンタクトの一括同期 | `POST /lists/{id}/members` (バッチ対応) | 500件/リクエスト、同時接続10 |
| タグの付与・削除 | `POST /lists/{id}/members/{hash}/tags` | — |
| セグメント作成 | `POST /lists/{id}/segments` | タグ条件でフィルタ可能 |
| キャンペーン作成 | `POST /campaigns` | type: regular, recipients で対象指定 |
| メール本文の設定 | `PUT /campaigns/{id}/content` | HTML or プレーンテキスト |
| 配信スケジュール | `POST /campaigns/{id}/actions/schedule` | UTC指定 |
| 即時送信 | `POST /campaigns/{id}/actions/send` | 取消不可 |
| 配信結果の取得 | `GET /reports/campaigns/{id}` | 開封・クリック・バウンス等 |

**公式Pythonライブラリ:** `mailchimp-marketing`（pip install mailchimp-marketing）

**API制限:**
- 同時接続: 10
- バッチ: 500件/リクエスト（大量同期はバッチエンドポイント使用）
- レート: 429エラー時はリトライ

## 設計

### 全体フロー

```
CDP（Google Sheets: 56,538人）
  ↓ cdp_mailchimp.py sync
Mailchimpオーディエンス同期（メール・名前・タグ）
  ↓ cdp_mailchimp.py segment "セミナー予約 & 未購入"
CDPでフィルタ → 該当者に cdp_ タグ付与
  ↓ cdp_mailchimp.py draft --segment cdp_seminar_no_purchase --subject "..."
AIがメール本文をライティング → Mailchimpキャンペーン下書き作成
  ↓ Slack/LINEで承認依頼
甲原さんが確認 → 承認
  ↓ cdp_mailchimp.py send {campaign_id}
Mailchimp APIで送信
  ↓ 配信後
結果レポートを自動取得 → Slack通知
```

### Journey と Campaign の使い分け

- `Journey`
  - 今後も同じシナリオを流すと決まっている時に使う
  - 役割は evergreen 用の自動化
  - 例: オプトイン後教育、個別3日間、購入後一定シナリオ
- `Campaign`
  - 1回きりのメール配信で、きちんと segment を切って送る時に使う
  - 役割は promotion 用の単発配信
  - 例: 期間限定プロモーション、特定条件の一斉案内
- current で確認できた representative campaign
  - `3/11 AIキャリアセミナーPR_2通目`
    - `Subscribed` 全体へ 255,250 通
  - `3/10 セミナーPR_3通目 (フリープラン)`
    - `freeplan_Buy` tagged contacts に 5,044 通
  - つまり current の campaign は `今この条件に当てはまる人だけへ単発で打つ` 運用になっている

### Addness current CTA rule

- Mailchimp で今後新しく作るメールの `main CTA` は、Journey / Campaign を問わず `short.io` を正とする
- 理由
  - BAN やリンク失効時に設置箇所を触らず差し替えられる
  - URL管理シートで正本管理できる
  - クリック分析を揃えやすい
  - direct LIFF の直貼り事故を減らせる
- `direct LIFF` は例外
  - 一時的なテスト
  - 失効対策も差し替えも分析も不要な特殊ケース
- したがって current に direct LIFF の campaign が残っていても、それは `現状の実装` であり、`今後の標準` ではない

### コンテンツを読む時の基準

- Mailchimp のメールは、単に情報を伝えるものではない
- `コミュニケーションの手段であり、ユーザーをこちらが意図した認識に変換するもの`
  として読む
- Addness の current では、特に次の 3 点で良し悪しを判断する
  - 何の認識を変えたいかが 1 通ごとに明確か
  - 次にしてほしい行動が 1 つに絞られているか
  - 前の接点から promise が切れず、次の接点へ滑らかにつながるか

### 良い / 悪いの判断

#### 良い

- 1 通の役割が明確
  - 例
    - 世界観理解
    - authority 補強
    - urgency 付与
    - 予約促進
- 件名、本文、CTA が同じ方向を向いている
- segment の温度に合っている
  - 全体配信なのに強すぎる刈り取りにしない
  - 温まった segment には回りくどすぎる説明をしない
- 次の接点の文脈を壊さない
  - 広告や LP で約束した promise をメールで引き継ぐ

#### 悪い

- 1 通で教育も予約も販売も全部やろうとする
- 件名で煽り、本文で説明に戻り、CTA で別オファーに飛ばす
- segment の温度と本文の強さがずれる
- 表示URLと実 hyperlink がずれる

### 新規設定の標準手順

1. Journey か Campaign かを決める
2. audience と対象条件を決める
   - tag
   - saved segment
   - exclude
3. メールの役割を 1 つに絞る
   - 教育
   - 予約
   - 直オファー
   - 横展開
   - 締切
4. main CTA を 1 本決める

5. 本文を入れたら、表示文字列ではなく実 hyperlink を確認する
6. 実際にクリックして、最終遷移先まで確認する
7. 送信後は `送信 -> 開封 -> クリック -> click-details` の順で見る

### AIライティング

- モデル: Claude（Anthropic API）
- インプット: セグメント情報 + 過去の配信実績（開封率・クリック率） + ブランドトーン
- アウトプット: 件名 + プレヘッダー + HTML本文
- テンプレート: 既存のMailchimpテンプレートをベースに本文部分のみAI生成

### 承認フロー

```
Phase 1（初期）: AI下書き → Slack通知 → 甲原承認 → 送信
Phase 2（安定後）: AI下書き → 自動送信（承認率が安定したら移行）
```

### Mailchimp ↔ CDP 双方向同期

Mailchimp側にしかないデータ（開封履歴・クリック履歴・バウンス）をCDPに還流:
- 配信後にレポートAPIで結果取得
- CDPに「メール開封率」「最終クリック日」等のカラムを追加（将来）
- これによりCDP側でさらに精度の高いセグメントが切れる

### current メールの作り方と製作者の意図

メール本文は、件名だけでなく `何の心理状態を作るか` で分けて読む方がズレにくい。current の代表型は大きく 5 つある。

#### 1. 本線ストーリー型

- representative
  - `SNS_Meta広告_7桁_1通目`
- 役割
  - みかみ本人の人物像と authority を先に入れて、読み手との距離を縮める
- 本文の動き
  - `どこで僕を知ってくれたか分からないので改めて...`
  - `東大中退 / 20億企業社長` などの肩書き
  - 外部動画やチャンネル実績
  - 最初の CTA
- 製作者の意図
  - いきなり売るのではなく、まず `この人の話なら読んでいい` を作る
  - main CTA を押さなくても、以降の nurture を読む理由を作る
- 良い理由
  - authority と story が先にあり、読者が「誰からのメールか」を理解しやすい
  - CTA は前に出しすぎず、教育導線の入口として置かれている

#### 2. 直オファー型

- representative
  - `センサーズFB広告_本気コンサルオファー (copy 05)`
  - `AIカレッジFB広告_AI導入プロコンサル (copy 02)`
- 役割
  - `今なら無料 / モニター募集 / 本気の人限定` の強い口実で個別相談へ寄せる
- 本文の動き
  - 冒頭で offer を一言で出す
  - `以前からたくさん声をもらっていた`
  - `精鋭コンサルティング部隊`
  - `6ヶ月後の売上が一桁変わるビジョン`
  - 誰に向いているかを箇条書き
  - CTA を 1 本に絞る
- 製作者の意図
  - 教育よりも予約率を優先し、`今押す理由` を作る
  - 「無料」「限定」「思いつき」の軽さで心理ハードルを下げる
- 良い理由
  - オファーが先、対象者定義が次、CTA が最後で、読む順番が明確
  - short.io が 1 本にまとまっていて、行動先が迷わない

#### 3. 横展開オファー型

- representative
  - `スキルプラスフリープラン1通目 (copy 03)`
- 役割
  - AI / SNS の本線読者を、スキルプラスの無料枝へ横展開する
- 本文の動き
  - `お金がなくて挑戦できない` という反論処理
  - `テレビCM / 累計4000名 / 林社長推薦` で社会的証明
  - `0円スキルGETプログラム`
  - 無料だが一部制限ありと明示
  - CTA を複数回差し込む
- 製作者の意図
  - 本線で温まった読者に対して、別商品ではなく `無料で入れる次の選択肢` を提示する
  - 費用障壁を崩して、接点数を増やす
- 良い理由
  - 反論処理 -> 権威付け -> 新オファー提示の順で自然
  - 既存の nurture から無理なくスキルプラス枝へ送れる

#### 4. 短期締切ブースト型

- representative
  - `AIカレッジFB広告_プロンプト企画3日目 (copy 02)`
  - `本気コンサルRe`
- 役割
  - すでに説明済みの offer に対して、期限と実例で最後の一押しをする
- 本文の動き
  - 冒頭で `本日23:59` などの締切
  - 受講者の声や具体例
  - いま参加しないデメリット
  - CTA の再提示
- 製作者の意図
  - 読者の迷いを切り、`今動く理由` を明確にする
  - 途中で開封した人を再点火する
- 良い理由
  - 既に理解している前提で書かれていて、説明が冗長でない
  - 実例と締切の組み合わせで、行動トリガーが強い

#### 5. 単発PR告知型

- representative
  - `3/11 AIキャリアセミナーPR_1通目`
  - `3/11 AIキャリアセミナーPR_2通目`
  - `3/10 セミナーPR_1通目 (フリープラン)`
- 役割
  - 今この期間だけ打ちたい promotion を、特定 segment または広い母集団へ一気に届ける
- 本文の動き
  - 件名で強く attention を取りにいく
  - 冒頭で対象者や問題を限定する
  - 今回の企画や offer を短く提示する
  - CTA はすぐ押せる位置に置く
- 製作者の意図
  - Journey のような長い教育ではなく、既存認知を前提に短く反応を取る
  - 期間限定やイベント告知で、判断を後ろ倒しにさせない
- 良い理由
  - `今の告知` に必要な情報だけへ絞りやすい
  - segment を切れば、広い配信より開封率とクリック率を上げやすい

### 良いコンテンツと悪いコンテンツの判断

- 良い
  - メールの役割が `教育 / 予約 / 横展開 / 締切ブースト / 単発PR` のどれかに明確に振り切れている
  - CTA が 1 本または少数に絞られている
  - subject, body, CTA の約束が一致している
  - 「誰に向けたメールか」が本文前半で分かる
  - ハイパーリンクを使う場合でも、表示文字列と実際の埋め込み URL が一致していることを配信前に確認している
- 悪い
  - 1 通の中に `教育 / オファー / 別商品案内` を詰め込みすぎる
  - authority を積みすぎて、何をしてほしいかが遅い
  - CTA が複数方向に割れている
  - 期限訴求だけ強く、文脈や対象者定義が薄い
  - 表示文字列の URL を見て安心し、実際のリンク先を押して確認していない

### current / legacy copy の読み方

- `copy` が付く = legacy ではない
- まず見るべきは
  - `journey status`
  - `step status`
  - `queue_count`
  - `emails_sent`
- `copy` でも
  - queue が残り
  - sent が積み上がり
  - click report に short.io / UTAGE の current CTA が出ていれば
  current として扱う
- Campaign でも
  - `status = sent`
  - `emails_sent > 0`
  - click report に main CTA が出ていれば
  `copy` 付きでも current 実績として扱う
- 逆に `copy` でなくても
  - paused
  - queue 0
  - click が枯れている
  なら current とは限らない

### current / legacy の判断辞書

まず `名前` ではなく `挙動` で切る。current の読み方は次の 4 区分がズレにくい。

#### 1. current

- 条件
  - `journey = sending`
  - `step = active`
  - `queue_count > 0`
- 読み方
  - 今もこの step に入る人が残っていて、送信待ちがある
- representative
  - `スキルプラスフリープラン1通目 (copy 03) = queue 191 / sent 31,173`
  - `センサーズFB広告_本気コンサルオファー (copy 05) = queue 1 / sent 17,987`
  - `AIカレッジFB広告_個別3日間シナリオ_3日目 (copy 02) = queue 1 / sent 28,146`
  - `AI新卒プロモーション_1通目(4/3) (copy 02) = sent 269,919 / open_rate 10.85% / main click は direct LIFF`
  - `Resend: AIキャンプ = sent 247,861 / main click は UTAGE page`

#### 2. completed but still valid

- 条件
  - `step = active`
  - `queue_count = 0`
  - `emails_sent` は十分ある
- 読み方
  - その step 自体は current の構成要素だが、今この瞬間の送信待ちはない
  - evergreen の末尾や、最近ちょうど queue が尽きた step に起こる
- 扱い
  - legacy と決めつけず、同 Journey の前後 step と click report を見る
- representative
  - `フリープラン_入会完了メール = active / queue 0 / sent 2,152`
  - `AIカレッジFB広告_メインシナリオ1通目 (copy 05) = active / queue 0 / sent 32,169`
  - `SNS_Meta広告_7桁_1通目 = active / queue 0 / sent 18,369`

#### 3. historical evidence

- 条件
  - `step = paused`
  - `emails_sent > 0`
- 読み方
  - 過去に実運用された証拠はあるが、今は止めている
- 扱い
  - content pattern や historical reference としては価値がある
  - current 修正対象には入れない
- representative
  - `月額プラン2通目 = paused / queue 3,823 / sent 170`
  - `止める = paused / queue 740 / sent 3,823`
  - `送信停止 = paused / queue 2,821 / sent 42,290`
- 補足
  - `paused` なのに `queue_count > 0` が残る step もある
  - これは `今も current` の意味ではなく、停止時点で queue が残った historical remnant として読む方が安全

#### 4. draft / legacy candidate

- 条件
  - `step = paused`
  - `queue_count = 0`
  - `emails_sent = 0` または click report が枯れている
- 読み方
  - current 本線から外れている可能性が高い
- 扱い
  - ただし `copy` や古い命名だけで断定せず、Journey 全体の status と周辺 step を確認する
- representative な見方
  - `paused + queue 0 + sent 0` は、まず draft / unused 候補
  - `paused + queue 0 + sent が極小` でも、同 Journey に current step が並ぶなら historical 残骸の可能性がある
  - つまり最後は `Journey 全体の status` と `前後 step の queue / sent` を必ず合わせて見る

#### Campaign 側の current / draft 判定

- current に近い
  - `type = regular`
  - `status = sent`
  - 日付付きタイトルや promotion 文脈が current 施策と一致
- draft / pending
  - `type = regular`
  - `status = save`
  - `emails_sent = 0`
- representative
  - `3/11 AIキャリアセミナーPR_1通目 (copy 02) = save / sent 0`
  - `Resend: AIキャンプ (copy 01) = save / sent 0`
  - `save + sent 0 + click 0` なら、配信候補や draft として読む
- 注意
  - `copy` が付いていても、それだけで old と決めない
  - ただし `save + sent 0` の段階では、まだ current 実績とは分ける
  - historical な `regular campaign` では direct LIFF を main CTA にしていることがある
  - ただし今後の新規実装では、campaign でも `main CTA = short.io` を正にする
  - `Resend` という名前も、それだけでは legacy 根拠にならない
    - 実際に `Resend: AIキャンプ` は `sent 247,861` の current 実績を持っていた

### 何が良い判定で、何が危険か

- 良い
  - `queue_count > 0` を最優先で見る
  - `copy` の有無より `step status / sent / click` を優先する
  - current 修正対象は `active + queue > 0` に絞る
- 悪い
  - `last_started_at` だけで current と決める
  - `copy` が付いているから old と決める
  - `step = active` だけで今も送られていると決める
- NG
  - paused step を current と誤認して本番修正する
  - sent が大きいだけで今も使っていると断定する

### 顧客行動データとして追う指標

Mailchimp 単体で閉じず、導線全体では次の 5 点を 1 本で追う必要がある。

1. メール送信数
2. 開封数 / 開封率
3. リンククリック数 / クリック率
4. 遷移先 UTAGE ページの訪問者数
5. 遷移先ページでのコンバージョン数

この 5 点をつなげて初めて、どこで離脱しているかを判断できる。

### 動線データ基盤の目的

CDP にそのまま入れず、別の `動線データ` として持つ前提で考える。  
第一目的は `分析用の可視化` と `マーケティング施策立案のためのデータ収集`。

スキルプラス事業でまず見るべき主要変数は次の 4 つ。

1. CPA
2. 個別相談数
3. 営業成約率
4. 解約率

この 4 変数を改善するために、`送信 -> 開封 -> クリック -> 訪問 -> 行動` の導線データを別レイヤーで持つ。

### current のコンテンツ構造と制作者の意図

コンテンツとは、`コミュニケーションの手段であり、ユーザーをこちらが意図した認識に変換するもの`。

Mailchimp では、この認識変換を `非リアルタイムの長文教育 / 心理変容 / オファー提示` で担う。

Mailchimp の役割は、単にメールを送ることではない。

- まだ冷たい、または半信半疑の見込み客に対して
- ストーリー、authority、実例、緊急性を使い
- `理解している / 信じられる / 今動く理由がある`
  状態まで温度を上げる

つまり Mailchimp は、Addness の導線の中では `認識と温度を段階的に変える本線` として読む。

したがって良い Mailchimp コンテンツは、
- 何を教えるか
- 何を信じさせるか
- 次に何を押してほしいか
が 1 通ごとに明確で、前後のメールとつながっている。

Mailchimp の current メールは、単に件名を量産しているのではなく、かなり明確な役割分担で作られている。

#### 1. 本線 7桁オプトインの役割

- 代表例
  - `AIカレッジFB広告_メインシナリオ1通目 (copy 05)`
  - `SNS_Meta広告_7桁_1通目`
- 共通する意図
  - いきなり売らず、まず `みかみ` の人物ストーリーと authority を入れる
  - 「第1話 / 第2話 ...」で open loop を作り、連続閲覧を促す
  - 広告で感じた期待を `自分にもできそう` に変える
- current で見えている CTA
  - AI 本線 1通目は `https://skill.addness.co.jp/meta-ai2`
  - SNS 本線 1通目は `https://skill.addness.co.jp/metaad-sns2`
- 本文で実際にやっていること
  - AI 本線 1通目は `東大生社長みかみがAI未経験から年収2億ベースで稼ぐまでの軌跡` を主題にして、人物ストーリーと実績で期待を維持している
  - SNS 本線 1通目は `元東大生社長みかみが年商2億稼ぐまでの軌跡` を主題にして、動画やストーリーを複数差し込みながら `もっと見たい` 状態を作っている
- 良いと判断できる点
  - subject が `ストーリー / 権威 / 続き` を担っている
  - 本文は長文でも、役割が `売る前の期待維持` なので整合している

#### 2. 途中の直オファー系メールの役割

- 代表例
  - `AIカレッジFB広告_AI導入プロコンサル (copy 02)`
  - `センサーズFB広告_本気コンサルオファー (copy 05)`
- 共通する意図
  - 本線で温めた読者へ、急に距離を縮める
  - `緊急告知 / 直接コンサル` のラフさで、堅い売り込み感を下げる
  - 個別相談やコンサルへの遷移を強く取りにいく
- current で見えている CTA
  - AI 側は `https://skill.addness.co.jp/meta-ai4`
  - SNS 側は `https://skill.addness.co.jp/metaad-sns5`
- 本文で実際にやっていること
  - AI 側は `【緊急告知！！】直接コンサルしましょうか？笑` というラフな件名で入り、個別に近い距離感を先に作ってから CTA へ寄せている
  - SNS 側の `本気コンサルRe` は `間も無く締め切ります` で urgency を前面に出し、迷っている人を押し込む役割が強い
- 良いと判断できる点
  - 件名と本文のトーンが `思いつき感 / 個別感` に寄っていて、直売りメール特有の警戒を下げている
  - 本線メールと比べて CTA がかなり明確

#### 3. スキルプラス横展開メールの役割

- 代表例
  - `スキルプラスフリープラン1通目 (copy 03)`
  - `スキルプラスフリープラン1通目 (copy 02)`
- 共通する意図
  - AI / SNS で集めた人に対し、別の価値提供口としてスキルプラスを差し込む
  - `無料 / 期間限定 / 挑戦しやすさ` を前面に出し、より広い層へ口を広げる
- current で見えている CTA
  - AI 側は `https://school.addness.co.jp/p/TzYRwepfqzFq?...`
  - SNS 側は `https://school.addness.co.jp/p/hNuqvGx9VJgZ?...`
- 本文で実際にやっていること
  - SNS 側 2通目は `たった10時間で1,300名が押し寄せた...無料のアレの正体とは？` という件名で、無料性と social proof を使って参加ハードルを下げている
  - AI / SNS ともに、高単価の直接訴求ではなく `無料で入れる別の価値提供口` として差し込んでいる
- 良いと判断できる点
  - 本線のストーリーで温度を上げた後に、低いハードルの offer を置いている
  - いきなり高単価へ行かず、無料体験やフリープランを挟んでいる

#### 4. 個別3日間シナリオの役割

- 代表例
  - `AIカレッジFB広告_個別3日間シナリオ_1日目 (copy 02)`
  - `センサーズFB広告_個別3日間シナリオ_1通目ABテストA`
- 共通する意図
  - `あなた専用` という言葉で、自分ごと化を一段上げる
  - 3日間で urgency と social proof を積み、面談や作成会へ押し込む
  - 1日目はプレゼント、2日目は反響、3日目は締切で詰める
- current で見えている CTA
  - AI 側は `school.addness.co.jp/p/uUfErKGsQYZ2?...`
  - SNS 側は `school.addness.co.jp/page/KAtTy0tucas2?...`
- 本文で実際にやっていること
  - SNS 1通目 ABテストB は `【今だけ無料！】"あなた専用" SNS運用 1st副業スタートプログラム開催！！` と promise を先に立て、自分ごと化を最短で取りにいく
  - AI 側の current では `💬Google創業者に聞いた、僕らがGAFAを超える方法` のように、大きい夢や思想を使って `みかみ側の世界観に共感した人だけを濃くする` 役割を持つものもある
- 良いと判断できる点
  - `件名 -> 冒頭コピー -> CTA` が同じ圧力でつながっている
  - AB テストでは promise を変えつつ、構造は大きく崩していない

#### Mailchimp コンテンツ全体の理解

- コンテンツとは、`コミュニケーションの手段であり、ユーザーをこちらが意図した認識に変換するもの`
- Mailchimp で作るコンテンツは、その中でも `非リアルタイムの長文教育` に強い
- Mailchimp は `長文で教育しながら温度を上げる` 本線
- 件名は `物語 / 権威 / 危機感 / 緊急性 / 個別感` のどれかを強く担う
- 本文は長くてもよく、その代わり `1通ごとの役割` がかなり明確
- Lステップのような短い operational message ではなく、`教育と心理変容` が主目的
- current の良し悪しは、文章量ではなく `この1通で相手の心理をどこまで進めるか` が明確かどうかで判断する

#### 制作者意図を読む時の問い

Mailchimp の current コンテンツを読む時は、本文をうまく書いているかより先に、次を順に見る。

1. このメールは、直前の接点でどんな誤解や温度不足を受けているか
2. この 1 通で、どの認識を変えたいか
3. 変えた結果、次に何を押してほしいか
4. 件名、冒頭、本文、CTA がその 1 つの目的に揃っているか
5. 次の接点でも同じ文脈が続いているか

つまり、良い Mailchimp コンテンツは `文章が上手い` より、`変えたい認識が 1 通ごとに明確` である。

#### 良い / 悪いの判断をさらに具体化

- 良い
  - 件名の promise と本文冒頭の promise が一致している
  - 本文のストーリーや実例が、最後の CTA の説得材料になっている
  - CTA 先の short.io / UTAGE ページが、本文で作った期待とズレない
  - 1 通の役割が `教育` なのか `直オファー` なのか混ざっていない
- 悪い
  - 件名は強いが、本文の役割が散っている
  - 本文で温めた温度に対して、CTA が急すぎる
  - 本文は教育なのに、CTA だけが hard sell になっている
  - クリック先ページの約束が、メール本文の約束と変わる

#### シームレスな顧客体験として見たときの Mailchimp

- Mailchimp 単体で完結させるのではなく、前後の接点と滑らかにつながることが重要
- つまり 1 通ごとに見るべきなのは
  1. 直前の接点でユーザーが何を見たか
  2. このメールで何を理解させるか
  3. 次の接点で何をしてほしいか
- 良い current メールは
  - subject
  - 冒頭
  - 本文のストーリー
  - main CTA
  が同じ方向を向いている
- 悪いメールは
  - subject だけ強い
  - 本文の役割が曖昧
  - CTA が文脈とずれる
  というズレが起きる

#### current のコンテンツ判断で見る順番

1. これは `Journey` か `Campaign` か
2. 役割は `教育 / 直オファー / 横展開 / 締切ブースト / 単発promotion` のどれか
3. subject, 冒頭, CTA が同じ役割を向いているか
4. short.io や UTAGE の遷移先が、その役割に合っているか
5. 送信後は `送信 -> 開封 -> クリック -> click-details` の順で結果を見る

#### 直近 regular campaign の認識変換の読み方

- `3/11 AIキャリアセミナーPR_1通目`
  - subject: `【🚨非IT職の方限定🚨】`
  - main CTA: `direct LIFF`
  - 変えたい認識:
    - `自分向けか分からない`
    - を
    - `これは自分向けの urgent な案内だ`
    に変える
- `3/11 AIキャリアセミナーPR_2通目`
  - subject: `【非IT職の方へ朗報📣】`
  - main CTA: `direct LIFF`
  - 変えたい認識:
    - `AI は IT 職向け`
    - を
    - `非IT職でも今のうちに動く価値がある`
    に変える
- `3/10 セミナーPR_1通目 (フリープラン)`
  - subject: `できる人ほど、気づくのが遅い。AI時代の損する働き方とは...`
  - main CTA: `direct LIFF`
  - 変えたい認識:
    - `自分はまだ大丈夫`
    - を
    - `今の働き方のままだと損をする`
    に変える
- つまり current regular campaign は、`すぐ売る` というより
  - 危機感
  - 当事者意識
  - urgency
  を先に作って、次の予約や登録へ送る使い方が多い

#### Journey と Campaign のコンテンツ差

- Journey
  - evergreen 前提
  - 連続接触で、認識を段階的に変える
  - 1 通単体より、前後の流れまで含めて設計する
- Campaign
  - 単発配信
  - 1 通で `今伝えるべきこと` を強く伝える
  - segment を切ったうえで、 urgency や topicality を前に出しやすい

つまり、同じ Mailchimp でも
- Journey は `流れの中で認識を育てる`
- Campaign は `その瞬間に必要な認識を一気に作る`
で役割が違う

### 実値で確認できたこと

- AI Facebook の `スキルプラスフリープラン1通目 (copy 03)` の本文には
  `school.addness.co.jp/p/TzYRwepfqzFq?...`
  が入っていた
- click report では
  `skill.addness.co.jp/ytad-ai2A`
  のような short.io が実際に押されていた

### URL 設置時の注意

- Mailchimp では、表示文字列と実際のハイパーリンク先がズレる事故が起こりうる
- そのため、本文内に URL を置いたら、配信前に必ず実際にクリックして確認する
- 確認順は次で固定する
  1. メール本文で CTA の表示文字列を確認する
  2. ハイパーリンクの実 URL を確認する
  3. 実際に押して、意図した short.io / UTAGE / 外部ページへ飛ぶか確認する
  4. short.io を使う場合は、short.io の最終遷移先まで確認する
- 特に `見えている URL` と `埋め込み URL` が違うケースを前提にする
