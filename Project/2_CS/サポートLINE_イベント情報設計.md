## 完成イメージ

受講生がリッチメニューの「イベント情報」を押すと、こう動きます。

```
STEP 1  リッチメニュー「イベント情報」をタップ
        ↓
STEP 2  トーク画面にイベント情報が表示される（2段構成）

        【1段目: カレンダー画像（単体表示）】
        ┌─────────────────────────┐
        │         3月              │
        │    月間カレンダー          │
        │  イベント日に●マーク       │
        └─────────────────────────┘

        【2段目: イベントカード（カルーセル）】
        ┌──────────┐ ┌──────────┐
        │グループ   │ │スキルプラス│
        │コンサル   │ │  合宿    │  ← 横にスワイプ
        │3/7(金)   │ │3/8-9    │
        │[予約する] │ │[予約する] │
        └────┬─────┘ └──────────┘
             ↓
STEP 3 「予約する」を押すと、UTAGEの詳細ページに飛ぶ
        → イベントの詳しい情報と魅力的な説明が書いてある
        → ページ内の予約ボタンを押すと予約完了
        → このボタンにLステップの流入経路リンクが仕込んであり、
          押した瞬間にタグが自動で付く

STEP 4  予約完了後、前日〜当日にLINEで案内が届く
        「本日20:00〜 グループコンサルです。Zoomリンク: https://...」
```

## 表示するイベントの種類

| イベント種別 | 表示先 |
|------------|-------|
| グループコンサル | サポートLINE |
| セミナー | サポートLINE |
| スキルプラス合宿 | サポートLINE |

| ランチ会 | **プライムLINEのみ**（サポートLINEには出さない） |

## 仕組みの全体図

```
リッチメニュー「イベント情報」タップ
  │
  │ リッチメニューの設定でテンプレート送信（2通連続）
  ▼
1段目: カレンダー画像（単体テンプレート）
2段目: イベントカード カルーセル
  │
  │ 各カードの「予約する」ボタン → UTAGEのページURLを開く
  ▼
UTAGEの詳細ページ
  │
  │ ページ内の予約ボタンを押す
  │ → ボタンのリンク = Lステップの流入経路リンク
  │ → 流入経路の設定でタグが自動付与される
  ▼
タグ「イベント予約_○○_○○」が付く
  │
  │ このタグを持っている人に一斉配信
  ▼
前日リマインド & 当日案内がLINEに届く
```

**タグ付けの仕組み（ポイント）**:
- UTAGEページの予約ボタンの中に、**Lステップの流入経路リンク**を設定する
- 流入経路の設定で「タグを追加する」アクションを入れておく
- 受講生がボタンを押す → 流入経路リンクを踏む → タグが自動で付く

## 作業手順

---

### 手順1: リッチメニューの設定

**場所**: Lステップ > リッチメニュー

「イベント情報」ボタンのタップ時アクションを設定します。

| 設定項目 | 値 |
|---------|-----|
| タップ時のアクション | **テンプレートを送信** |
| 送信するテンプレート | 手順4で作る「イベント一覧」テンプレート |

---

### 手順2: タグを準備する

**場所**: Lステップ > タグ管理

- [ ] タグフォルダ「★イベント予約」を作成
- [ ] 各イベント用のタグを作成

**タグの命名ルール**:
```
イベント予約_[イベント名の略称]_[日付4桁]

例:
  イベント予約_グルコン_0307
  イベント予約_合宿_0308
  イベント予約_セミナー_0311
```

---

### 手順3: 流入経路を作成する

**場所**: Lステップ > 流入経路分析

イベントごとに流入経路を1つ作成します。
この流入経路のURLが、UTAGEの予約ボタンに入るリンクです。

| 設定項目 | 値 |
|---------|-----|
| 流入経路名 | イベント予約_グルコン_0307 |
| アクション | **タグ追加**: 「イベント予約_グルコン_0307」 |

流入経路を作成すると**URL**が発行されます。このURLを控えておいてください。
→ 手順5でUTAGEの予約ボタンに設定します。

---

### 手順4: テンプレートを作る（2段構成）

**場所**: Lステップ > テンプレート

- [ ] フォルダ「★イベント情報」を作成

#### テンプレートA: カレンダー画像（1段目・単体表示）

- 月間カレンダー画像を1枚表示するだけのテンプレート
- イベント開催日に●マーク、種別ごとに色分け

#### テンプレートB: イベントカード カルーセル（2段目）

- フレックスメッセージ、パネルサイズ L
- 各パネル = 1イベント（日付が早い順）

```
[パネル1]        →  [パネル2]        →  ...
イベント①           イベント②
（日付が早い順）      （日付が早い順）
```

各パネルの構成:
- 画像ブロック: イベント種別のバナー
- テキストブロック: イベント名 / 日時 / 場所
- ボタンブロック: 「予約する」→ UTAGEの詳細ページURLを開く

---

### 手順5: UTAGEに詳細ページを作る

**場所**: UTAGE

イベントごとに1ページ作成します。

#### ページの構成

```
┌───────────────────────────────────┐
│                                   │
│  ヘッダー画像（イベントの魅力訴求）  │
│                                   │
├───────────────────────────────────┤
│                                   │
│  イベント名                        │
│                                   │
│  魅力・説明文                      │
│  「○○を一緒に学びませんか？         │
│   △△な方におすすめです」            │
│                                   │
│  ────────────────                 │
│                                   │
│  日時: 3/7(金) 20:00〜21:30       │
│  場所: Zoom（オンライン）           │
│  定員: 30名                        │
│  参加費: 無料（会員特典）            │
│                                   │
│  当日の内容:                       │
│  ・○○のワーク                     │
│  ・質疑応答タイム                  │
│  ・次のアクション共有               │
│                                   │
│  ┌───────────────────────┐        │
│  │    この日程で予約する    │        │  ← ★ここが重要★
│  └───────────────────────┘        │
│                                   │
└───────────────────────────────────┘
```

#### 予約ボタンの設定（重要）

ページ内の「この日程で予約する」ボタンに、**手順3で発行した流入経路リンクURL**を設定します。

```
予約ボタンのリンク先 = Lステップの流入経路URL

受講生がこのボタンを押す
  → 流入経路リンクを踏んだことになる
  → Lステップが自動でタグ「イベント予約_グルコン_0307」を付ける
  → 前日リマインド・当日案内の配信対象になる
```

---

### 手順6: 前日リマインド＆当日案内を準備する

**場所**: Lステップ > テンプレート & 一斉配信

イベントごとに「テンプレート作成 → 一斉配信の予約」をセットで行います。

#### 6-1. テンプレートを作る

**前日リマインド**:
```
┌─────────────────────────────┐
│                             │
│  明日のイベントのお知らせ     │
│                             │
│  グループコンサル vol.28      │
│  明日 3/7(金) 20:00〜       │
│                             │
│  お忘れなく！               │
│                             │
└─────────────────────────────┘
```

**当日案内**:
```
┌─────────────────────────────┐
│                             │
│  本日のイベントご案内         │
│                             │
│  グループコンサル vol.28      │
│  本日 20:00〜21:30          │
│                             │
│  ┌─────────────────────┐    │
│  │   Zoomに参加する      │    │  ← ボタンにZoomリンク
│  └─────────────────────┘    │
│                             │
│  ※ 5分前からの入室を        │
│    お願いします              │
│                             │
└─────────────────────────────┘
```

| テンプレート | 命名例 | フォルダ |
|------------|--------|---------|
| 前日リマインド | 【前日リマインド】グルコン 3/7 | ★イベント情報 |
| 当日案内 | 【当日案内】グルコン 3/7 | ★イベント情報 |

#### 6-2. 一斉配信を予約する

**場所**: Lステップ > 一斉配信

1件のイベントにつき2件の配信予約をします。

| 配信 | 配信日時 | 絞り込み条件 | テンプレート |
|------|---------|------------|------------|
| 前日リマインド | イベント前日 18:00 | タグ「イベント予約_グルコン_0307」あり | 【前日リマインド】グルコン 3/7 |
| 当日案内 | 当日 開催2時間前 | タグ「イベント予約_グルコン_0307」あり | 【当日案内】グルコン 3/7 |

## 週次の運用手順

毎週、翌週のイベント情報を更新するときの流れです。

```
1. 翌週のイベントスケジュールを確認する

2. 新しいイベントごとに:
   a. Lステップでタグを作成（イベント予約_[名称]_[日付]）
   b. Lステップで流入経路を作成 → URLを控える
   c. UTAGEで詳細ページを作成（予約ボタンに流入経路URLを設定）
   d. Lステップでリマインド・当日案内テンプレートを作成
   e. Lステップで一斉配信を予約（前日リマインド + 当日案内）

3. カルーセルテンプレートを更新
   a. 終了したイベントのパネルを削除
   b. 新しいイベントのパネルを追加（日付順に並べる）
   c. 「予約する」ボタンのリンク先をUTAGEページURLに設定
   d. カレンダー画像を差し替え

4. リッチメニューの設定は変更不要
   （テンプレートの中身を更新すれば自動的に反映される）
```

## プライムLINEへの展開

サポートLINEでの仕組みが完成したら、スキルプラス【プライム】にも同じ構成で展開します。
プライム側にはランチ会のカードを追加するだけで、仕組みは同じです。
