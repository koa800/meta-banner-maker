# 引き継ぎメモ

## 目的
- `導線ツール` を厳しい 10 点基準まで引き上げる
- 10 点の定義は、`役割 / ゴール / 必要変数 / ワークフロー / NG / 正誤判断` を持ち、exact な UI 手順で迷わず実装できる状態

## 固定前提
- UI を説明する時は、抽象語より画面ラベルを優先する
- テスト送信先は常に `甲原 海人`
- テスト資産は残さない
- `コンテンツ` は上位概念であり、広告 / SNS / LP / メール / LINE / 会員サイトまで含む
- `カルーセルメッセージ(新方式)` と `フレックスメッセージ` は、Lステップ UI 上の実名称で扱う

## 現在地
- Lステップの `テンプレート` 領域は live で end-to-end をかなり確認できている
  - `標準メッセージ`
  - `カルーセルメッセージ(新方式)`
  - `フレックスメッセージ`
  - `テンプレートパック`
- `フレックスメッセージ` は
  - `テンプレート -> 新しいテンプレート -> フレックスメッセージ`
  - `edit メッセージを作成`
  - `画像 / タイトル / テキスト / ボタン / 動画`
  - `ボタン -> アクション設定`
  - 一覧右端 `... -> テスト送信`
  の current 導線を固定済み
- `テンプレートパック` は
  - 作成
  - step 追加
  - `甲原 海人` への `テスト送信`
  - step 削除
  - pack 削除
  まで確認済み
- `アクション管理` は live で current field と save route を固定済み
  - 一覧: `https://manager.linestep.net/line/action`
  - 上部: `新しいフォルダ / 新しいアクション / 並び替え`
  - 新規 modal の action type:
    - `テキスト送信`
    - `テンプレート送信`
    - `タグ操作`
    - `友だち情報操作`
    - `シナリオ操作`
    - `メニュー操作`
    - `リマインダ操作`
    - `対応マーク・表示操作`
    - `イベント予約操作`
    - `コンバージョン操作`
  - 詳細取得: `GET /api/actions/{id}`
  - save: `POST /api/action/register`
  - helper:
    - `python3 System/scripts/lstep_auth.py`
    - `python3 System/scripts/lstep_action_catalog.py list --limit 20`
    - `python3 System/scripts/lstep_action_catalog.py inspect --id 27783572`
    - inspect は `GET /api/action/data/{id}` と `GET /api/actions/{id}/texts` に寄せた
  - ただし `新しいアクション -> 保存 -> 削除` の end-to-end はまだ未固定
  - 2026-03-12 時点では auth probe で
    - `chrome_auto / chrome_default / chrome_profile_9 / chrome_debug_default`
      が全て `401 Unauthorized`
    - `cdp.alive = false`
    - login page は `POST /account/login`
    - field は `_token / name / password`
    - `reCAPTCHA enabled=true`
    が確認済み
  - つまり current の blocker は `UI selector` に加えて `browser login 再取得が必要` になっている
  - 復旧優先順位は
    1. `CDP 既存セッション再利用`
    2. `CDP + 自動入力 + checkbox 試行`
    3. `手動 browser login`
    4. `cookie 直利用`
    の順で固定
  - ただし current Cursor sandbox では
    - `open URL`
    - `osascript -> Google Chrome`
    - `Chrome direct launch`
    が全て失敗したため、実際の復旧は `手動 browser login` が最短
- `テンプレート inspection helper` は current helper を追加で強化済み
  - `python3 System/scripts/lstep_template_catalog.py inspect --id 225310549`
  - `カルーセルメッセージ(新方式)` も `GET /api/line/template/{id}` から
    - `alt_text`
    - `panel text`
    - `action_type`
    - `action_form_id`
    - `action_liff_size`
    を exact に抜ける
- UTAGE は
  - `ファネル / ページ / 登録経路 / 商品 / detail / action / 会員サイト / 動画 / メディア`
  の役割を理解済み
  - `登録経路` は `どこから来たかを知りたい時に切る分析軸`
  - `商品 -> detail -> action -> bundle`
    の current 基本線は読めている
- Mailchimp は
  - `Journey = evergreen`
  - `Campaign = 単発セグメント配信`
  を固定済み
  - current regular campaign の recent 実測も取れている
  - `mailchimp_journey_snapshot.py` は
    - `--journey-id` の int/string 比較ズレを修正
    - `email_step_summary`
    - `current_email_steps`
    - `email_steps[].state_reason`
    を返すように補強済み
  - representative:
    - `10934 / UTAGE_AIカレッジ_Facebook_7桁オプトイン2025-10-15`
    - `total=36 / current=12 / queue_zero=24`
  - `mailchimp_campaign_snapshot.py` は
    - `GET /reports/{campaign_id}/click-details`
      を集計して `top_clicked_url` を返すように補強済み
    - representative:
      - `e836af06b7 / 3/11 AIキャリアセミナーPR_2通目`
      - `top_clicked_url = direct LINE`
      - `top_clicked_total_clicks = 145`
  - 直近は `direct LINE` main CTA が多いが、今後こちらが新規で作る標準は `main CTA = short.io`
- short.io は
  - 役割
  - 判断基準
  - URL 管理シート
  - API
  までかなり固い
- Zapier は relay layer として representative を追加済み
  - current 主戦場は `Webhooks by Zapier -> Mailchimp Add/Update Subscriber`
