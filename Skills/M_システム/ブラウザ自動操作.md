# ブラウザ自動操作メモ

## reCAPTCHA v2 checkbox 突破

- **方法**: `cliclick`（OSレベルマウス操作）で突破可能。Chrome拡張のクリックではiframe内に到達しないが、cliclickなら通る
- **手順**: ①AppleScriptでChromeウィンドウを固定位置に配置 `{0, 25, 1000, 800}` ②`screencapture -x` + Pillow で画面キャプチャ→座標特定 ③物理ピクセル÷2=macOSポイント座標 ④`cliclick c:X,Y`でクリック
- **注意**: 複数Chromeウィンドウがある場合、AppleScriptで対象タブのウィンドウを最前面にする必要あり

## Claude in Chrome MCP 接続切断の対策

### よくある切断パターン

| 症状 | 原因 | 対策 |
|------|------|------|
| 「Browser extension is not connected」 | Chrome拡張がスリープ or 切断 | claude.aiタブをリロード or Chrome再起動 |
| 「Cannot access a chrome-extension:// URL」 | 1Password等の他拡張が干渉 | JavaScript DOM操作で回避（直接click/typeしない） |
| 「Detached while handling command」 | タブが非アクティブでdetach | 操作前にtabs_context_mcpで状態確認 |
| 長時間操作中に突然切断 | 拡張のアイドルタイムアウト | 操作間隔を空けすぎない |

### MacBook（手動セッション）での復旧手順

1. **claude.ai のタブをリロード**（Cmd+R）— 最も効果的
2. ダメなら **Chrome拡張アイコン** をクリックして再接続
3. 最終手段: **Chrome 再起動**（タブは復元される）

### 予防策

- **1Password 干渉回避**: input フィールドへの直接 click/type ではなく JavaScript の `nativeInputValueSetter` を使う
- **操作の冪等性**: 途中で切断されても、再接続後に続きから作業できるよう状態を把握しておく
- **tabs_context_mcp を最初に呼ぶ**: セッション開始時に必ず実行してタブ情報を取得

### Mac Mini（日向エージェント）での自動復旧

- `hinata_agent.py` の `ensure_chrome_running()` が毎サイクルで CDP ポート 9223 の疎通を確認
- プロセスが生きていても CDP が応答しなければ自動で Chrome 再起動
- `claude_executor.py` の `_run_claude_with_retry()` が MCP 切断を検知 → Chrome 再起動 → --chrome で再試行
