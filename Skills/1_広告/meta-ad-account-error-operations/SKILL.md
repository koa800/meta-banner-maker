# meta-ad-account-error-operations

## Overview

Meta広告のエラー対応は、原因を全部説明することより `相手が今やること` を先に固定する方が強い。  
この skill は、`設定ミス / AI誤認 / ポリシー違反 / セキュリティ停止 / 長期未使用停止` を短く切り分け、本文は短く、詳細は資料へ逃がすための型。

## ゴール

- エラーを 1 段で切る
- 相手が今やることを 1-2 個に絞る
- 余計な一般論を本文に詰め込まない
- 必要なら詳細資料や公開マインドマップへつなげる

## 必要変数

- 何が止まったか
  - `広告`
  - `Facebookページ`
  - `Instagram連携`
  - `広告アカウント`
  - `ビジネスマネージャ`
  - `Facebook個人アカウント`
- エラー文面
- 新規作成直後か、既存運用中か
- `審査リクエスト` が出ているか
- `本人確認` が出ているか
- 広告、画像、LPにポリシー違反の心当たりがあるか

## 10秒切り分け

- `設定エラー`
  - 画面の指示どおり修正する
- `作りたてアカウントの制限`
  - `審査リクエスト` と `本人確認` を先に進める
- `広告却下`
  - テキスト、画像、LPを見直す
  - 誤判定っぽいなら再審査
- `セキュリティ停止`
  - 不正利用判定や本人性の問題を疑う
  - 本人確認とサポート連絡
- `長期未使用停止`
  - 再審査とサポート連絡を先に行う

## Workflow

1. まず `今やること` を 1 文で出す
2. 次に `なぜその対応か` を 1-2 文で添える
3. その場で使える詳細資料があれば貼る
4. 返信文は短く保つ
5. 詳細なポリシーや再作成手順は `references/workflow.md` に逃がす

## 返信文の型

基本形はこれで十分。

```text
お疲れ様です！
甲原です！
[今やること]
[理由]

詳しくは下の資料を見ながら進めてください！
[リンク]
```

`今やること` が曖昧なまま、原因候補を 5 個並べない。  
相手を止める返しではなく、相手を前に進める返しを優先する。
確証が薄い場面では `1番いい` のような断定を避け、自分たちが実際にやっていることと推定を混ぜない。
受講生への質問回答は単独で読まれる前提で、`この場合` のような指示語を避け、エラー名や対象を主語で置く。

## 高頻度パターン

### 作りたてアカウントの制限

- 先に伝えること
  - `審査リクエスト` を出す
  - `本人確認` があればそのまま進める
- 補足
  - AI誤認のことが多い
  - 放置より迅速対応を優先する

### 広告却下

- 先に伝えること
  - テキスト、画像、LPのどこを直すか確認する
  - 誤判定っぽければ再審査
- 補足
  - よく当たるのは `個人的特質 / 誤解を招く表現 / 迂回 / LP不整合`

### 返答待ちで止まった

- 先に伝えること
  - Metaサポートへ問い合わせる
  - チャットがあればチャット優先
- 補足
  - 返答は数日以上かかることがある
  - しつこすぎる連絡は逆効果

### 復活しない

- 先に伝えること
  - 再審査とサポートをやり切ったか確認する
  - それでも不可なら新規作成を検討する
- 補足
  - 先に新規作成へ飛ばない
  - 迂回や不誠実な再作成は勧めない

## NG

- 原因候補だけ並べて、今やることを出さない
- 毎回ポリシー全体を長文で説明する
- `色々な原因があります` で止める
- 再審査前に新規アカウント作成を勧める
- 迂回や不正に見える運用を勧める
