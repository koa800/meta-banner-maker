## このスキルが解く問題

- CR企画が甲原に完全属人化しており、試行回数が上がらない
- AIがCR企画をできるようになれば、試行回数が劇的に増える
- 各市場・ターゲット層（AI / SNS / スキルプラス）それぞれでCPAを安く獲得できるCRを量産する
- 現状の課題: ROASが200-300%台に留まっている。顧客層の質を上げつつ、各市場でのCPA効率も最大化する必要がある

## トリガー

- 新しいCRが必要なとき（既存CRの疲弊、新ファネル、新訴求テスト）
- 週次のCR効果検証後（勝ちパターンの横展開として）
- 競合リサーチで新しい切り口が見つかったとき

## 手順

### Step 0: リサーチ（前工程）

> **リサーチは独立したスキル（B5）として分離。詳細は `Skills/1_広告/広告リサーチ.md` を参照。**

CR企画に入る前に、リサーチスキルから以下のインプットを受け取る:

- N1分析で掴んだインサイト（避けたいコスト / 満たしたい需要）
- 参考元となるCR（何を参考にするか＝フォーマット/フック/訴求/ビジュアルを明示）
- 自社の傾向（6つの訴求パターンの実績・未使用要素）
- 市場トレンド（旬なCR・高エンゲージ投稿）

### Step 1: CR企画の生成（AIが実行）

リサーチ結果をもとに、以下のフォーマットでCR企画を**3-5案**生成する。

#### 企画フォーマット（1案ごと）

```
【CR企画: [仮タイトル]】

■ 基本設計
- 訴求パターン: [A〜Eのどれ / 複合] ※F.権威は主軸にしない（短期効果のみ）
- ターゲット: [具体的に誰。「会社員」「フリーランス」等を明示]
- 集めたい顧客像: [この企画が引き寄せる人の特徴。スキルが身につく人か]
- ファネル: [AI / SNS / スキルプラス]
- フォーマット: [動画 / 画像] ※動画推奨（大当たり88%が動画）

■ コンセプト
- インサイト: [ターゲットの「避けたいコスト」と「満たしたい需要」]
- 独自性: [競合にない切り口は何か]
- ベネフィット: [行動すると何が得られるか]

■ 構成
- フック（冒頭3秒）: [感情を揺さぶる具体的なセリフ/ビジュアル]
- 本編（15-30秒）: [問題提起 → 解決策の提示]
- CTA: [具体的な行動喚起]

■ LP一貫性
- 遷移先LP: [どのLPに飛ばすか]
- CRとLPの一貫性タイプ: [統合型 / 平行型]
- CR→LPで回収する訴求: [CRで提示した約束をLPのどこで回収するか]

■ 事前評価
- 3つの壁チェック: 第1の壁[○/△/×] / 第2の壁[○/△/×] / 第3の壁[○/△/×]
- 大当たりパターン適合度: [高/中/低 + 理由]
- 想定リスク: [恐怖訴求過多で質の低い層を集めないか / 権威訴求で広告学習が歪まないか等]
```

### Step 2: 企画の自己評価（AIが実行）

生成した3-5案を以下の基準でスコアリングし、推奨順位をつける。

| 評価軸 | 重み | 判断基準 |
|--------|------|----------|
| ROAS貢献度 | ★★★ | この企画はROAS改善に貢献するか（CPA効率 × 後ろの数値を総合判断） |
| 大当たりパターン適合 | ★★★ | 実績あるA〜Eパターンのどれかに当てはまるか（F.権威は主軸にしない） |
| ターゲット明確さ | ★★ | 「誰向けか」が一目で分かるか。どの市場・ターゲット層を狙うか |
| フックの強さ | ★★ | 冒頭3秒で感情が動くか |
| LP一貫性 | ★★ | 遷移先LPとの整合性があるか |
| 新規性 | ★ | 既存CRと被っていない切り口か |

> **CPA効率と顧客質は二者択一ではない。** 各市場・ターゲット層（AI/SNS/スキルプラス）それぞれで「CPA安く + 後ろの数値も良い」CRを見つけるのがゴール。今のアドネスの課題として顧客質の改善が特に重要だが、CPAが安いことも各市場で必要。

```
【AIの推奨】

1位: [企画名] — 理由: [最もROAS貢献度が高いと判断した根拠]
2位: [企画名] — 理由: [...]
3位: [企画名] — 理由: [...]

※ [企画名] は顧客質のリスクあり: [具体的な懸念]
```

### Step 3: 甲原が承認

- 企画案を見て、承認 / 修正指示 / 却下
- 修正指示があればAIが即座に反映して再提案
- **甲原の修正内容を学習データとして蓄積**（どこが足りなかったか）

### Step 4: 制作への引き渡し

承認された企画を制作フォーマットに変換する。

- 動画CR → Phase 1（台本）を完成させ、Phase 2以降の制作指示書を作成
- 画像CR → コピー + デザイン方向性を作成
- 引き渡し先: 制作担当 or AIが直接制作（skill_cr_productionスキルと連携）

## アウトプット

- CR企画書（3-5案 / 推奨順位つき）
- 承認済み企画の制作指示書

## 承認ポイント

| ポイント | 内容 |
|---------|------|
| Step 2の推奨案 | 企画を甲原が承認してから制作に進む |
| 承認率の目標 | 現在0% → まず30%（修正なしで制作GOが出る率）→ 100%で承認不要に |
| 承認率の計測 | 「甲原が修正なしでGOを出した企画数 / 全提案数」を週次で記録 |
| 学習ループ | 甲原の修正内容を記録し、次回の企画生成に反映する |

## エラー時の対処

| エラー | 原因 | 対処 |
|--------|------|------|
| DProのデータ取得失敗 | ログイン切れ or UI変更 | Chrome MCP経由で再ログイン |
| 企画が似たり寄ったりになる | インプット不足 | リサーチ範囲を広げる（別業界・別媒体も参照） |
| 顧客質の評価ができない | 後ろの数値とCRの紐付けデータがない | kpi_summary.jsonの媒体×ファネル別データで代替 |

## 実装ステップ

### Phase 1: 企画の下書き生成（Lv.1）
- 既存のフレームワーク + 大当たりCR分析をプロンプトに組み込む
- AIが3-5案を生成 → 甲原が全て修正・改善
- **甲原の修正パターンを蓄積する**（これが最も重要）

### Phase 2: 自己評価つき提案（Lv.2）
- Phase 1で蓄積した修正パターンを反映
- AIが評価・推奨順位をつけて提案
- 承認率を計測開始

### Phase 3: リサーチ→企画の自律実行（Lv.3）
- DProリサーチを自動実行
- 勝ちパターンの横展開を自動提案
- 承認率80%以上で移行
