## このスキルが解く問題

- CR企画が甲原に完全属人化しており、試行回数が上がらない
- AIがCR企画をできるようになれば、試行回数が劇的に増える
- 各市場・ターゲット層（AI / SNS / スキルプラス）それぞれでCPAを安く獲得できるCRを量産する
- 現状の課題: ROASが200-300%台に留まっている。狙った属性比率を上げつつ、各市場でのCPA効率も最大化する必要がある
- 現在の最優先は、`スキルプラスを市場No.1に近づける施策` の中で `Meta広告の集客` を伸ばすこと
- 主因は `クリエイティブのクオリティ問題`。特に `高LTVになりそうな人` に刺さるクリエイティブ不足を重く見る
- `CTRとフック率は通るがオプトイン以降で失敗` が失敗側の多数を占めるため、企画では `フックの強さ` だけでなく `誰を集めるか / LPで何を回収するか` まで一体で設計する必要がある
- ここでいう `高LTVの人` は結果定義であり、面談定性比較の分析用しきい値そのものではない
- 広告実務で主語にするのは `高LTVになりそうな人` であり、現時点では `事業者・経営者` `AIを事業導入したい層` `過去にすでに何かを試している層` `既に仕事や事業の差し込み先がある層` を優先して増やす前提で扱う
- 全事業共通の `高LTVの人` は、`買った価値の投下先が実際にある人` を指す
- スキルプラス文脈の `高LTVの人` は、`学んだ内容の差し込み先が実際にある人` を指す
- スキルプラス文脈の `高LTVになりそうな人` は、`差し込み先の素地があり、教育で何に使うかが具体化する人` を指す

## 今回の役割分担

- AI: 既存データから勝ち筋候補を整理し、企画案と比較軸を出す
- 甲原: 狙う属性、勝ち筋、やらない訴求を最終決定する
- 制作担当またはAI: 承認済み企画を制作に落とす

## トリガー

- 新しいCRが必要なとき（既存CRの疲弊、新ファネル、新訴求テスト）
- 週次のCR効果検証後（勝ちパターンの横展開として）
- 競合リサーチで新しい切り口が見つかったとき

## 手順

### Step 0: リサーチ（前工程）

> **リサーチは独立したスキル（B5）として分離。詳細は `Skills/1_広告/広告リサーチ.md` を参照。**

CR企画に入る前に、リサーチスキルから以下のインプットを受け取る:

- N1分析で掴んだインサイト（避けたいコスト / 満たしたい需要）
- 参考元となるCR（何を参考にするか＝フォーマット/フック/訴求/ビジュアルを明示）
- 自社の傾向（6つの訴求パターンの実績・未使用要素）
- 市場トレンド（旬なCR・高エンゲージ投稿）

#### `高LTVになりそうな人` を見る観点

| 判定軸 | 見ること | GO寄り | 再設計寄り |
|--------|----------|--------|------------|
| 差し込み先の素地 | 既に何へ使う現実があるか | 業務 / 事業 / 顧客 / 発信 / 商品に入れる話 | 使い道が未来の願望だけ |
| 今変えたい対象 | 今ある何を変えたいか見えるか | 効率化 / 集客 / 営業 / 顧客対応 / コンテンツ改善 | 漠然と稼ぎたい / 自由になりたい |
| 実装前提 | 学習の先に実装があるか | 導入 / 改善 / 回す / 仕組み化 | 学ぶ / 知る / もらうで止まる |
| 過去試行 | 既に何か試した課題があるか | 触ったが伸びない / 運用中だが詰まる | 何もしていない初心者広取り |
| 教育余地 | LP以降で何に使うか具体化できるか | 広告で興味、LPで接続先を明確化できる | 広告でもLPでも使い道が曖昧 |

- `差し込み先の素地` が弱い企画は、フックが強くても下流崩れ候補として扱う
- `今変えたい対象` と `教育で具体化する余地` が弱い企画は、再設計前提で扱う

#### 5軸を頭の中で見る前提

- 5軸は採点表として運用するものではなく、企画の質を見抜くための内部基準として使う
- 特に `差し込み先の素地` が弱い企画は原則見送り寄りで扱う

#### レビューの実行順

1. その人は、今どの現実にAIを差し込めるか
2. その人は、今何を変えたいのか
3. このCRは、学習より実装を想起させるか
4. このCRは、何を試した人向けなのか
5. このCRの続きをLPで見たら、何に使うかが自然に具体化するか

#### 表現チェック

- `AIを学ぶ` で終わるなら弱い
- `AIを既存の何かに入れる` まで言えていれば強い
- `稼げる` 単体は弱い
- `今の顧客 / 集客 / 業務 / 事業をどう変えるか` まで続けば強い

### Step 1: CR企画の生成（AIが実行）

リサーチ結果をもとに、以下のフォーマットでCR企画を**3-5案**生成する。

#### 企画フォーマット（1案ごと）

```
【CR企画: [仮タイトル]】

■ 基本設計
- 訴求パターン: [A〜Eのどれ / 複合] ※F.権威は主軸にしない（短期効果のみ）
- ターゲット: [具体的に誰。「会社員」「フリーランス」等を明示]
- 集めたい顧客像: [この企画が引き寄せる人の特徴。スキルが身につく人か]
- 狙いたい属性: [高LTVになりそうな人 / 成果が出やすい / 継続しやすいなど、狙うべき人の条件]
- 避けたい属性: [広く反応するが下流でズレやすい人。権威だけに反応する層など]
- 差し込み先の素地: [既にある仕事 / 事業 / 顧客 / 発信 / 商品のどこに入りうるか]
- 今変えたい対象: [何を改善したい人を集める企画か]
- 教育で具体化する余地: [広告では未確定でも、LPや販売で何に使うかを明確化できるか]
- ファネル: [AI / SNS / スキルプラス]
- フォーマット: [動画 / 画像] ※動画推奨（大当たり88%が動画）

■ コンセプト
- インサイト: [ターゲットの「避けたいコスト」と「満たしたい需要」]
- 独自性: [競合にない切り口は何か]
- ベネフィット: [行動すると何が得られるか]

■ 構成
- フック（冒頭3秒）: [感情を揺さぶる具体的なセリフ/ビジュアル]
- 本編（15-30秒）: [問題提起 → 解決策の提示]
- CTA: [具体的な行動喚起]

■ LP一貫性
- 遷移先LP: [どのLPに飛ばすか]
- CRとLPの一貫性タイプ: [統合型 / 平行型]
- CR→LPで回収する訴求: [CRで提示した約束をLPのどこで回収するか]

■ 事前評価
- 3つの壁チェック: 第1の壁[○/△/×] / 第2の壁[○/△/×] / 第3の壁[○/△/×]
- 大当たりパターン適合度: [高/中/低 + 理由]
- 下流適合の仮説: [LP・オファー・CSまで見たときに、狙う属性が下流へ進みそうか]
- 5軸観察メモ: [差し込み先 / 今変えたい対象 / 実装前提 / 過去試行 / 教育余地 をどう見たか]
- 想定リスク: [恐怖訴求過多で狙っていない属性を集めないか / 権威訴求で広告学習が歪まないか等]
```

### Step 2: 企画の自己評価（AIが実行）

生成した3-5案を以下の基準で比較評価し、推奨順位をつける。

| 評価軸 | 重み | 判断基準 |
|--------|------|----------|
| ROAS貢献度 | ★★★ | この企画はROAS改善に貢献するか（CPA効率 × 後ろの数値を総合判断） |
| 狙った属性適合 | ★★★ | 高LTVになりそうな人に寄るか。5軸で見て差し込み先の素地が弱くないか |
| 大当たりパターン適合 | ★★★ | 実績あるA〜Eパターンのどれかに当てはまるか（F.権威は主軸にしない） |
| ターゲット明確さ | ★★ | 「誰向けか」が一目で分かるか。どの市場・ターゲット層を狙うか |
| フックの強さ | ★★ | 冒頭3秒で感情が動くか |
| LP一貫性 | ★★ | 遷移先LPとの整合性があるか |
| 新規性 | ★ | 既存CRと被っていない切り口か |

> **CPA効率と狙った属性比率は二者択一ではない。** 各市場・ターゲット層（AI/SNS/スキルプラス）それぞれで「CPA安く + 後ろの数値も良い」CRを見つけるのがゴール。今のアドネスの課題として狙った属性比率の改善が特に重要だが、CPAが安いことも各市場で必要。
> **フックが強いだけで上位にしない。** 失敗側では `CTRとフック率は通るがオプトイン以降で失敗` が多数のため、`誰を集めるか` と `CR→LPの期待一致` を同じ重みで見る。

```
【AIの推奨】

1位: [企画名] — 理由: [最もROAS貢献度が高いと判断した根拠]
2位: [企画名] — 理由: [...]
3位: [企画名] — 理由: [...]

※ [企画名] は属性ズレのリスクあり: [具体的な懸念]
```

### Step 3: 甲原が承認

- 企画案を見て、承認 / 修正指示 / 却下
- 修正指示があればAIが即座に反映して再提案
- **甲原の修正内容を学習データとして蓄積**（どこが足りなかったか）

### Step 4: 制作への引き渡し

承認された企画を制作フォーマットに変換する。

- 動画CR → Phase 1（台本）を完成させ、Phase 2以降の制作指示書を作成
- 画像CR → コピー + デザイン方向性を作成
- 引き渡し先: 制作担当 or AIが直接制作（skill_cr_productionスキルと連携）

## アウトプット

- CR企画書（3-5案 / 推奨順位つき）
- 承認済み企画の制作指示書

## 承認ポイント

| ポイント | 内容 |
|---------|------|
| Step 2の推奨案 | 企画を甲原が承認してから制作に進む |
| 承認率の目標 | 現在0% → まず30%（修正なしで制作GOが出る率）→ 100%で承認不要に |
| 承認率の計測 | 「甲原が修正なしでGOを出した企画数 / 全提案数」を週次で記録 |
| 学習ループ | 甲原の修正内容を記録し、次回の企画生成に反映する |

## エラー時の対処

| エラー | 原因 | 対処 |
|--------|------|------|
| DProのデータ取得失敗 | ログイン切れ or UI変更 | Chrome MCP経由で再ログイン |
| 企画が似たり寄ったりになる | インプット不足 | リサーチ範囲を広げる（別業界・別媒体も参照） |
| 属性ズレの評価ができない | 後ろの数値とCRの紐付けデータがない | kpi_summary.jsonの媒体×ファネル別データで代替 |

## 実装ステップ

### Phase 1: 企画の下書き生成（Lv.1）
- 既存のフレームワーク + 大当たりCR分析をプロンプトに組み込む
- AIが3-5案を生成 → 甲原が全て修正・改善
- **甲原の修正パターンを蓄積する**（これが最も重要）

### Phase 2: 自己評価つき提案（Lv.2）
- Phase 1で蓄積した修正パターンを反映
- AIが評価・推奨順位をつけて提案
- 承認率を計測開始

### Phase 3: リサーチ→企画の自律実行（Lv.3）
- DProリサーチを自動実行
- 勝ちパターンの横展開を自動提案
- 承認率80%以上で移行
