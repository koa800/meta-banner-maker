# LTV CPA Market Priority

1つの商品で取りに行ける市場が複数ある時は、`広い市場` や `取りやすい市場` から順に追わない。

優先するのは、`その市場を取った時に、どれだけ大きな価値が残るか` と `そこに到達するまでの獲得コストがどれだけ軽いか` の両方が良い市場である。

この skill は、市場、顧客セグメント、広告訴求、流入元、オファー入口を `LTV x CPA` で優先順位づけする時に使う。

## Core Principle

- 市場選定は `母数` ではなく `経済合理性` で決める。
- `LTV` は高いほど良い。`CPA` は低いほど良い。
- `LTVだけ`、`CPAだけ` では決めない。必ず2軸で見る。
- 優先順位の目的は、最初に勝ちやすい市場ではなく、`利益が積み上がりやすい市場` から取りに行くことである。
- 同じ商品でも、市場ごとに `LTV` と `CPA` は変わる前提で扱う。
- 一度決めた市場順位は固定しない。実データが出たら、再分類して更新する。

## Required Variables

優先順位づけの前に、最低でも次を揃える。

- `対象事業`
- `対象商品`
- `市場候補` または `セグメント候補`
- 各候補の `LTV` の見立て
- 各候補の `CPA` の見立て
- 見立ての根拠が `実データ / 強い推定 / 仮説` のどれか

`LTV` は単価だけでなく、継続率、アップセル余地、返金率、紹介の出やすさまで含めて見る。

`CPA` は広告入札の高さだけでなく、教育コスト、訴求難易度、競争環境、成約までに必要な説明量まで含めて見る。

`高 / 中 / 低` の判定基準は事業ごとに変わるため、毎回先に定義する。

同じ `高LTV` でも、別事業では基準値が全く違ってよい。事業をまたいで同じ閾値を流用しない。

まず `この事業では何円以上を高LTVと置くか`、`この事業では何円以下を低CPAと置くか` を決め、その定義の中で市場同士を比較する。

## Ranking Rule

まず各市場を `高 / 中 / 低` に3段階で置く。

- `LTV`: 高 = 3, 中 = 2, 低 = 1
- `CPA`: 低 = 3, 中 = 2, 高 = 1

次に `市場スコア = LTV点 + CPA点` で並べる。

同点なら `LTVが高い方` を上に置く。

このルールで並べると、優先順位は次になる。

1. `高LTV x 低CPA`
2. `高LTV x 中CPA`
3. `中LTV x 低CPA`
4. `高LTV x 高CPA`
5. `中LTV x 中CPA`
6. `低LTV x 低CPA`
7. `中LTV x 高CPA`
8. `低LTV x 中CPA`
9. `低LTV x 高CPA`

この並びの意味は、`LTVを強く重視しつつ、CPAの差で前後を調整する` ことである。

## Workflow

1. `1商品` に対して `市場候補` を漏れなく並べる。
2. 各市場について `LTV` を `高 / 中 / 低` で仮置きする。
3. 各市場について `CPA` を `低 / 中 / 高` で仮置きする。
4. 各市場を `9象限` のどこに入るか決める。
5. 上の優先順位に従って、狙う順番を並べる。
6. 予算、訴求、制作工数、改善工数を上位市場から先に配分する。
7. 実運用後は `実LTV` と `実CPA` で再評価し、順位を更新する。

## Output Format

市場選定の提案や整理を出す時は、内部的に次の順でまとめる。

- `対象商品`
- `市場候補一覧`
- `各市場のLTV判定`
- `各市場のCPA判定`
- `根拠の確度`
- `優先順位`
- `最初に取りに行く市場`
- `後回しにする市場`
- `次に打つ施策`

## Anti-Patterns

- `CPAが安い` だけで優先市場にする。
- `LTVが高そう` だけで、獲得難易度を無視する。
- 同じ商品なのに、市場ごとの差を見ずに一括で扱う。
- 母数が大きい市場を、経済合理性より先に追う。
- 数字がない時に、確度を分けず全部を確定扱いする。
- 低LTV市場でCPAだけを改善し続け、全体利益を毀損する。

## Quality Check

- 各市場を `LTV x CPA` の両方で見たか。
- `LTV` に継続、アップセル、返金率まで含めたか。
- `CPA` に教育コストや説明難易度まで含めたか。
- 根拠の確度を分けたか。
- 予算と工数を本当に上位市場から配分する設計になっているか。
- `安く取れる市場` ではなく `利益が積み上がる市場` を先にしているか。
