# Movement Model

## Core Idea

人は自分の意思で動いているように感じるが、実際には何に注意が向き、どう認識し、どの選択肢が自分にとって良く見えるかによって動く。

実務上のモデルは次の通り。

1. `意識 / 注意`
2. `欲求`
3. `目的`
4. `現在地とのギャップ`
5. `問題認識`
6. `解決策探索`
7. `行動`

## Definitions

- `欲求`: 人生をより良くしたいという広い方向性。
- `目的`: 欲求を言葉にして具体化したもの。
- `問題`: 現在地と目的のギャップ。
- `解決策`: そのギャップを埋めると認識される手段。

## Practical Implication

送り手の依頼から書き始めない。

弱い例:
- 「これをやってください」

強い例:
- 「こうなりたいはずですよね」
- 「今はここが詰まっていますよね」
- 「だからこの一歩が有効です」

## Writing Checklist

- 相手のより良い未来を言葉にする。
- 今の詰まりや不足を見える形にする。
- 行動と問題解決を直結させる。
- 次の一歩を一つに絞る。
- 認識や行動に寄与しない文を削る。

## Short Example

弱い依頼:
- 「明日までに資料を提出してください。」

改善後:
- 「明日の打ち合わせで判断を前に進めるために、今日中にこの3点だけ埋めてもらえると助かります。ここが埋まると、その場で意思決定まで進められます。」

## Use Cases

このモデルは、依頼、マネジメント、採用、営業、コピーライティング、オンボーディング、UI導線など、人の行動を設計したい場面全般に使える。
