# MPT Page Design

ページを `読める資料` ではなく `次の手が動くインターフェース` として設計する。

必要に応じて、厚い原理は [../デザイン.md](../デザイン.md) を、公開運用は [../../6_システム/公開ページ運用.md](../../6_%E3%82%B7%E3%82%B9%E3%83%86%E3%83%A0/%E5%85%AC%E9%96%8B%E3%83%9A%E3%83%BC%E3%82%B8%E9%81%8B%E7%94%A8.md) を読む。

## Use This Skill When

- LP、共有ページ、修正ガイド、確認メモ、提案ページなどを設計・改善するとき
- 情報はあるのに、人が見たときに何から見ればいいか分かりにくいとき
- `MPT`、情報階層、余白、カード構造、開閉UI、PC/モバイル最適化を見直したいとき
- Apple や Mercury のように、静かだが洗練された質感へ寄せたいとき

## Core Principles

- **目的先行**: まず `このページは何を達成するか` を一文で固定する
- **意図**: すべての機能と装飾は `何のためにあるか` を説明できるものだけ残す
- **MPT**: 情報を減らすのではなく、必要な意味を削らず短時間で理解できる形に圧縮する
- **判断優先**: 最初の1画面で `何のページか / 何が問題か / どう動くか` を分からせる
- **表より判断単位**: 一覧性より修正速度が重要なら、表よりカードや accordion を優先する
- **Progressive Disclosure**: 詳細は後ろへ下げてよいが、初期表示で核となる情報は隠さない
- **PCとモバイルは別設計**: ただ縮めるのではなく、横幅、列数、固定UIを別々に調整する
- **演出より証拠**: 根拠セクションはモックより事実、実スクリーンショット、確認ログを優先する
- **洗練は秩序**: Apple 的な質感は、派手さではなく整列、余白、静かな面の重なりで作る

## Workflow

1. ページの `目的` と `見た人に取ってほしい行動` を一文ずつ書く
2. ページに残すべき `必要な意味` を列挙し、重複を消す
3. 各要素に `意図` を付け、説明できない要素を削る
4. ファーストビューを `ページの意味 / 今日の判断 / 問題の場所 / 直し方` の順で組む
5. 情報を `1件ごとの判断単位` に切り、表・カード・accordion のどれが速いかで選ぶ
6. 強い色、太い文字、固定UIは最小限にし、優先度差だけを視覚化する
7. PCでは `max-width` と列設計、mobile では `1列化` と `固定UIの弱さ` を先に見る
8. 証拠は `実際に確認した事実` に置き換え、フェイクな演出を減らす
9. PC / mobile のスクリーンショットを撮り、`資料に見える箇所` と `迷う箇所` を洗い直す

## Design Heuristics

- 見出しは、雰囲気ではなく結論を書く
- 小見出しと大見出しで同じことを繰り返さない
- 濃い背景色は本当に重要な面だけに使う
- 数値は比較しやすい位置にまとめる
- 余白は装飾ではなく `意味の境界線` として使う
- sticky は便利でも邪魔なら失敗。特に mobile で厳しく見る
- 開閉UIは便利でも、開かないと意味が分からない構造にしない

## Anti-Patterns

- 便利そうだからと補助機能を足す
- 同じ意味を見出し、本文、表で何度も読ませる
- モバイルだけ整っていて、PCで間延びする
- 強い色面を多用して可読性を落とす
- 初期表示で hero や結論をアニメーション待ちにする
- 実証が必要な箇所を、雰囲気だけのモックで済ませる

## Output Format

この skill を使ってページ改善案を出すときは、内部的に次の順で整理する。

- `目的`
- `見た人に取ってほしい行動`
- `残す要素とその意図`
- `削る要素とその理由`
- `ファーストビューの再構成`
- `本文ブロックの再構成`
- `PCでの調整点`
- `モバイルでの調整点`
- `まだ120点に届かない差分`

## Quality Check

- 目的に効かない機能や装飾が残っていないか
- 最初の1画面で何を見るべきか分かるか
- 重要な情報が detail / accordion の内側に隠れていないか
- 表よりカードの方が速い場面で、まだ表に頼っていないか
- PCで横に広がりすぎていないか
- mobile で固定UIが本文の邪魔をしていないか
- 根拠が `演出` ではなく `確認した事実` になっているか
