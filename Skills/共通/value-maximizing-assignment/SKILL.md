# Value Maximizing Assignment

人にバリューを出してもらいたい時は、`空いている人に渡す` ではなく、`どのボールならこの人が最も強くなり、価値が積み上がり、適性も合うか` で決める。

この skill は、タスク委譲、役割設計、担当替え、採用後の配置、1on1 での再設計に使う。

## Core Principles

- 優先順位は常に `条件1: こだわれるか` → `条件2: 価値が残りやすいか` → `条件3: 適性が合うか` の順に見る。
- 条件1が弱いボールは、条件2と条件3が良くても原則そのまま渡さない。まずボールの切り方を変える。
- `こだわり` とは思い込みではなく、実際にその領域で一番を取りにいけるだけの反復、判断精度、改善量が出る状態を指す。
- `価値が残りやすい領域` とは、その人が離れても判断基準、資産、仕組み、成果物が残り、次の価値につながる領域を指す。
- `適性` は最後に見る。適性があっても、こだわれない領域や価値が残らない領域を主戦場にしない。
- 人に合わせて仕事を丸ごと受け入れるのではなく、価値が出る単位にボールを切り直してから渡す。

## Condition 1: Let Them Become the Strongest

最初に見るのは、その人がその領域で本当に強くなれるかである。

- その人が細部にこだわり続けられるか。
- 反復回数が十分にあり、判断精度が上がるか。
- 成果の良し悪しが見え、改善ループが回るか。
- 周辺ではなく中核の判断に触れられるか。
- `自分の領域は自分が一番だ` と事実ベースで言える未来があるか。

ここで弱いなら、その人を変える前にボールを変える。

## Condition 2: Hand Over Durable Value

次に、そのボールが将来にも価値を残すかを確認する。

- 仕組み、テンプレ、判断基準、資産、再利用可能な成果物が残るか。
- 一回きりの処理ではなく、次の速度や品質を上げるか。
- 属人的な対応で終わらず、他者や次回にも効くか。
- 目先の消火より、後から効いてくる構造改善に近いか。

同じ人に任せるなら、`消える作業` より `残る作業` を優先する。

## Condition 3: Match Aptitude

最後に、その人の適性と噛み合うかを見る。

- 認知特性、思考の粒度、スピード感に合うか。
- 苦なく高水準を維持できる型か。
- 本人の強みが、その領域の勝ち筋と一致しているか。

適性は重要だが、優先順位は3番目である。適性だけで任せ先を決めない。

## Workflow

1. 渡そうとしている `ボール` を一文で定義する。
   `ボール = 何を持つか + 何を判断するか + 何を残すか`
2. そのボールのゴールを固定する。
   例: ただ処理するのではなく、仕組み化まで持つのか、品質責任まで持つのかを明確にする。
3. 条件1を判定する。
   `この人はこの領域で本当に強くなれるか。こだわりが積み上がるか。`
4. 条件1が弱ければ、担当者選びを続ける前にボールを切り直す。
   判断単位、責任範囲、改善余地が見える形に再設計する。
5. 条件2を判定する。
   `このボールは価値が残るか。将来の速度と品質を上げるか。`
6. 条件2が弱ければ、同じ目的でより残る領域に寄せられないか考える。
   例: 単発対応よりテンプレ化、都度判断より基準化。
7. 条件3を判定する。
   `この人の得意な型と噛み合うか。`
8. `誰に渡すか` だけでなく、`どの単位で渡すか` と `どこまで持たせるか` を決める。
9. 最後に、利用者目線で `この配置で本当に価値が最大化されるか` を見直す。

## Output Format

この skill を使って配置や委譲を提案する時は、内部的に次の順で整理する。

- `ゴール`
- `渡したいボール`
- `条件1の評価`
- `条件2の評価`
- `条件3の評価`
- `推奨アサイン`
- `必要ならボールの切り直し案`
- `今すぐ渡すべき次の一手`

## Anti-Patterns

- 空いている人、手が早い人、文句を言わない人にそのまま渡す。
- こだわれない領域を、根性や責任感でカバーさせる。
- 消える作業ばかりをエースに集める。
- 適性だけで任せ先を決め、価値の残りやすさを見ない。
- `誰に渡すか` だけ考え、`どんな単位で渡すか` を設計しない。
- 本人の希望をそのまま採用し、実際に一番を取りにいけるかを検証しない。

## Quality Check

- 条件1を最優先で見たか。
- その人が中核の判断に触れられる設計になっているか。
- 一回きりで消える仕事ではなく、何かが残るか。
- 適性は見たが、適性だけで決めていないか。
- `人を合わせる` ではなく `ボールを設計する` 発想になっているか。
- 配置後に、その人の強さがさらに積み上がる未来が見えるか。
