## 決済データ（加工）

- **シート名**: 【アドネス株式会社】スキルプラス着金データ（加工）

- **タブ**: `日別売上数値` / `月次照合` / `売上サマリー` / `データソース管理` / `データ追加ルール`
- **役割**: スキルプラス事業の決済 raw を、着金売上・返金額・解約請求回収額の日別正本へ再生成する加工シート
- **主なルール**:
  - `日別売上数値` は `日付 / 新規着金売上 / 分割回収売上 / 継続課金売上 / 会員向け単発売上 / 着金売上 / 返金額 / 純着金売上 / 解約請求回収額 / 新規着金件数 / 分割回収件数 / 継続課金件数 / 会員向け単発件数 / 着金件数 / 返金件数 / 解約請求回収件数`
  - `月次照合` は `月 / 指標 / 加工値 / 元値 / 差分 / 判定 / 補足` を持ち、月ごとの金額・件数がソース集計とゼロ差分かを確認する
  - `新規着金売上` は、非会員向け商品の成功売上、`頭金お支払い`、`日本プラム / CBS / 京都信販 / きらぼし銀行 / CREDIX` の初回承認・振込を集計する
  - `分割回収売上` は、スキルプラス本体の分割金回収を集計する。UnivaPay の空欄 recurring のうち、分割金として確定した金額帯に加え、動画広告分析Pro の `初回3ヶ月間` もここへ寄せる
  - `継続課金売上` は、商品マスタの `購入形態 = 月額`、または UTAGE detail で継続課金と exact に確認できた売上を集計する。`センサーズ継続 / AIカレッジ継続 / スキルプラス継続利用 = 21,780円/月`。動画広告分析Pro は `1媒体 = 4,980円 x 3回 -> 9,800円/月`、`3媒体 = 14,940円 x 3回 -> 29,400円/月`
  - `会員向け単発売上` は、会員向けかつ買切り/イベント系の商品を集計する。`プライム合宿（59,800円）`、`スキルプラスイベント`、`みかみとお茶会` など
  - `プライム合宿（非会員向け）` は `148,000円` だけを対象にし、`新規着金売上` へ入れる
  - `スキルプラスイベント` は、名前付きの独立イベント以外をまとめる共通商品として扱う
  - `みかみとお茶会` は `スキルプラスイベント` にまとめず、別商品として扱う
  - `着金売上` は `新規着金売上 + 分割回収売上 + 継続課金売上 + 会員向け単発売上`
  - `返金額` は `お客様相談窓口_進捗管理シート` を案件正本にし、決済データ raw は突合と補完にだけ使う
  - `商品名空欄` のうち `日本プラム / きらぼし銀行 / CBS / 京都信販 / CREDIX` の成功売上は、`スキルプラス受講生データ（元）` の一次データで確認でき、かつ `他事業の成約リスト` と競合しないものだけを `新規着金売上` へ含める
  - `UnivaPay` の `頭金お支払い` は過去運用上 100% スキルプラス販売として `新規着金売上` に含める
  - `UnivaPay / INVOY` の商品名空欄も、`他事業の成約リスト` と競合せず、除外根拠がなければスキルプラス事業として扱う。UnivaPay は `頭金お支払い = 新規`、`分割金` と確定した recurring は `分割回収売上`、それ以外の exact に継続課金と確認できたものは `継続課金売上` へ振る
  - `raw 決済の exact duplicate` は加工時点で除外する。同一行が複数回収集されていても、加工では1件だけ採用する
  - `スキルプラスではないと分かっている売上` と `きらぼし銀行の明らかなゲートウェイ精算入金` は `除外確定` として扱い、売上にも unknown 監視にも入れない
  - `他事業の成約リスト` は負の証拠として使う
    - `Ver.5❶管理シート_260208 / 成約済み`
    - `Ver.5❶管理シート_260208 / ｲﾚｷﾞｭﾗｰ`
    - `【アドネス＆代理店】統合営業管理2026年度 / 全社_クライアント台帳`
    - `法人事業部管理シート / 受注クライアント管理`
  - `payment_daily_sync.py` の最後で毎回再生成するため、Drive に新規CSVがない run でも返金更新は反映される
