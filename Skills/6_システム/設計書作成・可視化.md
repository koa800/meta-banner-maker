## 手順

### Step 1: 要件の整理

- テーマから関連ファイルを検索（Project/, Skills/, Master/）
- 既存の設計や類似パターンがあれば参照する
- 不明点があれば甲原さんに確認（選択肢+自分の意見を添える）

### Step 2: 設計書の作成（markdown）

- 配置先: `Project/` 配下の適切なフォルダ
- 含める内容:
  - **完成イメージ**: ユーザー体験のステップ（ASCII図で視覚的に）
  - **仕組みの全体図**: システム間の連携フロー
  - **作るもの一覧**: 何を / どこで / いくつ
  - **作業手順**: ステップバイステップ
  - **運用手順**: 定常的に必要な作業（あれば）
- 粒度: 全体図レベル。画像素材リスト等の詳細は不要（聞かれたら追記）

### Step 3: Miroボード準備

- **URL指定あり** → そのボードを使用
- **ボード名指定あり** → REST APIで新規作成（名前をそのまま使用）
- **指定なし** → REST APIで新規作成（設計テーマ名をボード名にする）

**ボード新規作成方法（REST API）:**
```bash
curl -X POST "https://api.miro.com/v2/boards" \
  -H "Authorization: Bearer $(cat System/config/miro.json | python3 -c 'import sys,json;print(json.load(sys.stdin)["access_token"])')" \
  -H "Content-Type: application/json" \
  -d '{"name":"ボード名"}'
```
→ レスポンスの `id` を以降の Miro MCP ツールの `board_id` に使用

### Step 4: Miro可視化

使用ツール: Miro MCP

**作るもの:**

| コンテンツ | Miroツール | 配置 |
|-----------|-----------|------|
| 全体フロー図 | diagram_create（flowchart） | 左側 |
| 担当者の作業リスト | doc_create | 右上 |
| 運用手順（あれば） | doc_create | 右下 |
| 分類テーブル（あれば） | table_create + table_sync_rows | フロー図の下 |

**フロー図のルール:**
- クラスター（cluster）で関連ステップをグルーピング（例: LINE画面 / UTAGE / Lステップ）
- 色分け: デフォルトパレット（黄=#fff6b6 / 青=#c6dcff / 緑=#adf0c7 / 紫=#dedaff）
- 判断ノード=青、開始・終了=緑、その他=黄

**作業リストのルール:**
- 担当者がいれば「○○さんの作業リスト」として作成
- 番号付きで手順を明記
- 最重要ポイントを末尾に強調

### Step 5: 共有

- MiroボードのURLを甲原さんに共有
- 変更点のサマリーを添える

## アウトプット

1. `Project/*.md` — 設計書（markdown）
2. Miroボード — フロー図 + 作業リスト + テーブル
3. 共有URL — 甲原さん（+ 実装担当者がいればその人にも）

## 承認ポイント

- Step 2完了後: 設計の方向性が合っているか甲原さんに確認
- Step 3完了後: Miro URLを共有 → 修正フィードバックがあれば反映

## エラー時の対処

| エラー | 原因 | 対処 |
|--------|------|------|
| Miro MCP接続失敗 | MCP認証切れ | `/mcp` で再認証 |
| REST APIで401 | トークン期限切れ | アプリ設定ページで再取得 → `System/config/miro.json` 更新 |
| ボードが見つからない | URL不正 or 権限なし | 甲原さんにURL再確認 |
| diagram_createで図が崩れる | DSLの構文エラー | DSL仕様を再取得して修正 |
