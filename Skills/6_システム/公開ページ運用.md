# 公開ページ運用

最終更新: 2026-03-09

## 目的

LP、修正ガイド、確認メモのようなHTMLを、`file://` ではなくURLでそのまま共有できる状態にする。

## 使うもの

- 公開先: GitHub Pages
- 公開用リポジトリ: `koa800/share-pages`
- 公開URLの基本形: `https://koa800.github.io/share-pages/<slug>/`
- 公開スクリプト: `System/scripts/publish_public_page.py`
- ページたたき台: `System/public_pages/page_template.html`

## この運用を使う場面

- LPや案内ページをURLで共有したいとき
- 修正案や確認メモを、開閉UIごと共有したいとき
- 画像ではなく、後から更新できるページとして持ちたいとき

## 使わない方がいい場面

- 外部公開してはいけない情報が入っているとき
- KPIや個人情報のように、URLを知っている人に見えてはいけない情報が含まれるとき

## 作り方

1. 元になるHTMLを用意する
   - 単体のHTMLならそのままでよい
   - 画像やCSSを使うなら `index.html` を含むディレクトリで作る
2. 必要なら `System/public_pages/page_template.html` を複製して土台にする
3. 以下の形式で公開スクリプトを実行する

```bash
python3 System/scripts/publish_public_page.py \
  --source "/絶対パス/公開したいHTMLまたはディレクトリ" \
  --slug "share-page-slug" \
  --title "共有ページのタイトル" \
  --summary "一覧で見せる短い説明"
```

4. 出てきた `公開URL` をそのまま共有する

## ページ設計の基準

- すべての機能と装飾に、目的に対する意図を持たせる
- 「便利そうだから」で機能を足さない。今回の目的に効く理由を一文で説明できるものだけ残す
- 最初の1画面で「何のページか」「何を見ればいいか」が分かるようにする
- 最初の1画面で、`問題 / 今日見る場所 / 直し方` の3点まで入るのが理想
- 実務ページでも、`読ませる資料` ではなく `そのまま作業に入れるUI` を目指す
- 同じ意味の要素は同じ見た目で反復する
- 小見出しと大見出しで同じ言葉を繰り返さない
- 強い背景色は最重要箇所だけに使う
- 開閉UIは、文言を増やしすぎず、プラス記号だけで伝わる形を優先する
- 長文ページでは、本文の行長と行間を先に整える
  - 行長の目安: 66ch 前後
  - 行間の目安: 1.5 以上
- 共有前に、PC幅とスマホ幅の両方で崩れていないか確認する
- 1スクロールごとにテーマを1つに絞り、MPTを落とさない
- モバイルでは、目次やナビを常時張り付かせすぎない。本文の邪魔をしないことを優先する
- 長文の共有ページは、mobile では `目次` の disclosure にし、desktop では sticky でもよい
- hero や結論は、初期表示で隠さない。scroll reveal や fade-in で最初の理解を遅らせない
- 表は一覧には強いが、修正判断には弱い。修正単位が少ないときはカードを優先する
- URL補助機能は必須ではない。`開く` や `コピー` は、作業速度を本当に上げる理由がある場合だけ入れる
- クリエイティブな質感は、濃い色や装飾量ではなく、余白、面の重なり、静かなグラデーションで足す

## 命名ルール

- `slug` は英数字とハイフンだけにする
- URLを見ただけで中身が分かる短い名前にする
- 例
  - `stopped-line-link-guide`
  - `skillplus-review`
  - `campaign-fix-202603`

## 運用ルール

- 公開ページの中に秘密情報を入れない
- 同じページを更新するときは、同じ `slug` を再利用する
- LINEやチャットに送るときは `公開URL` を送る
- `short.io` はこの公開フローには必須ではない
- 公開後は、ローカル確認だけで終わらず、公開URLでもPC幅とスマホ幅を確認する

## 確認コマンド

共有前の最低確認は、PC幅とスマホ幅のスクリーンショットを1枚ずつ取ること。

```bash
npx playwright screenshot --browser=chromium --wait-for-timeout=3000 \
  --viewport-size="1400,1800" "https://公開URL" "/tmp/page-desktop.png"

npx playwright screenshot --browser=chromium --wait-for-timeout=3000 \
  --viewport-size="390,1600" "https://公開URL" "/tmp/page-mobile.png"
```

ローカルHTMLを確認したい場合は、URLの部分を `file:///絶対パス/index.html` に置き換える。

## 補足

- 公開用リポジトリは配信箱として使う。ページの正本は `cursor/` 側に置く
- たたき台には、上部ナビ、ヒーロー、数値カード、開閉セクションの基本構造を入れてある
