# 動画広告クリエイティブ制作フロー

## このワークフローについて

動画広告CRを作りたいときは、このフローに沿って進める。

**注意：100%完全なものではない。以下を意識すること。**

1. ブラッシュアップ前提でこのワークフローを運用する
2. 常に改善していく必要がある

---

## Phase 1: 企画・構成

### 1-1. 参考動画の分析
- 参考にしたい動画を共有
- 動画のスタイル、テンポ、構成を分析

### 1-2. 目的・ゴールの明確化
- 広告の最終目的を定義（例: LP誘導、ウェビナー登録）
- ターゲット、訴求ポイントを整理

### 1-3. 台本・構成表の作成
- シーンごとの構成（フック → 本編 → 結論 → CTA）
- 各シーンのテロップ、狙い、演出を整理
- Googleドキュメント等で管理

---

## Phase 2: 素材制作

### 2-1. 画像生成（AI）
- **ツール**: Lovart（Nano Banana Proモデル）
- 各シーンのプロンプトを作成
- スタイルの一貫性を保つ（キャラクター、雰囲気）
- 生成後、番号付きでリネーム（01_シーン名.png）

### 2-2. 動画生成（AI）
- **ツール**: Vidu（img2video）
- 画像をアップロードし、モーションプロンプトで動画化
- 推奨設定: 4秒 or 8秒、1080p、H265
- 生成後、番号付きでリネーム（01_シーン名.mp4）

### 2-3. 参考動画のダウンロード
- **ツール**: yt-dlp
```bash
yt-dlp "動画URL" -o "参考動画.%(ext)s"
```

### 2-4. 音声分離（参考動画から）
- **ツール**: Fish Audio（オーディオ分離機能）
- 参考動画からボーカルとBGMを分離
- 声のトーン参考、BGM素材として活用

### 2-5. ナレーション作成（ボイスクローン）
- **ツール**: Fish Audio（ボイスクローン機能）
- 分離したボーカルで声をクローン
- 台本テキストを入力してナレーション生成

---

## Phase 3: ラフ動画制作

### 3-1. 素材の結合（ffmpeg）
```bash
# 動画結合
ffmpeg -f concat -safe 0 -i concat_list.txt -c:v libx264 output.mp4

# ナレーション + BGM追加
ffmpeg -i video.mp4 -i narration.mp3 -i bgm.mp3 \
  -filter_complex "[1:a]volume=1.0[n];[2:a]volume=0.3[b];[n][b]amix=inputs=2[a]" \
  -map 0:v -map "[a]" output.mp4
```

### 3-2. 確認・判断
- ラフ動画で全体の流れを確認
- **AIの限界**: 細かいカット、タイミング調整は難しい
- → 編集者に依頼する判断

---

## Phase 4: 編集者への引き渡し

### 4-1. フォルダ整理
```
プロジェクト名/
├── 動画/
│   ├── 【使用】01_シーン名.mp4
│   ├── 【使用】02_シーン名.mp4
│   ├── ...
│   ├── 【参考】元動画.mp4
│   └── 【参考】ベース動画（ラフ）.mp4
├── 音声/
│   ├── 【使用】ナレーション.mp3
│   ├── 【使用】BGM.mp3
│   └── 【参考】元動画_ボーカル.mp3
└── 画像/
    └── 【参考】01_シーン名.png ...
```

**命名ルール**:
- 【使用】= 編集で使う素材
- 【参考】= スタイル確認用（使わない）
- 番号プレフィックス（01_, 02_...）で順番を明確に

### 4-2. 指示書作成
- 作りたいもの（コンセプト）
- 使用する素材の説明
- お願いしたいこと（カット調整、テロップ、書き出し設定）
- LINE送信用にプレーンテキスト版も作成

### 4-3. ファイル共有
- フォルダをZIP圧縮
- GIGAファイル便等でアップロード
- URLと指示書をLINEで送信

## 学んだこと・注意点

1. **AIの得意/不得意を見極める**
   - 得意: 画像生成、動画生成、音声生成
   - 不得意: 細かいカット編集、タイミング調整

2. **フォルダ整理は最初から意識**
   - 番号プレフィックスで順番管理
   - 【使用】【参考】で用途を明確に

3. **編集者への引き渡しを意識した素材作り**
   - 参考動画も一緒に渡す
   - 指示書はシンプルに

4. **確認不足によるロスの削減**
   - 認識のズレが起こる前にメタ認知して確認する
   - 「これで合っているか？」を自問し、曖昧なまま進めない
   - シンプルに済むことはシンプルにやる

5. **サイト・ツールの理解を深める**
   - 各サイトやツールの役割・操作方法を把握してから使う
   - 初めて使うツールは操作体系を先に理解する

6. **全体像の把握と相談を最初にやる**
   - 制作・プロジェクト開始時は最初に全体像・ゴールを確認する
   - 指示されていなくても、進め方をユーザーに相談してから着手
   - EnterPlanMode を積極的に使い、進め方の合意を取る

## Gemini画像生成のベストプラクティス

### 同じスレッドで続ける（重要）
- **新しいチャットを開かず、同じスレッド内で修正・再生成を繰り返す**
- コンテキストが途切れると前の学習が引き継がれず、毎回ゼロからやり直しになる

### 参照写真は顔アップを使う
- 人物画像生成時は、顔ができるだけアップになった写真を参照に使う
- 全身・引きの写真だと顔の形が崩れて別人になりやすい
- 1500x1500以上の正面顔アップが理想

### 「Edit this photo」アプローチ
- ゼロから生成するより、参照写真を「編集」するプロンプトのほうが顔の一致度が高い
- 例: 「Edit this photo: Keep the EXACT same face... Change the outfit to...」
- 服装・背景・ライティングを変更しつつ、顔はそのまま保持できる

### PDCA自己フィードバックループ
- 生成後、必ずズームインして参照写真の顔と比較する
- チェック項目: 顎のライン、目の形、鼻、顔の輪郭、全体のプロポーション
- 「これなら大丈夫」レベルになるまで再生成。妥協しない
