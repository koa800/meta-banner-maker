# Lステップ タグ管理 workflow

## current で固定している事実

- 一覧 URL
  - `https://manager.linestep.net/line/tag`
- 上部 button
  - `新しいフォルダ`
  - `新しいタグ`
  - `CSVアップロード`

## live で確認した exact 手順

1. `タグ管理`
2. folder を開く
3. `新しいタグ`
4. dialog title
   - `タグの新規作成`
5. field
   - `タグ名`
6. button
   - `キャンセル / 作成`
7. 一覧 row 右端 `more_vert -> 削除`

## 読み方

- visible 一覧が空でも、folder 件数や action 側で実際に使われていることがある
- つまり `タグ一覧が少ない = 何もしていない` とは見ない

## 30秒レビューの順番

1. そのタグが何の状態を表すか言う
2. 新規作成か既存流用かを決める
3. folder が正しいか見る
4. naming が current と衝突しないか見る
5. action / シナリオ / 流入経路のどこで使うか言う

## 完成条件

- folder と `タグ名` の置き場所を説明できる
- `新しいタグ` で作ったタグが一覧に現れることを確認している
- 既存 current naming と衝突していない
- テスト用なら `削除` まで終えている

## ここで止めて確認する条件

- `タグ名` を見ても状態が分からない
- 同じ意味のタグを新しく作ろうとしている
- 作った後にどこで付与されるか言えない

## 保存前の最小チェック

- 目的と次行動を 1 文で言える
- 入力値や action の実体が確定している
- テスト後に残すか削除するか決めている

## 保存後の最小チェック

- 一覧と実挙動の両方で確認した
- `テスト送信` があるものは `甲原 海人` で確認した
- テスト用は削除して残っていない
