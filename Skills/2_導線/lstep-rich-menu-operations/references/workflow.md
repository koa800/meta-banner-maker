- `リッチメニュー`
- `新しいリッチメニュー`

- `/line/richmenu`

## 作成方法の current 分岐

- `画像をアップロードして作成`
- `テンプレートをベースに作成`

重要
- `新しいリッチメニュー` を押しても、そのまま editor には入らない
- まず作成方法の選択がある

## representative の読み方

### 制限版 -> フル版 切替型

- オンボーディングの進行でメニューを切り替える

### イベント案内型

- `イベント情報`
- テンプレート送信でイベント一覧へつなぐ

### 受講生ナビ型

- `受講生サイト`
- `日報`
- `お問い合わせ・質問`
のように、受講後動線の主要入口を並べる

## exact review 順番

1. current 一覧で visible 件数を見る
2. `旧 / 作成中 / 下書き` の明示がないかを見る
3. 役割が current のテンプレートや導線とつながるかを見る
4. 必要なら edit 画面を開いて実内容を確認する

## Addness 側で見るべき補足

- current / legacy の見分け
- どの account で使うか
- どの template や form につなぐか
