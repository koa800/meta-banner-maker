# Lステップ リッチメニュー exact メモ

## current 入口

- `リッチメニュー`
- `新しいリッチメニュー`

一覧 URL
- `/line/richmenu`

## 作成方法の current 分岐

- `画像をアップロードして作成`
- `テンプレートをベースに作成`

重要
- `新しいリッチメニュー` を押しても、そのまま editor には入らない
- まず作成方法の選択がある

## `画像をアップロードして作成` の exact

- route
  - `/line/richmenu/new?group=0`
- 最初の画面で見えるのは
  - `画像`
  - `メニュー画像選択`
  - footer の `登録`
- `メニュー画像選択` modal
  - `登録メディア一覧`
  - `新規アップロード`
  - `アップロード済み`
  - `ファイルを選択する`
  - `決定`
- file 条件
  - `1MBまで`
  - `jpg,png`
  - `2500×1686` または `2500×843`
- row を選んで `決定` を押すと最初の `登録` が enabled になる
- 最初の `登録` は final save ではなく editor 状態への遷移

## 受け手視点で見る時の順番

1. 最初にどのボタンが目に入るか
2. `今の状態の人` に必要な action が上段にあるか
3. 押させたい main action が埋もれていないか
4. 各ボタンの文言だけで役割が分かるか
5. 押した後に、`テンプレート / 回答フォーム / URL / アクション` が意図どおり出るか

## 30秒レビューの順番

1. `リッチメニュー`
2. 一覧の `タイトル`
3. current か draft / old か
4. 初期表示か切替用か
5. 何人向けの menu か
6. `テンプレート` の有無
7. 各ボタンの役割
8. どの `テンプレート / 回答フォーム / URL / アクション` へつなぐか
9. 必要なら edit を開く
10. 保存後に不要なテスト menu を消す

## exact review 順番

1. current 一覧で visible 件数を見る
2. `旧 / 作成中 / 下書き` の明示がないかを見る
3. 役割が current のテンプレートや導線とつながるかを見る
4. 必要なら edit 画面を開いて実内容を確認する

## 保存前の最小チェック

- その menu が誰向けか言える
- 初期表示か切替用か決まっている
- 各ボタンの役割が重複していない
- `テンプレート送信 / 回答フォーム / URL / アクション` のどれにつなぐか決まっている
- 画像だけ先に決めて、役割が未定のまま進めていない
- `テンプレートをベースに作成` なら required URL を全部埋められる
- 1回目の `登録` が editor 入口で、final `登録` が保存だと切れている

## 保存後の最小チェック

- final `登録` が通っている
- 一覧に戻って `タイトル` が想定どおり見える
- edit を開き直して `テンプレート / コンテンツ設定` が崩れていない
- 実際の LINE 上で、押させたい順に menu が見える
- テスト用なら `more_vert -> 削除 -> 削除する` まで行う

## ここで止めて確認する条件

- current menu を上書きする可能性がある
- どのボタンをどこへつなぐか決まっていない
- `テンプレートをベースに作成` と `画像をアップロードして作成` の選択理由が曖昧
- 1 つの menu に説明も行動も詰め込みすぎている
- `テンプレート送信 / 回答フォーム / URL / アクション` の主役が決まっていない
- required URL を埋め切れないまま `テンプレートをベースに作成` を選びたい

## 削除の exact

1. 一覧 row 右端 `more_vert`
2. `削除`
3. confirm `この操作は取り消せません。本当に削除しますか？`
4. `削除する`
5. row が一覧から消えることを確認
