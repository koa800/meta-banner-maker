# Lステップ リッチメニュー exact メモ

## current 入口

- `リッチメニュー`
- `新しいリッチメニュー`

一覧 URL
- `/line/richmenu`

## 作成方法の current 分岐

- `画像をアップロードして作成`
- `テンプレートをベースに作成`

重要
- `新しいリッチメニュー` を押しても、そのまま editor には入らない
- まず作成方法の選択がある

## `画像をアップロードして作成` の exact

- route
  - `/line/richmenu/new?group=0`
- 最初の画面で見えるのは
  - `画像`
  - `メニュー画像選択`
  - footer の `登録`
- `メニュー画像選択` modal
  - `登録メディア一覧`
  - `新規アップロード`
  - `アップロード済み`
  - `ファイルを選択する`
  - `決定`
- file 条件
  - `1MBまで`
  - `jpg,png`
  - `2500×1686` または `2500×843`
- row を選んで `決定` を押すと最初の `登録` が enabled になる
- 最初の `登録` は final save ではなく editor 状態への遷移

## テンプレート選択時の exact 注意

  - `ボタン1URL`

  - `ボタン3URL`

## representative の読み方

### 制限版 -> フル版 切替型

- オンボーディングの進行でメニューを切り替える

### イベント案内型

- `イベント情報`
- テンプレート送信でイベント一覧へつなぐ

### 受講生ナビ型

- `受講生サイト`
- `日報`
- `お問い合わせ・質問`
のように、受講後動線の主要入口を並べる

## exact review 順番

1. current 一覧で visible 件数を見る
2. `旧 / 作成中 / 下書き` の明示がないかを見る
3. 役割が current のテンプレートや導線とつながるかを見る
4. 必要なら edit 画面を開いて実内容を確認する

## 削除の exact

1. 一覧 row 右端 `more_vert`
2. `削除`
3. confirm `この操作は取り消せません。本当に削除しますか？`
4. `削除する`
5. row が一覧から消えることを確認
