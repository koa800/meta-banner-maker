1. `新しいアクション`
2. `テキスト送信`
3. `保存するアクション名`
4. `1. テキスト送信`
5. `メッセージ`
6. visible `[contenteditable="true"]` に本文
7. `この条件で決定する`

2026-03-14 live 検証

- action 名
  - `ZZ_TMP_20260314_ActionSmoke_04`
- 本文
  - `これはLステップのアクション管理 live smoke です。`
- `保存するアクション名` は、右ペイン上部に出る visible textbox の 1 つ目を使う
- `名前` placeholder の input も見えるが、これは action 名ではなく本文側の変数入力
- 保存後は `検索` に action 名を入れて `search`
- row 右端 `more_vert` を開くと `content_copy コピー / delete 削除`
- `delete 削除 -> 削除する`
- 再度 `検索 -> search` で row が消える

## reusable の代表的な読み方

### 状態更新型

- タグ解除
- タグ追加
- 対応マーク変更

### 送信連動型

- テンプレート送信
- テキスト送信

### 制御型

- シナリオ停止
- メニュー変更
- イベント予約操作

### 複合 macro 型

- 例: `アップセル者対応`
- 読み方
  - タグ解除
  - タグ追加
  - テンプレ送信
  - シナリオ停止
  - 対応マーク変更
  を 1 action に束ねる

2026-03-14 API 実例
- `アップセル者対応`
  - `タグ[会員]を解除`
  - `タグ[PRMアップセル]を追加`
  - `テンプレ[アップセル者向け（配信停止のお知らせ）]を送信`
  - `タグ[ストリーク停止]を追加`
  - `シナリオを停止`
  - `対応マークにアップセルを代入`
