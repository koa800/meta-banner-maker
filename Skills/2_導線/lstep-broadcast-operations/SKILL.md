# lstep-broadcast-operations

## Overview

Lステップの `一斉配信` を、`新規配信` の作成だけでなく `下書き保存 -> 下書き一覧で削除` まで exact に再現する skill です。

Addness 固有の命名、送るべき account、配信戦略は `Master/knowledge/lstep_structure.md` を正本にする。この skill は、会社をまたいで再利用できる exact 手順だけを持つ。

## ツールそのものの役割

`一斉配信` は、対象を切った友だちに対して同じ message をその時だけまとめて送る機能です。

## アドネスでの役割

アドネスでは `一斉配信` を、単発の告知、周知、promotion を送る時に使う。継続接触は `シナリオ配信` に寄せ、今この瞬間に伝える内容を `一斉配信` に寄せる。

## 役割

`一斉配信` は、LINE 友だち全体または絞り込み済みセグメントに対して、今この瞬間に必要な message をまとめて送る機能です。

## ゴール

次を迷わずできる状態を作る。
- `一斉配信 -> 新規配信` の正しい入口を選ぶ
- 管理用タイトルと本文を入れる
- `下書き保存`
- `下書きを開く` から一覧を確認する
- テスト用の下書きなら削除する

## 必要変数

- 管理用タイトル
- 配信の目的
- 誰に送るか
- 本文
- 画像やURLの有無
- 送信前に下書きで止めるかどうか
- テスト用か本番用か

## 使い分け

- `新規配信`
  - 1件ずつ新規で作る時
- `テンプレート配信`
  - 既存テンプレートを流用する時

最小検証では、まず `新規配信` を exact に通す。

## Workflow

1. `一斉配信`
2. `新規配信`
3. 管理用タイトルを入れる
4. 本文を入れる
5. `下書き保存`
6. `下書きを開く`
7. 一覧で対象を確認する
8. テスト用なら選択して削除する

## exact 手順

### create

1. `一斉配信`
2. `新規配信`
3. create 画面で、画面上部の visible `input.form-control` に管理用タイトルを入れる
4. 本文は visible `.ProseMirror` に入れる
5. `下書き保存`

### 保存後の確認

- 成功時は `下書き [テキスト]... を新たに保存しました。` の toast が出る
- `下書きを開く` を押すと `/line/draft/magazine` に移動する

### 削除

削除は row menu ではありません。

1. `下書きを開く`
2. 対象 row 左端 checkbox の label を押して選択する
3. 上部 `チェックをした下書きを削除する`
4. confirm modal は出ない前提で見る
5. `選択した下書きを削除しました。` の toast を確認する
6. 再読込して row が消えていることを確認する

## 検証

最低でも次を確認する。
- 管理用タイトルが正しい
- 本文が visible editor に入っている
- 下書き保存できる
- `下書きを開く` から一覧に出る
- テスト用なら一覧から消えている

## 実装前の最小チェック

- 何を実装する画面か 1 文で言える
- 現在触るべき account と current / legacy を切っている
- テスト送信や削除の要否を先に決めている
- 次の行動や状態変化を 1 つに絞っている

## 保存前の最小チェック

- 目的、対象、次行動を 1 文で言える
- 入力値や action の実体が確定している
- current の row やフォルダを取り違えていない
- テスト後に残すか削除するか決めている

## 保存後の最小チェック

- 一覧に想定どおり表示される
- 必要なら `テスト送信` や preview で実機確認した
- action や遷移先が意図どおり動く
- テスト用なら削除して row が消えたことを確認した

## ここで止めて確認する条件

- 何を変える設定か 1 文で言えない
- current / legacy の切り分けに迷う
- 新しい外部連携や本番影響の大きい分岐を足す
- 既存 high-quality な実例と大きく異なる構成にしたい

## 完成条件

- 正しい入口から作成または編集できている
- 保存後に一覧と実挙動で確認できている
- テスト送信が必要な場合は `甲原 海人` で確認している
- テスト用の残骸が残っていない

## NG

- hidden input に入っているだけで本文入力したと判断する
- create 画面で消したつもりになる
- 下書き削除を row menu 探索で迷う
- テスト用下書きを残す
- Addness 固有の naming rule を skill 内に焼き込む

## 正誤判断

正しい状態
- `一斉配信 -> 新規配信` の正しい入口から入っている
- 管理用タイトルと本文を visible field に入れている
- `下書き保存` 後に下書き一覧まで確認できている
- テスト用は削除済み

間違った状態
- `テンプレート配信` と混同している
- 本文を hidden field や別 editor に入れている
- create 画面を閉じただけで後片付け完了と思っている
