# lstep-cross-analysis-operations

## Overview

Lステップの `クロス分析` を、`新しいクロス分析` の作成だけでなく `保存 -> 一覧再確認 -> 削除` まで exact に再現する skill です。

Addness 固有の命名規則、どの軸をよく掛け合わせるか、current representative は `Master/knowledge/lstep_structure.md` を正本にする。この skill は、会社をまたいで再利用できる exact 手順だけを持つ。

## ツールそのものの役割

`クロス分析` は、複数の条件や状態を掛け合わせて比較する分析定義を保存する機能です。

## アドネスでの役割

アドネスでは `クロス分析` を、流入経路、タグ、予約状態などを掛け合わせて差が出ている場所を見つける比較面として使う。結果一覧ではなく、比較軸そのものを設計する機能として扱う。

## 役割

`クロス分析` は、2つ以上の軸を掛け合わせて傾向を見る分析定義を保存する機能です。

## ゴール

次を迷わずできる状態を作る。
- `クロス分析` の正しい入口を選ぶ
- `新しいクロス分析` を作る
- `評価軸(縦軸)` と `評価項目(横軸)` を設定する
- 保存条件を満たして登録する
- 一覧で再確認する
- テスト用なら削除する

## 必要変数

- 管理名
- フォルダ
- 評価対象
- 評価軸(縦軸)
- 評価項目(横軸)
- オプション
- テスト用か本番用か

## Workflow

1. `クロス分析`
2. `新しいクロス分析`
3. 基本情報を入れる
4. `評価軸(縦軸)` と `評価項目(横軸)` を設定する
5. `保存`
6. 一覧に戻って作成物を確認する
7. テスト用なら削除する

## exact 手順

### create

1. `クロス分析`
2. `新しいクロス分析`
3. create 画面で次を入れる
   - `管理名`
   - `フォルダ`
   - `評価対象`
   - `評価軸(縦軸)`
   - `評価項目(横軸)`
   - 必要なら `オプション`
4. `保存`

### 保存条件

`評価軸(縦軸)` が空だと保存前に止まる。

- message
  - `評価軸(縦軸)は必須です`

### 作成後の確認

1. 一覧に戻る
2. 作成した `管理名` の row が visible か確認する
3. 必要なら再読込する

### 削除

1. 一覧 row 右端 `more_vert`
2. `削除`
3. 確認 dialog で `削除する`
4. 一覧再読込後に row が消えていることを確認する

## 検証

最低でも次を確認する。
- `管理名` が正しい
- `評価軸(縦軸)` を入れている
- 一覧に作成物が visible
- テスト用なら削除済み

## 実装前の最小チェック

- 何を実装する画面か 1 文で言える
- 現在触るべき account と current / legacy を切っている
- テスト送信や削除の要否を先に決めている
- 次の行動や状態変化を 1 つに絞っている

## 保存前の最小チェック

- 目的、対象、次行動を 1 文で言える
- 入力値や action の実体が確定している
- current の row やフォルダを取り違えていない
- テスト後に残すか削除するか決めている

## 保存後の最小チェック

- 一覧に想定どおり表示される
- 必要なら `テスト送信` や preview で実機確認した
- action や遷移先が意図どおり動く
- テスト用なら削除して row が消えたことを確認した

## ここで止めて確認する条件

- 何を変える設定か 1 文で言えない
- current / legacy の切り分けに迷う
- 新しい外部連携や本番影響の大きい分岐を足す
- 既存 high-quality な実例と大きく異なる構成にしたい

## 完成条件

- 正しい入口から作成または編集できている
- 保存後に一覧と実挙動で確認できている
- テスト送信が必要な場合は `甲原 海人` で確認している
- テスト用の残骸が残っていない

## NG

- `評価軸(縦軸)` を空のまま保存しようとする
- 分析結果画面だと思って、定義保存だと理解せずに触る
- テスト用の分析定義を残す
- Addness 固有の軸名を skill 内に焼き込む

## 正誤判断

正しい状態
- `クロス分析` が保存型の分析定義だと説明できる
- `評価軸(縦軸)` と `評価項目(横軸)` を分けて設定している
- 保存後に一覧再確認している
- テスト用は削除済み

間違った状態
- 結果を見るだけの画面だと思っている
- `評価軸(縦軸)` を空のまま進める
- テスト作成後に row を残す
