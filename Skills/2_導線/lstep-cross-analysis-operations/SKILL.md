# lstep-cross-analysis-operations

## Overview

Lステップの `クロス分析` を、`新しいクロス分析` の作成だけでなく `保存 -> 一覧再確認 -> 削除` まで exact に再現する skill です。

Addness 固有の命名規則、どの軸をよく掛け合わせるか、current representative は `Master/knowledge/lstep_structure.md` を正本にする。この skill は、会社をまたいで再利用できる exact 手順だけを持つ。

## 役割

`クロス分析` は、2つ以上の軸を掛け合わせて傾向を見る分析定義を保存する機能です。

## ゴール

次を迷わずできる状態を作る。
- `クロス分析` の正しい入口を選ぶ
- `新しいクロス分析` を作る
- `評価軸(縦軸)` と `評価項目(横軸)` を設定する
- 保存条件を満たして登録する
- 一覧で再確認する
- テスト用なら削除する

## 必要変数

- 管理名
- フォルダ
- 評価対象
- 評価軸(縦軸)
- 評価項目(横軸)
- オプション
- テスト用か本番用か

## Workflow

1. `クロス分析`
2. `新しいクロス分析`
3. 基本情報を入れる
4. `評価軸(縦軸)` と `評価項目(横軸)` を設定する
5. `保存`
6. 一覧に戻って作成物を確認する
7. テスト用なら削除する

## exact 手順

### create

1. `クロス分析`
2. `新しいクロス分析`
3. create 画面で次を入れる
   - `管理名`
   - `フォルダ`
   - `評価対象`
   - `評価軸(縦軸)`
   - `評価項目(横軸)`
   - 必要なら `オプション`
4. `保存`

### 保存条件

`評価軸(縦軸)` が空だと保存前に止まる。

- message
  - `評価軸(縦軸)は必須です`

### 作成後の確認

1. 一覧に戻る
2. 作成した `管理名` の row が visible か確認する
3. 必要なら再読込する

### 削除

1. 一覧 row 右端 `more_vert`
2. `削除`
3. 確認 dialog で `削除する`
4. 一覧再読込後に row が消えていることを確認する

## 検証

最低でも次を確認する。
- `管理名` が正しい
- `評価軸(縦軸)` を入れている
- 一覧に作成物が visible
- テスト用なら削除済み

## NG

- `評価軸(縦軸)` を空のまま保存しようとする
- 分析結果画面だと思って、定義保存だと理解せずに触る
- テスト用の分析定義を残す
- Addness 固有の軸名を skill 内に焼き込む

## 正誤判断

正しい状態
- `クロス分析` が保存型の分析定義だと説明できる
- `評価軸(縦軸)` と `評価項目(横軸)` を分けて設定している
- 保存後に一覧再確認している
- テスト用は削除済み

間違った状態
- 結果を見るだけの画面だと思っている
- `評価軸(縦軸)` を空のまま進める
- テスト作成後に row を残す
