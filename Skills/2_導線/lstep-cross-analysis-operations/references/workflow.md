# Lステップ クロス分析 workflow

## current で固定している事実

- 一覧 URL
  - `https://manager.linestep.net/line/board`
- create URL
  - `https://manager.linestep.net/line/board/edit/new`
- 画面上部ボタン
  - `新しいフォルダ`
  - `新しいクロス分析`
  - `並び替え`

## live で確認した exact 手順

1. `クロス分析`
2. `新しいクロス分析`
3. visible field
   - `管理名`
   - `フォルダ`
   - `評価対象`
   - `評価軸(縦軸)`
   - `評価項目(横軸)`
   - `オプション`
4. `評価軸(縦軸)` を空のまま保存しようとすると
   - `評価軸(縦軸)は必須です`
5. 保存後は一覧へ戻って row を確認
6. テスト用なら `more_vert -> 削除 -> 削除する`

## 読み方

- `クロス分析` は結果閲覧画面ではなく、`どの軸を掛け合わせるかを定義して保存する機能`
- 精度は、事前の `タグ / 友だち情報 / 流入経路` の整備に依存する
- Addness では `流入経路 × 予約`、`プラン × 合宿参加日` のような用途と相性が良い

## 30秒レビューの順番

1. 何と何を掛け合わせたいかを 1 文で言う
2. `評価対象` を決める
3. `評価軸(縦軸)` と `評価項目(横軸)` の役割を切る
4. 事前に必要な `タグ / 友だち情報 / 流入経路` があるか確認する
5. 保存後に一覧で再確認する

## 完成条件

- `管理名`、`評価対象`、`評価軸(縦軸)`、`評価項目(横軸)` を理由付きで設定できる
- 何を掛け合わせて見たい分析かを 1 文で言える
- 保存後に一覧で再確認できる
- テスト用なら削除まで終えている

## ここで止めて確認する条件

- 掛け合わせたい軸を 1 文で言えない
- `評価軸(縦軸)` と `評価項目(横軸)` の違いが曖昧
- 前提データが未整備なのに、結果画面だけ作ろうとしている

## 保存前の最小チェック

- 目的と次行動を 1 文で言える
- 入力値や action の実体が確定している
- テスト後に残すか削除するか決めている

## 保存後の最小チェック

- 一覧と実挙動の両方で確認した
- `テスト送信` があるものは `甲原 海人` で確認した
- テスト用は削除して残っていない
