# Lステップ フレックスメッセージ exact メモ

## current 入口

- `テンプレート`
- `新しいテンプレート`
- `フレックスメッセージ`

代表 route
- 新規: `/line/template/edit_v3/new?group=0&editMessage=1`
- 既存: `/line/template/edit_v3/{id}?group=0&editMessage=0`

## current で見えている主要ラベル

- `テンプレート名`
- `PC版・通知欄での代替テキスト`
- `edit メッセージを作成`
- `＋ブロックを追加`
- `メッセージを保存`
- 外側の `保存`

## block 追加候補

- `画像`
- `タイトル`
- `テキスト`
- `ボタン`
- `動画`

## 画像 block の current ラベル

- `画像を追加`
- `新規アップロード`
- `登録メディア`
- `ここにファイルをドロップ`
- `ファイルを選択する`
- `閉じる`
- `保存`

重要
- upload だけでは不十分
- file row を明示的に選んで `選択中のファイル数: 1 / 1` にしてから保存する

## ボタン block の current ラベル

- `アクション設定`
- `種類`
- `閉じる`
- `保存する`

確認済みの action 種類
- `URLを開く`
- `回答フォームを開く`
- `電話をかける`
- `LINEアカウントを友だち追加する`
- `メールを送る`
- `その他`

## representative の読み方

### 単発 CTA 型

- 例: `3月 スキルプラス合宿 集客`
- 構成
  - 画像 1
  - テキスト 3
  - ボタン 1
- 読み方
  - `重要告知 -> 補足 -> main CTA`

### segmentation 型

- 例: `選べる3種のPR`
- 構成
  - 画像 4
  - テキスト 4
  - 3 panel
- 読み方
  - `どれを売るか` より `どの入口を選ばせるか`

### 長尺訴求型

- 例: `AIキャンプ告知`
- 構成
  - 画像多め
  - テキスト多め
  - image action 混在
- 読み方
  - 一枚物ではなく、スクロール型 sales page を message 内に圧縮している

## exact smoke 手順

1. 最小構成で作る
   - `タイトル`
   - `テキスト`
   - `ボタン`
2. button の action は `URLを開く`
3. 保存
4. 一覧へ戻る
5. 行右端 `... -> テスト送信`
6. 実機確認
7. テスト用なら削除
