# lstep-event-reservation-operations

## Overview

Lステップの `イベント予約` を、`新しいイベント` の作成だけでなく `友だち予約URL / 予約枠一覧 / プレビュー` の確認と削除まで exact に再現する skill です。

Addness 固有の命名規則、どの account でイベント予約を使うか、current representative は `Master/knowledge/lstep_structure.md` を正本にする。この skill は、会社をまたいで再利用できる exact 手順だけを持つ。

## ツールそのものの役割

`イベント予約` は、単発イベントの予約枠、予約ページ、予約者管理を束ねる機能です。

## アドネスでの役割

アドネスでは `イベント予約` を、合宿やセミナーなどの event 単位で予約と参加者管理をする時に使う。個別の面談予約より、イベント中心で読む。

## 役割

`イベント予約` は、セミナー、合宿、オフ会、研修のような単発イベントの予約を受け付ける機能です。

## ゴール

次を迷わずできる状態を作る。
- `イベント予約` の正しい入口を選ぶ
- `新しいイベント` を作る
- `友だち予約URL` を確認する
- `予約枠一覧` を確認する
- `プレビュー` で見え方を確認する
- テスト用なら削除する

## 必要変数

- イベント名
- 開催日時
- 開催場所
- 場所URL
- 予約枠
- 予約ページURLの保管先
- テスト用か本番用か

## Workflow

1. `イベント予約`
2. `新しいイベント`
3. イベント基本情報を入れる
4. `登録`
5. 一覧で `友だち予約URL / 友だち予約履歴URL / 予約枠一覧 / CSV出力 / 予約者一覧 / プレビュー` を確認する
6. テスト用なら削除する

## exact 手順

### create

1. `イベント予約`
2. `新しいイベント`
3. create 画面で、イベント名と必要な基本情報を入れる
4. `登録`

### 作成後の確認

一覧では少なくとも次を確認する。
- `友だち予約URL`
- `友だち予約履歴URL`
- `予約枠一覧`
- `CSV出力`
- `予約者一覧`
- `プレビュー`

`友だち予約URL` は新規予約導線、`友だち予約履歴URL` は既予約者の履歴や確認導線として読む。

### 削除

1. 一覧 row 右端 `more_vert`
2. `削除`
3. 確認 dialog が出たら `削除する`
4. 一覧再読込後に row が消えていることを確認する

## 検証

最低でも次を確認する。
- イベント名が正しい
- `友だち予約URL` が発行されている
- `予約枠一覧` が開ける
- `プレビュー` でページが見られる
- テスト用なら削除済み

## 実装前の最小チェック

- 何を実装する画面か 1 文で言える
- 現在触るべき account と current / legacy を切っている
- テスト送信や削除の要否を先に決めている
- 次の行動や状態変化を 1 つに絞っている

## 保存前の最小チェック

- 目的、対象、次行動を 1 文で言える
- 入力値や action の実体が確定している
- current の row やフォルダを取り違えていない
- テスト後に残すか削除するか決めている

## 保存後の最小チェック

- 一覧に想定どおり表示される
- 必要なら `テスト送信` や preview で実機確認した
- action や遷移先が意図どおり動く
- テスト用なら削除して row が消えたことを確認した

## ここで止めて確認する条件

- 何を変える設定か 1 文で言えない
- current / legacy の切り分けに迷う
- 新しい外部連携や本番影響の大きい分岐を足す
- 既存 high-quality な実例と大きく異なる構成にしたい

## 完成条件

- 正しい入口から作成または編集できている
- 保存後に一覧と実挙動で確認できている
- テスト送信が必要な場合は `甲原 海人` で確認している
- テスト用の残骸が残っていない

## NG

- `カレンダー予約` と混同する
- URL を控えずに閉じる
- `予約枠一覧` を見ない
- `プレビュー` を見ずに完了扱いにする
- テスト用イベントを残す

## 正誤判断

正しい状態
- 単発イベント向きかどうか説明できる
- `友だち予約URL` と `予約枠一覧` を両方確認している
- `プレビュー` まで見ている
- テスト用は削除済み

間違った状態
- 面談や個別予約に使う前提で触っている
- URL だけ控えて予約枠や見え方を見ない
- テスト作成後に row を残す
