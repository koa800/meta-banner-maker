# Lステップ イベント予約 workflow

## current で固定している事実

- 一覧 URL
  - `https://manager.linestep.net/line/reserve/event`
- 画面上部ボタン
  - `新しいフォルダ`
  - `新しいイベント`
- row action
  - `友だち予約URL`
  - `友だち予約履歴URL`
  - `予約枠一覧`
  - `CSV出力`
  - `予約者一覧`
  - `プレビュー`

## live で確認した exact 手順

1. `イベント予約`
2. `新しいイベント`
3. イベントの基本情報を入力して `登録`
4. 一覧で `友だち予約URL`
5. `友だち予約履歴URL`
6. `予約枠一覧`
7. `CSV出力`
8. `予約者一覧`
9. `プレビュー`
10. テスト用なら `more_vert -> 削除 -> 削除する`

## 読み方

- `イベント予約` は、単発イベントや合宿の予約受付を回す operational screen
- `友だち予約URL` は新規予約の入口
- `友だち予約履歴URL` は既予約者向けの履歴・確認導線
- current の Addness では、`スキルプラス【サポートLINE】` が告知口、`【予約窓口】スキルプラス合宿` が実務予約口に寄っている
