# lstep-auto-response-operations

## Overview

Lステップの `自動応答` を、`新しい自動応答` の作成だけでなく `一覧確認 -> 削除` まで exact に再現する skill です。

Addness 固有の命名規則、folder の読み方、current / legacy の判断は `Master/knowledge/lstep_structure.md` を正本にする。この skill は、会社をまたいで再利用できる exact 手順だけを持つ。

live で触る前に、必ず `python3 System/scripts/funnel_tool_live_readiness.py` を実行し、`lstep.auth_alive=true` に加えて `lstep.account_name` が intended な公式LINE account か確認する。`auth_alive=true` でも別 account 文脈を見ていることがある。

## ツールそのものの役割

`自動応答` は、特定の trigger やキーワードに対して即時に response を返す機能です。

## アドネスでの役割

アドネスでは `自動応答` を、入口直後や特定 trigger に対する即時の受け皿として使う。長い教育ではなく、すぐ返すべき内容で friction を減らす用途です。

## 役割

`自動応答` は、特定のキーワードに対して応答内容や action を返す機能です。

## ゴール

次を迷わずできる状態を作る。
- `自動応答` の正しい入口を選ぶ
- `新しい自動応答` を作る
- 一覧で確認する
- テスト用なら削除する

## 必要変数

- 自動応答名
- folder
- 応答設定
  - `キーワードで応答`
  - `一律で応答`
- キーワード
- マッチ条件
- アクション有無
- テスト用か本番用か

## Workflow

1. `自動応答`
2. 必要な folder を開く
3. `新しい自動応答`
4. `自動応答名`
5. `キーワード`
6. 必要なら `アクションを設定する`
7. `登録`
8. 一覧で確認
9. テスト用なら削除

## exact 手順

### create

1. `自動応答`
2. 必要な folder を開く
3. `新しい自動応答`
4. field `自動応答名`
5. `応答設定`
   - 通常は `キーワードで応答`
6. field `キーワード`
7. 必要なら `アクションを設定する`
8. `登録`

### delete

1. 一覧で対象 row を確認
2. row 右端 `more_vert`
3. `削除`
4. `削除する`
5. 一覧再読込後に row が消えていることを確認する

## 検証

最低でも次を確認する。
- folder が正しい
- `自動応答名` が正しい
- `キーワード` が正しい
- 一覧に出る
- アクション有無が意図どおり
- テスト用なら削除済み

## 実装前の最小チェック

- `lstep.account_name` が intended な公式LINE account と一致している
- 何を実装する画面か 1 文で言える
- 現在触るべき account と current / legacy を切っている
- テスト送信や削除の要否を先に決めている
- 次の行動や状態変化を 1 つに絞っている

## 保存前の最小チェック

- 目的、対象、次行動を 1 文で言える
- 入力値や action の実体が確定している
- current の row やフォルダを取り違えていない
- テスト後に残すか削除するか決めている

## 保存後の最小チェック

- 一覧に想定どおり表示される
- 必要なら `テスト送信` や preview で実機確認した
- action や遷移先が意図どおり動く
- テスト用なら削除して row が消えたことを確認した

## ここで止めて確認する条件

- 何を変える設定か 1 文で言えない
- current / legacy の切り分けに迷う
- 新しい外部連携や本番影響の大きい分岐を足す
- 既存 high-quality な実例と大きく異なる構成にしたい

## 完成条件

- 正しい入口から作成または編集できている
- 保存後に一覧と実挙動で確認できている
- テスト送信が必要な場合は `甲原 海人` で確認している
- テスト用の残骸が残っていない

## NG

- folder を見ずに作る
- `キーワードで応答` と `一律で応答` を曖昧に選ぶ
- テスト用の自動応答を残す
- visible 一覧だけ見て `何に使うか` を判断しない

## 正誤判断

正しい状態
- 何の状態入力か、何の返信かを説明できる
- 正しい folder で作っている
- キーワードと応答意図が一致している
- 一覧で確認している
- テスト用は削除済み

間違った状態
- 自動返信か状態更新かを区別せずに作る
- キーワードを仮置きのまま残す
- 作成後の一覧確認や削除をしない
