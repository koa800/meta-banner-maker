# lstep-answer-form-operations

## Overview

Lステップの `回答フォーム` を、作成して終わりではなく `質問追加 -> 保存 -> テスト送信 -> 削除` まで exact に再現する skill です。

Addness 固有の命名、テスト送信先、current 例は `Master/knowledge/lstep_structure.md` を正本にする。この skill は、会社をまたいで再利用できる作成手順だけを持つ。

## 役割

`回答フォーム` は、ユーザーから structured data を回収し、次の配信や予約や営業判断の入力を作るための form type です。

## ゴール

次を迷わずできる状態を作る。
- `回答フォーム` の正しい入口を選ぶ
- フォーム名とフォームタイトルを作る
- 質問を追加する
- 保存する
- 必要なら `テスト送信`
- テスト用なら削除する

## 必要変数

- フォーム名
- フォームタイトル
- 回収したい項目
- 必須項目かどうか
- 回答後に何を判断したいか
- テスト送信するか
- テスト後に残すか削除するか

## Workflow

1. `回答フォーム`
2. `新しい回答フォーム`
3. `フォーム名`
4. `フォームタイトル`
5. 質問を追加
6. `保存`
7. 一覧に戻る
8. 必要なら `... -> テスト送信`
9. 実機確認
10. テスト用なら `... -> 削除`

## exact 手順

### create

1. `回答フォーム`
2. `新しい回答フォーム`
3. `フォーム名`
4. `フォームタイトル`
5. 質問追加
6. `保存`

### 質問追加

最初に exact 化しておくべきなのは `記述式(テキストボックス)` です。質問 block を追加し、質問名を入れて保存するところまでをまず固定する。

### テスト送信

行右端 `... -> テスト送信` を使う。送信先や重複送信ルールは `Master/knowledge/lstep_structure.md` を正本にする。

### 削除

行右端 `... -> 削除 -> 削除する` で消す。消えたことを一覧で確認する。

## 検証

最低でも次を確認する。
- フォーム名が正しい
- フォームタイトルが正しい
- 質問が表示される
- 必須 / 任意の意図がずれていない
- 回答者目線で違和感がない
- テスト送信から開ける

## NG

- 回収したい情報が曖昧なまま作る
- 質問文が短すぎて回答者が迷う
- 一覧で開けるだけで完成扱いにする
- テスト送信せずに本番へ回す
- テスト用フォームを残す
- Addness 固有ルールを skill に焼き込む

## 正誤判断

正しい状態
- 何を回収したいフォームか一読で分かる
- 質問の順番が自然
- 回答者が迷わない
- 実機で開ける
- テスト用は削除済み

間違った状態
- フォーム名と用途がずれる
- 質問だけ作って送信確認していない
- 質問文が運用側のメモになっている
- current 一覧で消えているか確認していない
