## live で確認した exact 手順

1. funnel を開く
2. `登録経路`
3. `追加`
4. create 画面で visible field
   - `グループ`
   - `管理名称`
   - `ファネルステップ`
   - `ページ`
5. 保存
6. `データ(合算)` と `データ(日別)` で `登録経路` 条件を確認
7. 必要なら `グループ管理`
8. 必要なら `表示順変更`

## 読み方

- `登録経路` は広告IDだから必ず切る機能ではない
- 基本思想は `どこからユーザーが訪れたかを知りたい時に切る`
- 同じページを使うなら、まずページ複製ではなく `登録経路` を検討する

## 30秒レビューの順番

1. どこから来た人を知りたいかを 1 文で言う
2. `ページを増やす` べきか `登録経路` で足りるか決める
3. `グループ / 管理名称 / ファネルステップ / ページ` を確認する
4. 保存後に `データ(合算)` か `データ(日別)` でどう識別するか決める
5. current と重複しないか見る

## 保存前の最小チェック

- page を増やすべきか、`登録経路` を増やすべきか言える
- `どこから来たかを知りたい` という目的がある
- `グループ / 管理名称 / ファネルステップ / ページ` を説明できる

## 保存後の最小チェック

- 一覧に `管理名称` が意図どおり出ている
- `データ(合算)` か `データ(日別)` で識別できる
- 同じ意味の `登録経路` を重複作成していない

## ここで止めて確認する条件

- page を分けるべきか `登録経路` で足りるか曖昧
- `広告IDだから切る` だけになっていて、分析目的を言えない
- current の `グループ` に置くべきか迷う

## 完成条件

- `登録経路` を作る目的を `どこから来たかを知りたい` で説明できる
- `ページを増やす` のではなく `登録経路で足りる` 理由を説明できる
- `グループ / 管理名称 / ファネルステップ / ページ` を迷わず設定できる
- 保存後に `データ(合算)` か `データ(日別)` で識別できる
