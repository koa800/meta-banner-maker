## Addness 側での重要解釈

- 動画とメディアは page の付属物ではなく、再利用される asset library
- page 修正時は、まず media/video 側に asset があるか確認した方が安全
- 動画は「上げた」で終わりではなく、
  - サムネイル
  - チャプター
  - 分析
  まで見て初めて運用理解になる

## 保存前の最小チェック

- どの folder に置くべきか言える
- 画像か動画かを分けて考えている
- 動画なら `サムネイル / チャプター / 分析` のうち何を触るか言える

## 保存後の最小チェック

- asset が intended folder に入っている
- 動画なら `サムネイル変更 / チャプター設定 / 分析` を必要な範囲で確認した
- 埋め込み先があるなら、埋め込み先でも表示を確認する
- 差し替え案件なら、旧 asset と新 asset の取り違えが起きていないか名前と用途で再確認する

## 変更後の影響確認順

1. intended folder に戻る
2. asset 名を見て、旧 asset と新 asset を取り違えていないか確認する
3. 動画なら `埋め込み用URL` を再取得する
4. `サムネイル変更 / チャプター設定 / 分析` のうち触った項目を開き直して保存結果を見る
5. page / lesson など埋め込み先があるなら、実際にその画面で表示と再生を確認する

## 構築精度だけを見る時のチェック

1. 対象が `メディア管理` か `動画管理` か
2. folder と用途を 1 文で言えるか
3. `サムネイル変更 / チャプター設定 / 分析` のどこが主役か切れているか
4. 既存差し替えか新規追加かを切れているか
5. test asset を残さない前提になっているか

## ここで止めて確認する条件

- folder をどこに置くべきか曖昧
- 動画の差し替えで既存 page / lesson への影響が読めない
- `チャプター設定` と `分析` のどちらを優先すべきか判断できない
- 同名または類似名の asset が複数あり、差し替え対象を 1 つに絞れない

## 完成条件

- 対象 asset を `画像 / 動画` のどちらかで固定できている
- intended folder に正しく置けている
- 動画なら `サムネイル / チャプター / 分析` のうち触った項目を説明できる
- 埋め込み先がある asset は、埋め込み先での表示や再生も確認している
- 差し替え時は既存 page / lesson への影響有無を説明できる
