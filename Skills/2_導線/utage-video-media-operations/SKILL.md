# utage-video-media-operations

## Overview

UTAGE の `動画管理` と `メディア管理` を、asset library として安全に扱う skill です。

Addness 固有の folder 名、current 代表 asset、どの page / 会員サイトで使っているかは `Master/addness/utage_structure.md` を正本にする。この skill では、どの案件でも再利用できる exact 手順だけを持つ。

## ツールそのものの役割

`動画管理` と `メディア管理` は、画像、資料、動画を置いて、他の page や会員サイトから再利用する asset library です。

## アドネスでの役割

アドネスでは `動画管理` と `メディア管理` を、LP、会員サイト、講義、導線の付属物ではなく `再利用前提の source asset` として扱う。したがって、page 側で直に差し替える前に `media / video` 側の正本を先に確認する。

## 役割

`動画管理` と `メディア管理` は、LP、会員サイト、講義、セールス page で使う画像・動画の source asset を管理する層です。

## ゴール

次を迷わずできる状態を作る。
- `メディア管理` で画像や資料を上げる
- `動画管理` で動画を上げる
- `サムネイル変更` をする
- `チャプター設定` をする
- `分析` で視聴維持率を見る
- どの asset をどこで再利用しているか、事故なく扱う

## 必要変数

- 対象が `メディア` か `動画` か
- asset 名
- 入れたい folder
- 用途
  - LP
  - 会員サイト
  - 講義
  - セールス
- 動画なら
  - サムネイル変更が必要か
  - チャプターが必要か
  - 分析を見るか

## 依頼を受けたら最初に埋める項目

- 対象が `メディア管理` か `動画管理` か
- intended な folder
- asset 名
- どこで使うか
  - `LP / 会員サイト / 講義 / セールス`
- 動画なら
  - `サムネイル変更 / チャプター設定 / 分析`
  のどこまでやるか
- exploratory か、本線 asset か

## 実装前の最小チェック

- 対象が `メディア管理` か `動画管理` か切れているか
- どの folder に置くべきか先に決めたか
- その asset をどこで使うか言えるか
- 動画なら `サムネイル変更 / チャプター設定 / 分析` のどこまで必要か先に切ったか

## 入口選択の exact な問い

- 今やりたいのは `画像 / PDF / 静的 asset` を扱うことか
  - なら `メディア管理`
- 今やりたいのは `動画そのもの / サムネイル / チャプター / 分析` を扱うことか
  - なら `動画管理`
- 今やりたいのは `ページの見せ方や CTA` を変えることか
  - それはこの skill ではなく `ページ`

つまり、この skill の入口は `asset が静的か動画か` で決まる。

## 判断フレーム

## 10秒判断

- `メディア管理`
  - 画像
  - PDF
  - 資料
  の静的 asset を扱う
- `動画管理`
  - 動画本体
  - サムネイル
  - チャプター
  - 分析
  を扱う
- `チャプター設定`
  - 長尺動画の視聴体験を上げたい
- `分析`
  - 離脱位置や CTA 手前の落ち方を見たい

この 10 秒判断が言えない時は、`新規アップロード` や `チャプター設定` を押す前に stop する。

### `メディア管理` を使う時

次のどれかなら `メディア管理` を使う。
- 画像
- PDF
- 資料
- page 内で使う静的 asset

### `動画管理` を使う時

次のどれかなら `動画管理` を使う。
- 講義動画
- セミナー動画
- アーカイブ動画
- page や会員サイトに埋め込む動画

### `チャプター設定` を先に疑う時

次のどれかなら `チャプター設定` を先に見る。
- 長尺講義
- セミナーアーカイブ
- 視聴体験を上げたい
- 動画内の区切りを明示したい

### `分析` を先に見る時

次のどれかなら `分析` を先に見る。
- どこで離脱しているか知りたい
- CTA 手前まで見られているか知りたい
- 長尺講義の retention を見たい

## Workflow

1. `メディア管理` または `動画管理`
2. folder を確認
3. `新規アップロード` または `新規フォルダ`
4. 動画なら一覧カードから
   - `サムネイル変更`
   - `チャプター設定`
   - `分析`
   を必要に応じて触る
5. page / 会員サイト側で参照する
6. 使わない test asset は残さない

## 最小 downstream smoke

asset は upload して終わりにしない。

1. intended な folder
2. intended な asset 名
3. intended な preview or card
4. page か会員サイト側の参照先
5. intended な見え方か

source asset と参照先の両方が intended でない限り、asset 作業は完了にしない。

## 最小 live change

既存 current asset を触る時は、最初の 1 回を `1 asset の軽い変更だけ` に制限する。

1. 対象 folder を 1 つ固定
2. 対象 asset を 1 つ固定
3. 変更対象を 1 つに固定
   - `名称変更`
   - `サムネイル変更`
   - `チャプター設定 1 行追加`
   のような downstream 影響を切り戻しやすいものを優先
4. 保存
5. 一覧へ戻る
6. intended な folder と asset 名を再確認
7. page / 会員サイト側の参照先を 1 箇所だけ確認
8. 必要なら元に戻す

最初から本線動画の差し替えや、複数 asset のまとめ変更をしない。最初は `1 asset 1変更` で change-safe な edit を通す。

## 最小 rollback

exploratory に変更した時は、`folder / asset 名 / 変更した項目` を先に固定する。

1. 対象 `folder`
2. 対象 `asset`
3. 変更した項目を 1 つに固定
   - `名称変更`
   - `サムネイル変更`
   - `チャプター設定`
4. 同じ画面で元の値へ戻す
5. 一覧へ戻る
6. intended な `folder` と `asset 名` に戻っていることを確認
7. page / 会員サイト側の参照先も 1 箇所だけ再確認

`新規アップロード` の exploratory asset は、元に戻すのではなく削除して cleanup する。`チャプター設定` を exploratory に足した時は、行を消してから保存し、元の視聴体験に戻っていることまで確認して rollback 完了とする。

## 変更後の最小 smoke

asset 単体の見え方だけで判定しない。

1. intended な `folder`
2. intended な `asset 名`
3. 動画なら intended な `サムネイル変更 / チャプター設定 / 分析`
4. intended な `page / 会員サイト / レッスン` 側の参照先
5. intended な利用体験になっているか

## 最小 live test

最初の 1 本は `既存 asset を読む` か `隔離 folder に 1 件だけ置く` を優先する。

1. `メディア管理` または `動画管理`
2. representative な folder
3. representative な asset row または card
4. 動画なら
   - `サムネイル変更`
   - `チャプター設定`
   - `分析`
   の存在確認
5. 最終的に page / 会員サイト側の参照先を確認

最初から本線 asset を差し替えない。まず `どこに置かれ、どこで参照され、何を変えると downstream が変わるか` を読めることを優先する。

### save を伴う最小 live test の条件

次をすべて満たす時だけ、最小 live save を許可する。
- `テスト` か `ZZ_TEST` と分かる隔離 folder がある
- current 本線で参照されていない
- 変更対象が 1 asset に限定される
- 保存後に自分で削除または戻せる

隔離 folder がない時は、動画 / メディアの live test は `読む / 分析 / 参照先確認` までで止める。

## representative pattern を読む時の問い

- この asset 操作の主目的は `LP asset / 講義動画 / 分析改善` のどれか
- `メディア管理` で十分か、`動画管理` まで見るべきか

## 最初の複雑パターン

最初に live で詰める複雑パターンは、`名称変更 + サムネイル変更 + チャプター1行` の 3 点だけで閉じる。

1. `動画管理`
2. representative な動画を 1 本固定
3. `名称変更`
4. `サムネイル変更`
5. `チャプター設定`
6. `00:00 はじめに` のように 1 行だけ入れる
7. `分析`
8. intended な動画が開いているか確認
9. exploratory なら元に戻す

最初から
- 複数動画の一括変更
- 長いチャプター全文追加
- page 側の差し替え
を同時にやらない。

## ベストな活用場面

- 既存 page や会員サイトで使う source asset を安全に差し替えたい時
- 長尺動画の視聴体験を `チャプター設定` で改善したい時
- 動画離脱を見て、コンテンツや CTA の改善余地を探したい時

## よくあるエラー

- 用途を決めずに folder へ upload して、後から source of truth が曖昧になる
- 長尺動画なのに `チャプター設定` を付けず、視聴体験を悪くする
- `分析` を見ずに、動画内容だけで離脱理由を断定する
- page 側で個別に差し替えて、`動画管理 / メディア管理` 側の正本を見ない

## エラー時の切り分け順

1. 対象が `メディア管理` か `動画管理` かを切る
2. intended な folder に asset があるか確認する
3. 動画なら `埋め込み用URL` と `サムネイル変更` を確認する
4. 長尺動画なら `チャプター設定` の有無を確認する
5. 離脱改善なら `分析` で `play_positions / play_rates` を読む

## 分析後の改善仮説の切り方

- 冒頭で大きく落ちる
  - サムネイル
  - 導入
  - 誰向けかの明示
  を疑う
- 中盤で段差的に落ちる
  - 構成
  - テンポ
  - チャプター不足
  を疑う
- CTA 手前で落ちる
  - CTA 前の布石
  - CTA の自然さ
  - 次行動の明確さ
  を疑う
- 最後まで見られるが行動が弱い
  - 動画外の導線
  - page / 会員サイト / LINE 側の接続
  を疑う

## 参照元を先に切る順

asset を変える前に、まず次を切る。

1. どの folder の正本か
2. どの page で使うか
3. どの会員サイト / コース / レッスンで使うか
4. 既存 asset 差し替えか、新規 asset 追加か

この順を飛ばして upload から始めると、source of truth を壊しやすい。

2. folder

## 完成条件

- asset の置き場所と用途を 1 回で説明できる
- 動画なら `サムネイル / チャプター / 分析` まで意図を持って見られる
- test asset を残さない

## 検証

最低でも次を確認する。
- asset が正しい folder にある
- 動画なら `埋め込み用URL` が取れる
- `サムネイル変更` は intended な見た目になっている
- `チャプター設定` は `時刻 + 見出し` で保存できている
- `分析` は intended 期間で読める
- test asset を残していない

## 保存後の最小チェック

- intended な folder に置けているか

## 変更後の確認順

1. intended な folder にあるか
2. intended な asset 名か
3. 動画なら `サムネイル変更 / チャプター設定 / 分析` のどれを触ったか言えるか
4. intended な page / 会員サイト / レッスン側で参照できるか

asset 単体で完了にしない。`どこで使われるか` までつないで確認する。

## 構築精度だけを見る時のチェック

1. 対象が `メディア管理` か `動画管理` か最初に切れているか
2. `folder` と `用途` を 1 文で言えるか
3. asset の `source of truth` を page 側ではなく `メディア管理 / 動画管理` 側に置けているか
4. 動画なら `サムネイル変更 / チャプター設定 / 分析` のうち、どこまで必要か説明できるか
5. 長尺動画なのに `チャプター設定` を見落としていないか
6. 既存 asset の差し替えか、新規 asset 追加かを切れているか
7. test asset を残さない前提になっているか

## 症状から最初に疑う場所

- 動画はあるのに受講体験が弱い
  - まず `サムネイル`
  - 次に `チャプター設定`
  - その後に lesson 側の見せ方
- 正しい asset を置いたはずなのに意図した場所で使われない
  - まず intended な `folder`
  - 次に asset 名
  - その後に page / 会員サイト / lesson 側の参照先
- 既存動画を差し替えたつもりで別用途まで壊しそう
  - まず `差し替え` か `新規追加` か
  - `source of truth` を asset 側に戻してから判断する
- 長尺動画なのに見られ方が悪い
  - まず `チャプター設定`
  - 次に `分析`
  - 動画 upload だけで完了扱いにしない
- folder の選び方に迷う
  - まず用途
  - 次に再利用先
  - `どこで使うか` が言えないなら upload しない

## NG

- page 側で毎回ファイルを持ち直し、media/video を source of truth にしない
- folder を見ずに upload する
- 長尺動画なのに `チャプター設定` や `分析` を一度も見ない
- asset 名や用途を確認せずに流用する

## 正誤判断

正しい状態
- `メディア` と `動画` の役割を分けている
- folder を先に決めて upload している
- 動画なら `サムネイル / チャプター / 分析` を必要に応じて見ている

間違った状態
- asset を置いた場所が分からない
- 動画を上げただけで終わる
- どこで再利用されるか分からないまま差し替える

## ここで止めて確認する条件

- current の folder ルールが読めず、置き場所を決めきれない
- 既存動画の差し替えか新規動画追加か判断できない
- チャプターを付けるべきか、単純な asset として置けばよいか切れない
- どの page / 会員サイト / 講義で使われている asset か特定できない
