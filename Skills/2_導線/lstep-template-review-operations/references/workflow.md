# Lステップ テンプレートレビュー exact メモ

## 使うタイミング

- 既存テンプレートを benchmark にしたい時
- 新規作成前に良い例を分解したい時
- 作ったテンプレートをレビューしたい時

## レビューの順番

1. `テンプレート種`
2. `誰に送るか`
3. `何の認識を変えるか`
4. `何を行動させるか`
5. `画像 / テキスト / ボタン / action` の役割
6. `前後の導線との接続`

## 代表的な読み方

### 単発 CTA 型

- 何のオファーか
- urgency は十分か
- CTA が 1 つに絞られているか

### 重要告知型

- 何が変わるか
- なぜ知る必要があるか
- CTA が不要なら無理に置いていないか

### pack 型

- step 1 は認識変換か
- step 2 は行動か
- 役割が重複していないか

## 最低限見るべき UI

- `テンプレート`
- `プレビュー`
- `別窓`
- 行右端 `...`
- `テスト送信`

## テスト送信時の確認

- 実際の見た目
- 押した時の挙動
- main CTA の明確さ
- 画像と text の順番

## output の最小形

- 目的
- 変えたい認識
- 起こしたい行動
- 良い点
- 悪い点
- 再現すべき点
- 変えるならどこか
